## Introduction

The <a href="https://www.kernelgroup.it/" target="_blank" rel="noopener noreferrer">Kernel Sistemi</a> PLC KERNEL is a programmable logic controller with an ARM microprocessor and an Ethernet port. This guide walks through connecting a PLC KERNEL device to ThingsBoard PE via MQTT, configuring the LogicPaint application for telemetry reporting, and verifying data on the platform.

## Prerequisites

- PLC KERNEL device with ARM microprocessor and Ethernet port
- LogicPaint programming software installed on your PC
- Serial cable for PC-to-PLC connection
- ThingsBoard PE instance

<Aside type="note">
This procedure applies to all KERNEL PLCs with the ARM microprocessor equipped with an Ethernet port.
</Aside>

## Create a device

<Steps>
1. Log in to your ThingsBoard PE instance.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-login.png" alt="ThingsBoard PE login page" width="50%" />

2. Go to **Device Groups** and open the **ALL** group.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-device-groups.png" alt="Device Groups page listing all device group" />

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-device-group-all.png" alt="ALL device group — empty device list" />

3. Click **+** in the top-right corner of the table, then select **Add Device**.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-add-device.png" alt="Add Device dialog — enter device name and click Add" />

4. Enter a device name, for example `PLC KERNEL Device`, and click **Add**.

5. Confirm the new device appears at the top of the list.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-device-created.png" alt="Newly created PLC KERNEL device at the top of the device list" />
</Steps>

## Configure LogicPaint

### Get the access token

ThingsBoard identifies and authenticates the device using an access token. Copy it from ThingsBoard before opening LogicPaint.

<Steps>
1. Click the device row to open device details.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-device-details.png" alt="Device details panel" />

2. Click **Copy access token**. Save the token to a safe place — you will need it in the next steps.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-copy-token.png" alt="Copy access token button in device details" />
</Steps>

### Configure MQTT settings

<Steps>
1. Open LogicPaint.

   <DocImage src="/src/assets/devices-library/kernel/kernel-logicpaint-app.jpg" alt="LogicPaint application main window" />

2. Connect the PLC KERNEL to your PC via serial cable.

3. Go to **File > Show Ethernet Port Configuration**.

   <DocImage src="/src/assets/devices-library/kernel/kernel-logicpaint-ethernet-menu.png" alt="File menu with Show Ethernet Port Configuration option" />

4. Click **MQTT Configuration**.

   <DocImage src="/src/assets/devices-library/kernel/kernel-logicpaint-mqtt-button.png" alt="Ethernet port configuration — MQTT Configuration button" />

5. Paste the access token copied from ThingsBoard into the indicated field.

   <DocImage src="/src/assets/devices-library/kernel/kernel-logicpaint-token-field.png" alt="MQTT configuration — access token field with pasted token" />

6. Fill in the MQTT connection fields: broker host, port, username, and password.

   <DocImage src="/src/assets/devices-library/kernel/kernel-mqtt-connection-fields.png" alt="MQTT connection fields — broker host, port, and credentials" />

7. Add a slot for each value to be monitored.

   <DocImage src="/src/assets/devices-library/kernel/kernel-mqtt-slots.png" alt="Slot configuration — one slot per monitored sensor value" />

8. Close both open windows with the **Close** button.

9. Click **Compile + Send Application** to compile and deploy the configuration to the PLC.
</Steps>

### Verify telemetry

Once the application is running on the PLC, telemetry data appears in ThingsBoard automatically.

<Steps>
1. Click the device row to open device details.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-device-details.png" alt="Device details panel" />

2. Go to the **Latest telemetry** tab to confirm incoming readings.

   <DocImage src="/src/assets/devices-library/kernel/kernel-tb-telemetry.png" alt="Latest telemetry tab showing published sensor readings" />
</Steps>
