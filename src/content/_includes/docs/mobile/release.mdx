## Before you start

This guide assumes you have already created an app bundle in the Mobile Center and downloaded the generated `configs.json` file.

If you need to change any settings, update them in the Mobile Center, re-download `configs.json`, and replace the file in your project.

<Aside type="caution">
  Do not edit <code>configs.json</code> manually. Always manage settings in the Mobile Center to keep all environments in sync and avoid misconfigurations.
</Aside>

## iOS — build and release

Follow the official <a href="https://docs.flutter.dev/deployment/ios" target="_blank" rel="noopener noreferrer">Build and release an iOS app</a> guide to prepare your app for the <a href="https://developer.apple.com/app-store/submitting/" target="_blank" rel="noopener noreferrer">App Store</a> and <a href="https://developer.apple.com/testflight/" target="_blank" rel="noopener noreferrer">TestFlight</a>.

<Aside type="caution">
  When producing a build archive, you must include both <code>--no-tree-shake-icons</code> and <code>--dart-define-from-file</code> in your build command.
</Aside>

```bash
flutter build ipa --no-tree-shake-icons --dart-define-from-file configs.json
```

## Android — build and release

Follow the official <a href="https://docs.flutter.dev/deployment/android" target="_blank" rel="noopener noreferrer">Build and release an Android app</a> guide to prepare your app for <a href="https://support.google.com/googleplay/android-developer/answer/9859152" target="_blank" rel="noopener noreferrer">Google Play</a>.

<Aside type="caution">
  When producing a build archive, you must include both <code>--no-tree-shake-icons</code> and <code>--dart-define-from-file</code> in your build command.
</Aside>

```bash
flutter build appbundle --no-tree-shake-icons --dart-define-from-file configs.json
```

## Build flags

**`--no-tree-shake-icons`** — prevents Flutter from removing unused icons during the build. This ensures the app can display any icon configured in the notification center.

**`--dart-define-from-file configs.json`** — loads the mobile app configuration from the JSON file downloaded from the Mobile Center, so the app uses the correct settings for production.
