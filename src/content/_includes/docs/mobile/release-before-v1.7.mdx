Before publishing your app, you may want to polish it. Each app should have its own identification. Set a unique label name and app launch icon before releasing it.

## Change app name

### Install the rename package

Activate the rename package globally:

```bash
flutter pub global activate rename
```

### Set app name for both platforms

Use this command to set identical app names for both iOS and Android:

```bash
rename setAppName --targets android,ios --value "Your desired app name"
```

### Set different names for each platform

To set different names per platform, run each command separately:

```bash
rename setAppName --targets ios --value "Your iOS app name"
```

```bash
rename setAppName --targets android --value "Your Android app name"
```

## Change package name

<Aside type="caution">
  Your package names must match the package name configured in your Mobile center on the ThingsBoard platform.
</Aside>

### Set package names

```bash
rename setBundleId --targets android --value "org.yourCompany.app"
```

```bash
rename setBundleId --targets ios --value "org.yourCompany.app.ios"
```

<Aside type="caution">
  Each app must have a unique package name even if they target different platforms. That is why the `.ios` suffix is added to the iOS package name. You can use any package name you want, as long as the iOS and Android package names are different.
</Aside>

## Build and release an Android version

When preparing a release version for publishing on <a href="https://support.google.com/googleplay/android-developer/answer/9859152" target="_blank" rel="noopener noreferrer">Google Play</a>, follow the official <a href="https://docs.flutter.dev/deployment/android" target="_blank" rel="noopener noreferrer">Build and release an Android app</a> guide.

<Aside type="caution">
  When producing a build archive, you must include the `--no-tree-shake-icons` flag:
</Aside>

```bash
flutter build appbundle --no-tree-shake-icons
```

## Build and release an iOS version

When preparing a release version for publishing on the <a href="https://developer.apple.com/app-store/submitting/" target="_blank" rel="noopener noreferrer">App Store</a> and <a href="https://developer.apple.com/testflight/" target="_blank" rel="noopener noreferrer">TestFlight</a>, follow the official <a href="https://docs.flutter.dev/deployment/ios" target="_blank" rel="noopener noreferrer">Build and release an iOS app</a> guide.

<Aside type="caution">
  When producing a build archive, you must include the `--no-tree-shake-icons` flag:
</Aside>

```bash
flutter build ipa --no-tree-shake-icons
```
