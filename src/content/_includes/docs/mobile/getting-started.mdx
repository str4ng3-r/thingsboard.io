### App won't build

ThingsBoard mobile app supports specific Flutter versions. A newer Flutter version may not yet be compatible with the current release.

<Aside type="caution">
Always use the exact Flutter version specified for your app version. Otherwise, builds might fail or behave unexpectedly.
</Aside>

1. Check the release notes on GitHub: {props.product === Products.MOBILE_PE ? (

) : (

)}.

2. Check the `.fvmrc` file in the project root:

**Installing the correct Flutter version:**

**Option 1 (Recommended): Use <a href="https://fvm.app/" target="_blank" rel="noopener noreferrer">FVM</a> — Flutter Version Manager**

FVM lets you manage multiple Flutter versions across projects and switch between them per project.
Install it following the <a href="https://fvm.app/documentation/getting-started/installation" target="_blank" rel="noopener noreferrer">official guide</a>, then prefix Flutter commands with `fvm`:

```bash
fvm flutter run --dart-define-from-file configs.json
fvm flutter build apk --dart-define-from-file configs.json
```

**Option 2: Manually set the global Flutter version** (not recommended if you work on multiple projects)

```bash
cd ~/dev/flutter
git checkout 3.24.4
```

### Warning "Push notifications are not configured"

This usually means Firebase wasn't set up correctly.

<Aside type="note">
You must have completed [Step 2](#step-2-optional-configure-push-notifications) first.
</Aside>

<Steps>
1. Delete the file: `lib/firebase_options.dart`
2. Re-run the FlutterFire CLI:
   ```bash
   flutterfire configure
   ```
3. Ensure `firebase_options.dart` is regenerated and contains configuration for Android, iOS, or both.
</Steps>

### Dashboard not loading

The mobile app opens the dashboard in a WebView — the ThingsBoard platform handles all actual loading. For the dashboard to work, the app and platform must be able to communicate.

Check the log for this message:

```bash
Invoked tbMobileReadyHandler
```

- **If missing**: The platform isn't signaling the app that it's ready. Open an issue in the <a href="https://github.com/thingsboard/thingsboard" target="_blank" rel="noopener noreferrer">ThingsBoard platform repository</a>.
- **If the log appears but the dashboard still won't load**: Open an issue in the {props.product === Products.MOBILE_PE ? (
  <a href="https://github.com/thingsboard/flutter_thingsboard_pe_app/issues" target="_blank" rel="noopener noreferrer">PE mobile repository</a>
) : (
  <a href="https://github.com/thingsboard/flutter_thingsboard_app/issues" target="_blank" rel="noopener noreferrer">CE mobile repository</a>
)}.

Include credentials to your instance privately via ThingsBoard Support — never post them publicly.
