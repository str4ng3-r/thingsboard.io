Built-in GET/SET RPC methods let you read and write telemetry and attribute values on connected devices
without any additional connector configuration. The method name and parameter string are sent as the
RPC payload from the ThingsBoard RPC debug terminal on the gateway dashboard.

The following connectors support built-in GET/SET RPC methods:

<Tabs>
  <TabItem label="OPC-UA">
    Every telemetry and attribute parameter has **GET** and **SET** methods automatically, based on
    the `path` defined in the connector configuration.

    For example, given a timeseries entry:

    ```json
    {
      "timeseries": [
        {
          "key": "temperature",
          "path": "${ns=3;i=1001}"
        }
      ]
    }
    ```

    #### GET

    Read the current value of a node by its OPC-UA node ID:

    ```bash
    get ns=3;i=1001;
    ```

    Response:

    ```json
    {"result": 25.34}
    ```

  <TabItem label="Modbus">
    #### GET

    Read the value of a Modbus slave register:

    ```bash
    get type=<type>;functionCode=<functionCode>;objectsCount=<objectsCount>;address=<address>;
    ```

    | Parameter | Description |
    |-----------|-------------|
    | `type` | Data type of the value to read |
    | `functionCode` | Modbus function code |
    | `objectsCount` | Number of objects to read |
    | `address` | Register address |

    Example â€” read a 16-bit integer from register 1 (outdoor temperature):

    ```bash
    get type=16int;functionCode=3;objectsCount=1;address=1;
    ```

    Response:

    ```json
    {"result": {"value": 13}}
    ```

    <ImageGallery images={[
      {
        src: '/src/assets/images/gateway/get-set-connector-rpc/modbus-get-set-rpc-1.png',
        alt: 'Modbus GET RPC result showing register value',
        caption: 'Modbus GET â€” reading the outdoor temperature register.',
      },
    ]} />

    #### SET

    Write a value to a Modbus slave register:

    ```bash
    set type=<type>;functionCode=<functionCode>;objectsCount=<objectsCount>;address=<address>;value=<value>;
    ```

    | Parameter | Description |
    |-----------|-------------|
    | `type` | Data type of the value to write |
    | `functionCode` | Modbus function code |
    | `objectsCount` | Number of objects to write |
    | `address` | Register address |
    | `value` | Value to write |

    Example â€” write 80 to register 2 (room light level):

    ```bash
    set type=16int;functionCode=6;objectsCount=1;address=2;value=80;
    ```

    Response:

    ```json
    {"result": {"value": "80"}}
    ```

  <TabItem label="MQTT">
    #### GET

    Subscribe to a response topic and publish a request to read a value:

    ```bash
    get requestTopicExpression=<requestTopic>;responseTopicExpression=<responseTopic>;value=<value>;
    ```

    | Parameter | Description |
    |-----------|-------------|
    | `requestTopicExpression` | Topic to publish the request to |
    | `responseTopicExpression` | Topic to subscribe to for the response |
    | `value` | Value to send with the request |

    Example â€” read room light level from `data/light_level`, receiving the response on `data/response`:

    ```bash
    get requestTopicExpression=data/get_light_level;responseTopicExpression=data/response;value=${params};
    ```

    ```bash
    get requestUrlExpression=<url>;value=<value>;
    ```

    | `value` | Value to include in the request |

    Example â€” read the room light level from `http://127.0.0.1:8000/light-level`:

    ```bash
    get requestUrlExpression=http://127.0.0.1:8000/light-level;value=${params};
    ```

    Example â€” send a light level of 80 to a sensor listening on `192.168.0.200:50003`:

    ```bash
    set address=192.168.0.200;port=50003;value=80;
    ```
