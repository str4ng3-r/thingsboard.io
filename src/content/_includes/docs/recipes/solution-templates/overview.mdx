ThingsBoard provides IoT solution templates to significantly reduce the time-to-market for your IoT products.
Each solution template represents a ready-to-use PoC or MVP and includes:

- Pre-configured interactive <DocLink product={props.product} path='user-guide/dashboards' target="_blank">dashboards</DocLink>
- <DocLink product={props.product} path='concepts/data-processing' target="_blank">Data processing</DocLink> and business logic
- Sample <DocLink product={props.product} path='user-guide/devices' target="_blank">devices</DocLink> with emulated telemetry
- All required <DocLink product={props.product} path='user-guide/digital-twins/entities' target="_blank">entities</DocLink> and relations

Solution templates allow you to quickly evaluate a use case, demonstrate functionality, or use them as a baseline for further customization.
Reach for a solution template when you want a working starting point — dashboards, rule chains, and device profiles already wired together — instead of building each piece manually; you can always customize it further after installing it.

Only Tenant Administrators can install, manage, or delete solution templates.

</IoTHubBanner>

## Browse Solution Templates

<Steps>
1. In the left-hand menu, click **IoT Hub**.
2. On the IoT Hub landing page, click the **Solution Templates** card.
3. Use the **Category** filters, the **Use case** filter, or the search box to narrow down the list, and sort results (e.g. **Most installed**).
</Steps>

Each result shows a dashboard preview, the template name, its creator, and the number of installs. An already-installed template shows an **Installed** badge instead of the **Install** button.

### View Details

Click a template card (not the **Install** button) to open its details view. It includes a screenshot carousel and a description, plus the template's **Creator**, **Category**, and **Use Cases**. An extended write-up below covers highlights, key features, and real-world use cases — use this to understand what the template offers before installing it.

If the template is already installed, the details view also shows an **Installed** badge and two icons next to **Creator**, **Category**, **Use Cases**: an info icon to reopen the solution instructions, and a delete icon to remove the template.

## Install a Solution Template

For the complete step-by-step reference, including removing a template and checking for updates, see <DocLink product={Products.IOT_HUB} path='user-guides/solution-templates' target="_blank">Installing a Solution Template</DocLink> in the IoT Hub docs. The essentials:

<Steps>
1. Click **Install** on a template card, or open its details view and click **Install** there.
2. Confirm in the dialog that appears — installing creates and configures the required resources (dashboards, rule chains, devices, and other entities) within your tenant.
3. Once installation completes, a **Solution template successfully installed** dialog shows the auto-generated instructions and a table of every entity that was created.
4. Click **Go to main dashboard** to start using the solution, or **Close** to dismiss the dialog.
</Steps>

The auto-generated instructions cover the dashboard, pre-provisioned devices and their access tokens, example telemetry commands, configured <DocLink product={props.product} path='user-guide/alarm-rules' target="_blank">alarm rules</DocLink>, and — for supported templates — <DocLink product={props.product === Products.CE ? Products.EDGE : Products.EDGE_PE} path='why-thingsboard-edge' target="_blank">ThingsBoard Edge</DocLink> computing setup.

After installing, reopen these instructions at any time from the template's details view: click the info icon next to **Creator**, **Category**, **Use Cases**.

### Delete a Solution Template

Open the template's card or its details view and click the delete (trash) icon, then confirm by clicking **Remove anyway**. This removes the installed solution template along with the <DocLink product={props.product} path='user-guide/digital-twins/entities' target="_blank">entities</DocLink> that were created during installation (dashboards, rule chains, devices, and other associated resources).

<Aside type="caution">
Entities that you create yourself through the solution dashboard after installation (users, devices, etc.) are not part of the original installation and may not be removed automatically.
</Aside>

## Connecting Real Devices

Each solution template includes detailed instructions describing:

- Expected telemetry and attribute payload formats
- Required keys and data structure
- Example commands for sending data to the platform

The provided examples use auto-generated devices and credentials created during template installation.
Try the examples first to understand how the solution works, then refer to the <DocLink product={props.product} path='user-guide/connectivity-guide' target="_blank">How to connect your device</DocLink> guide to connect real devices using production credentials.
