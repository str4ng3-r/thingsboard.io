HTTP Integration exposes an HTTP endpoint on the Edge that accepts inbound device data (uplink) and can send commands back to devices (downlink). It is useful when:

- You are streaming device or asset data from an external system or connectivity provider.
- You have a custom application that posts data over HTTP.
- You want to connect a device to ThingsBoard Edge using a custom HTTP-based protocol.

## Create converter and integration templates

Integration and Converter templates are created in the Cloud (ThingsBoard PE server) and then assigned to Edge instances.

<Steps>

1. In the Cloud, go to **Edge management > Integration templates** and click the **+** button. Select **HTTP** as the type and name it **Edge HTTP Integration**, then click **Next**.

   <Aside type="note">
     Enable **Debug mode** during initial setup to inspect incoming and outgoing messages in the **Events** tab. Disable it in production to reduce overhead.
   </Aside>

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/add-http-integration-template-1-edge.png',
       alt: 'Create HTTP Integration template — type selection and name',
       caption: 'Select HTTP type and name the integration template.',
     },
   ]} />

2. Create the **Uplink converter**. The uplink converter decodes the raw HTTP payload into ThingsBoard telemetry and attributes.

   <Tabs>
     <TabItem label="TBEL (recommended)">
       ```js
       // Decode an uplink message from a buffer
       // payload - array of bytes
       // metadata - key/value object

       var data = decodeToJson(payload);

       var deviceName = data.deviceName;
       var deviceType = data.deviceType;

       var result = {
          deviceName: deviceName,
          deviceType: deviceType,
          attributes: {
              model: data.model,
              serialNumber: data.param2,
          },
          telemetry: {
              temperature: data.temperature
          }
       };

       // Helper functions 'decodeToString' and 'decodeToJson' are built-in
       return result;
       ```
     </TabItem>
     <TabItem label="JavaScript">
       ```js
       // Decode an uplink message from a buffer
       // payload - array of bytes
       // metadata - key/value object

       var data = decodeToJson(payload);

       var deviceName = data.deviceName;
       var deviceType = data.deviceType;

       var result = {
          deviceName: deviceName,
          deviceType: deviceType,
          attributes: {
              model: data.model,
              serialNumber: data.param2,
          },
          telemetry: {
              temperature: data.temperature
          }
       };

       function decodeToString(payload) {
          return String.fromCharCode.apply(String, payload);
       }
       function decodeToJson(payload) {
          var str = decodeToString(payload);
          return JSON.parse(str);
       }

       return result;
       ```
     </TabItem>
   </Tabs>

   Select **Create new**, paste the script into the Decoder function field, and click **Next**.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/add-http-integration-template-2-edge.png',
       alt: 'Uplink converter — TBEL decoder',
       caption: 'Uplink converter with TBEL decoder.',
     },
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/add-http-integration-template-2-java-edge.png',
       alt: 'Uplink converter — JavaScript decoder',
       caption: 'Uplink converter with JavaScript decoder.',
     },
   ]} />

3. Create the **Downlink converter**. The downlink converter encodes outbound messages (e.g. attribute updates) into the format expected by the device.

   <Tabs>
     <TabItem label="TBEL (recommended)">
       ```js
       // Encode downlink data from incoming Rule Engine message
       // msg        - JSON message payload
       // msgType    - e.g. 'ATTRIBUTES_UPDATED', 'POST_TELEMETRY_REQUEST'
       // metadata   - key/value pairs with message metadata
       // integrationMetadata - key/value pairs defined in the Integration

       var result = {
           contentType: "JSON",
           data: JSON.stringify(msg),
           metadata: {}
       };

       return result;
       ```
     </TabItem>
     <TabItem label="JavaScript">
       ```js
       // Encode downlink data from incoming Rule Engine message
       // msg        - JSON message payload
       // msgType    - e.g. 'ATTRIBUTES_UPDATED', 'POST_TELEMETRY_REQUEST'
       // metadata   - key/value pairs with message metadata
       // integrationMetadata - key/value pairs defined in the Integration

       var result = {
           contentType: "JSON",
           data: JSON.stringify(msg),
           metadata: {}
       };

       return result;
       ```
     </TabItem>
   </Tabs>

   Select **Create new**, paste the script into the Encoder function field, and click **Next**.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/add-http-integration-template-3-edge.png',
       alt: 'Downlink converter — TBEL encoder',
       caption: 'Downlink converter with TBEL encoder.',
     },
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/add-http-integration-template-3-java-edge.png',
       alt: 'Downlink converter — JavaScript encoder',
       caption: 'Downlink converter with JavaScript encoder.',
     },
   ]} />

4. On the **Connection** page, set the **Base URL** to the IP address and port of your Edge instance in `host:port` format.

   To reuse this template across multiple Edge instances with different addresses, use the placeholder `${{baseUrl}}` instead of a hardcoded value. The placeholder is replaced at assignment time by the value of the `baseUrl` attribute on each Edge.

   Click **Add** to save the integration template.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/add-http-integration-template-4-edge.png',
       alt: 'Integration Connection page — Base URL set to placeholder ${{baseUrl}}',
       caption: 'Set Base URL to the placeholder ${{baseUrl}} to allow per-Edge configuration.',
     },
   ]} />

</Steps>

## Modify Edge Root Rule Chain for downlinks

To send downlink messages from the Rule Engine to the HTTP integration, add an **integration downlink** node to the Edge Root Rule Chain.

<Aside type="note">
  Starting with Edge version 4.0, Rule Chains can be created and edited directly on the Edge. On earlier versions, Rule Chains must be configured as templates in the Cloud and then assigned to the Edge.

## Send uplink message

<Steps>

1. On the Edge, go to **Integration center > Integrations**, click the HTTP integration, and copy the **HTTP endpoint URL**.

2. Send a test uplink message using `curl`. Replace `$DEVICE_NAME` and `$YOUR_HTTP_ENDPOINT_URL` with actual values:

   ```bash
   curl -v -X POST \
     -d '{"deviceName":"$DEVICE_NAME","temperature":33,"model":"test"}' \
     $YOUR_HTTP_ENDPOINT_URL \
     -H "Content-Type:application/json"
   ```

3. Open the **Events** tab on the integration. A message with status **OK** should appear. Click the three dots in the **Message** column to inspect the payload.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/send-uplink-1-edge.png',
       alt: 'HTTP integration Events tab showing the endpoint URL',
       caption: 'Copy the HTTP endpoint URL from the integration, then send the uplink.',
     },
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/send-uplink-2-edge.png',
       alt: 'Events tab showing the uplink message with status OK',
       caption: 'The uplink message appears with status OK. Click the three dots to inspect it.',
     },
   ]} />

4. A new device is created automatically. Go to **Entities > Devices** on the Edge to see it.

   <ImageGallery images={[
     {

       alt: 'Newly created device in Entities > Devices on the Edge',

5. To inspect how the Uplink Converter processed the message, go to **Integration center > Data converters**, open the uplink converter, and select the **Events** tab.

## Send downlink message

<Steps>

1. On the Edge, go to **Entities > Devices**, select your device, and open the **Attributes** tab. Under **Shared attributes**, click **+** and add a new attribute — for example, key `firmware`, value `01052020.v1.1`. Save.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/add-shared-attribute-1-edge.png',
       alt: 'Adding a shared attribute to the device on the Edge',
       caption: 'Add a shared attribute (e.g. firmware) to the device. This triggers a downlink.',
     },
   ]} />

2. Verify the downlink was sent to the integration by opening the integration's **Events** tab.

   <ImageGallery images={[
     {
       src: '/src/assets/images/docs/edge/user-guide/integrations/http/downlink-message-1-edge.png',
       alt: 'Integration Events tab showing the downlink message',
       caption: 'A downlink event is recorded when the attribute update is propagated to the integration.',

   ]} />

3. Send another uplink message with the same `curl` command to receive the downlink response:

   ```bash
   curl -v -X POST \
     -d '{"deviceName":"$DEVICE_NAME","temperature":33,"model":"test"}' \
     $YOUR_HTTP_ENDPOINT_URL \
     -H "Content-Type:application/json"
   ```

   The terminal will show the response returned by ThingsBoard.
