Build ThingsBoard Edge from source on **Ubuntu 22.04 LTS / 24.04 LTS** or **CentOS 8/9**.

## Required tools

### Java 17

ThingsBoard Edge requires Java 17.

<Tabs>
<TabItem label="Ubuntu">

```bash
sudo apt update && sudo apt install openjdk-17-jdk-headless
```

Configure your operating system to use OpenJDK 17 by default:

```bash
sudo update-alternatives --config java
```

Verify the installation:

```bash
java -version
```

</TabItem>
<TabItem label="CentOS/RHEL">

```bash
sudo dnf install java-17-openjdk-headless
```

Configure your operating system to use OpenJDK 17 by default:

```bash
sudo update-alternatives --config java
```

Verify the installation:

```bash
java -version
```

</TabItem>
</Tabs>

The expected output:

```text
openjdk version "17.x.xx"
OpenJDK Runtime Environment (...)
OpenJDK 64-Bit Server VM (build ...)
```

### Maven

ThingsBoard Edge requires Maven 3.1.0 or later.

<Tabs>
<TabItem label="Ubuntu">

```bash
sudo apt-get install maven
```

</TabItem>
<TabItem label="CentOS/RHEL">

```bash
sudo yum install maven
```

</TabItem>
</Tabs>

<Aside type="caution">
  Some Linux package managers set an older JVM as the system default after installing Maven. Run <code>sudo update-alternatives --config java</code> to confirm Java 17 is active.
</Aside>

## Clone the source code

<Code code={`git clone -b ${EDGE_BRANCH} git@github.com:thingsboard/thingsboard-edge.git\ncd thingsboard-edge`} lang="bash" />

## Build

Run the following command from the `thingsboard-edge` directory:

```bash
mvn clean install -DskipTests
```

## Build local Docker images

Make sure <a href="https://docs.docker.com/engine/install/" target="_blank">Docker</a> is installed, then run:

```bash
mvn clean install -DskipTests -Ddockerfile.skip=false
```

## Build artifacts

After a successful build, the `.deb`, `.rpm`, and Windows `.zip` packages are available at:

```bash
application/target
```

## Tips

**Clear Maven cache** if you encounter dependency resolution errors:

```bash
rm -rf ~/.m2/repository
```

**Clear Gradle cache** if the UI build fails:

```bash
rm -rf ~/.gradle/caches/
```

**Clear Node modules** if the Angular frontend build fails:

```bash
rm -rf ui-ngx/node_modules
```

**Build in parallel** with license headers and Docker images:

```bash
mvn -T 0.8C license:format clean install -DskipTests -Ddockerfile.skip=false
```
