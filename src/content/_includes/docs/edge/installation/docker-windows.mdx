This guide provides step-by-step instructions for running **ThingsBoard Edge** on **Windows** using **Docker**.

<Aside type="caution" title="Version compatibility — read before proceeding">
  Before installing ThingsBoard Edge, you must have an active ThingsBoard account {props.product === Products.EDGE_PE ? <>— either a self-hosted <DocLink product={props.product} path="installation" target="_blank" bold={false}>ThingsBoard Server</DocLink> or <a href="https://thingsboard.cloud/signup" target="_blank">ThingsBoard Cloud</a> —</> : <>with a self-hosted <DocLink product={props.product} path="installation" target="_blank" bold={false}>ThingsBoard Server</DocLink>,</>} updated to the latest version.

  **Compatibility rules:**
  - ThingsBoard Edge version X.Y.Z works with ThingsBoard Server version X.Y.Z and the next **two** later versions.
    - Example: ThingsBoard Edge 3.8.0 works with ThingsBoard Server 3.8.0, 3.9.0, and 3.9.1.
  - ThingsBoard Edge version X.Y.Z **does not work** if it is newer than the ThingsBoard Server version — Edge will not start.
</Aside>

### Hardware requirements

The hardware requirements for ThingsBoard Edge depend on the number of connected devices and platform interaction intensity.

| Workload | RAM | Description |
|---|---|---|
| Light | 1 GB | Fewer than 100 devices, minimal dashboard/device management usage |
| Heavy | 4 GB | 100+ devices or intense platform interactions |

## Manual installation and configuration

If you are unable to use the guided instructions above, follow the steps below to install and configure Edge manually.

### Step 1. Copy Edge credentials

In the Edge instance details panel, click **Copy Edge Key** and **Copy Edge Secret**. Store these values securely — you need them in the next step.

### Step 2. Create Docker volumes

On Windows, ThingsBoard Edge requires pre-created named Docker volumes. Open **PowerShell** or **Command Prompt** and run:

```bash
docker volume create mytb-edge-data
docker volume create mytb-edge-logs
docker volume create mytb-edge-data-db
```

<Aside type="note" title="CLOUD_RPC_HOST">
  Set <code>CLOUD_RPC_HOST</code> to the IP address of the machine running ThingsBoard Server:

  <ul>
    <li>Do <strong>not</strong> use <code>localhost</code> — it refers to the Edge container itself, not the host machine.</li>
    {props.product === Products.EDGE_PE && (
      <li>Use <code>thingsboard.cloud</code> or <code>eu.thingsboard.cloud</code> if connecting to <a href="https://thingsboard.cloud/signup" target="_blank">ThingsBoard Cloud</a>. Also set <code>CLOUD_RPC_SSL_ENABLED</code> to <code>true</code>.</li>
    )}
    <li>Use <code>X.X.X.X</code> IP address when Edge connects to a server on the same network or Docker environment.</li>
  </ul>
</Aside>

<Aside type="caution" title="Port collision when running on the same machine">
  If ThingsBoard Edge runs on the **same machine** as the ThingsBoard Server, update the port mapping to prevent conflicts. Ensure ports **18080**, **11883**, and **15683–15688** are not in use.

  Run the following command to update `docker-compose.yml`:

  ```bash
  sed -i 's/8080:8080/18080:8080/; s/1883:1883/11883:1883/; s/5683-5688:5683-5688\/udp/15683-15688:5683-5688\/udp/' docker-compose.yml
  ```
</Aside>

### Step 4. Start ThingsBoard Edge

In the directory containing the `docker-compose.yml` file, run:

```bash
docker compose up -d && docker compose logs -f mytbedge
```

### Step 5. Open ThingsBoard Edge UI

Once the Edge service is running, open the Edge UI at <a href="http://localhost:8080" target="_blank">http://localhost:8080</a>.

{props.product === Products.EDGE_PE && (
  <p>Log in using your tenant credentials from <strong>ThingsBoard Server</strong> or <a href="https://thingsboard.cloud/signup" target="_blank"><strong>ThingsBoard Cloud</strong></a>.</p>
)}

### Step 6. Inspect logs, stop, and start

To detach from the session terminal without stopping the container, press **Ctrl+p** then **Ctrl+q**.

To view ThingsBoard Edge logs:

```bash
docker compose logs -f mytbedge
```

To stop the container:

```bash
docker compose stop mytbedge
```

To start the container:

```bash
docker compose start mytbedge
```

To stop and remove all containers and networks:

```bash
docker compose down
```

<Aside type="caution">
  On Windows, named volumes declared with <code>external: true</code> are not removed by <code>docker compose down</code>. To delete the volumes and all Edge data, run:

  ```bash
  docker volume rm mytb-edge-data mytb-edge-logs mytb-edge-data-db
  ```
</Aside>

To recreate and start all containers:

```bash
docker compose up -d
```

## Troubleshooting

### DNS issues

If you observe errors related to DNS resolution, for example:

```
127.0.1.1:53: cannot unmarshal DNS message
```

Configure your system to use Google Public DNS. For details, see the <a href="https://developers.google.com/speed/public-dns/docs/using#windows" target="_blank">Windows instructions</a>.
