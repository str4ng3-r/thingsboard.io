## Manual installation and configuration

If you are unable to use the guided instructions above, follow the steps below to install and configure Edge manually.

### Step 1. Install Java 17 (OpenJDK)

ThingsBoard Edge requires Java 17. Follow these steps:

<Steps>
<ol>
<li>
  Visit the <a href="https://adoptium.net" target="_blank" rel="noopener noreferrer">Adoptium download page</a>. Go to <strong>Other platforms and versions</strong>, select <strong>Windows</strong> as the operating system, <strong>x64</strong> as the architecture, <strong>17 - LTS</strong> as the version, and download the <strong>JDK .msi</strong> package.
</li>
<li>
  Run the downloaded MSI package and follow the installer. Make sure both <strong>Add to PATH</strong> and <strong>Set JAVA_HOME variable</strong> options are set to <strong>Will be installed on local hard drive</strong>.
</li>
<li>
  Visit the <a href="https://jdbc.postgresql.org/download/" target="_blank" rel="noopener noreferrer">PostgreSQL JDBC Driver download page</a> and download the latest available version.
</li>
<li>
  Create the folder <code>C:\Program Files\JDBC</code> and copy the downloaded <code>.jar</code> file there.
</li>
<li>
  Add a global <code>CLASSPATH</code> environment variable. Open <strong>PowerShell as Administrator</strong> and run the command below — replace the filename with the version you downloaded:

  ```powershell
  [System.Environment]::SetEnvironmentVariable("CLASSPATH", '.;"C:\Program Files\JDBC\postgresql-42.2.18.jar"', [System.EnvironmentVariableTarget]::Machine)
  ```
</li>
</ol>
</Steps>

Verify the installation by opening a Command Prompt and running:

```bat
java -version
```

Expected output:

```text
C:\Users\User>java -version
openjdk version "17.x.xx"
OpenJDK Runtime Environment Temurin-17.x.xx (...)
OpenJDK 64-Bit Server VM Temurin-17.x.xx (...)
```

#### Database configuration

Locate the `# SQL DAO Configuration` block and set your PostgreSQL password:

```yaml
# SQL DAO Configuration
spring:
  data:
    jpa:
      repositories:
        enabled: "true"
  jpa:
    hibernate:
      ddl-auto: "none"
  datasource:
    driverClassName: "${SPRING_DRIVER_CLASS_NAME:org.postgresql.Driver}"
    url: "${SPRING_DATASOURCE_URL:jdbc:postgresql://localhost:5432/tb_edge}"
    username: "${SPRING_DATASOURCE_USERNAME:postgres}"
    password: "${SPRING_DATASOURCE_PASSWORD:YOUR_POSTGRES_PASSWORD_HERE}"
```

Replace `YOUR_POSTGRES_PASSWORD_HERE` with your actual PostgreSQL password.

  |---|---|

### Step 5. Complete the installation

Open **Command Prompt as Administrator** and navigate to the ThingsBoard Edge installation directory:

```bat
cd "C:\Program Files (x86)\tb-edge"
```

Run the installation script to register ThingsBoard Edge as a Windows service — it will start automatically on system boot:

```bat
install.bat
```

Expected output:

```text
Detecting Java version installed.
CurrentVersion 170
Java 17 found!
Installing ThingsBoard Edge...
...
ThingsBoard Edge installed successfully!
```

To uninstall ThingsBoard Edge from Windows services, run `uninstall.bat` from the same directory.

Start the service:

```bat
net start tb-edge
```

Expected output:

```text
The ThingsBoard Edge Server Application service is starting.
The ThingsBoard Edge Server Application service was started successfully.
```

To stop or restart the service:

```bat
net stop tb-edge
net start tb-edge
```

Once the service is running, open the Edge UI at <a href="http://localhost:8080" target="_blank" rel="noopener noreferrer">http://localhost:8080</a>.

## Windows Firewall settings

To allow external access to the ThingsBoard Edge UI and device connectivity (HTTP, MQTT, CoAP), create a new inbound rule in Windows Firewall with Advanced Security.

<Steps>
<ol>
<li>Open <strong>Windows Firewall</strong> from Control Panel.</li>
<li>Click <strong>Advanced settings</strong> on the left panel.</li>
<li>Select <strong>Inbound Rules</strong> on the left, then click <strong>New Rule...</strong> on the right.</li>
<li>On the <strong>Rule Type</strong> step, select <strong>Port</strong>.</li>
<li>On the <strong>Protocol and Ports</strong> step, select <strong>TCP</strong> and enter <strong>8080, 1883, 5683</strong> in the <strong>Specific local ports</strong> field.</li>
<li>On the <strong>Action</strong> step, leave <strong>Allow the connection</strong> selected.</li>
<li>On the <strong>Profile</strong> step, select the Windows network profiles to apply this rule to.</li>
<li>Give the rule a name (e.g., <em>ThingsBoard Edge Networking</em>) and click <strong>Finish</strong>.</li>
</ol>
</Steps>
