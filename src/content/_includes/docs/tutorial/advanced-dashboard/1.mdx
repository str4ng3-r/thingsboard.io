## Step 1.1 Add Assets

<DocLink bold={false} product={props.product} path='user-guide/assets' target="_blank">Assets</DocLink> represent real-world objects such as buildings and office spaces.
Create four assets: **Building A**, **Building B**, **Office A**, and **Office B**.

<Steps>
1. Go to **Entities &#8702; Assets**.
2. Click **+ Add asset &#8702; Add new asset**.
3. Enter name **Building A**. 
4. In **Asset profile**, type `building` and click **Create a new one!** → **Add**.
5. Click **Add** to save.
</Steps>

Repeat to create **Building B** (profile: `building`), **Office A** (profile: `office`), and **Office B** (profile: `office`).

## Step 1.2 Import Devices

Use <DocLink bold={false} product={props.product} path='user-guide/bulk-provisioning' target="_blank">bulk import</DocLink> to create four devices at once from a CSV file.

Save the following content as `devices.csv`:

```csv
name,type,label
SD-001,air-sensor,Indoor Air Quality Sensor
EM-002,energy-sensor,Energy Meter
WM-003,water-sensor,Water Flow Meter
AM-307,air-sensor,IAQ Sensor
```

Or <a href="/resources/docs/tutorial/advanced-dashboard-guide-1/devices.csv" download="devices.csv">download a pre-made one</a>.

<Steps>
1. Go to **Entities &#8702; Devices**.
2. Click **+ Add device &#8702; Import device**.
3. Drag the CSV file into the import window and click **Continue**.
4. Verify the delimiter, then click **Continue**.
5. Map columns: **name &#8702; Name**, **type &#8702; Device Profile**, **label &#8702; Label**.
6. Click **Continue**, then **OK**.
</Steps>

## Step 1.5 OpenStreet Map Widget

The OpenStreet Map <DocLink bold={false} product={props.product} path='user-guide/widgets' target="_blank">widget</DocLink> will display Building A and Building B on the map. To place markers correctly, each building needs `latitude` and `longitude` attributes. Clicking a marker will show the building's address, contact info, and photo — also stored as attributes. Start with the preparatory work below.

### Upload Building Images

Store building photos in the <DocLink bold={false} product={props.product} path='user-guide/image-gallery' target="_blank">Image gallery</DocLink>.
The embedded links will be saved as attributes and displayed in the map tooltip.

1. Go to **Resources &#8702; Image gallery**.
2. Click **Upload image** and upload the photo for **Building A**, then for **Building B**.
3. Click the **Embed image** icon next to each uploaded image and copy the generated link.
   You will need these links in the next step.

### Add Building Attributes

Add the following <DocLink bold={false} product={props.product} path='user-guide/digital-twins/attributes' target="_blank">server-side attributes</DocLink> to **Building A**:

| Key | Type | Value |
|-----|------|-------|
| `latitude` | Double | `40.75912` |
| `longitude` | Double | `-73.97600` |
| `address` | String | `645 5th Ave, New York` |
| `email` | String | `contact@building-a.com` |
| `phone` | String | `+121 244 55 66` |
| `contactPerson` | String | `Thomas Johnson` |
| `buildingImage` | String | *(embedded link from the gallery)* |

For **Building B**, use: coordinates `40.75901` / `-73.96997`, address `641 Lexington Ave, New York`, email `contact@building-b.com`, phone `+121 555 66 77`, contact person `Jack Williams`, and the Building B embed link.

<Aside type="note">
  The `latitude` and `longitude` attributes are required by the OpenStreet Map widget to place entity markers on the map.
</Aside>

<Steps>
1. Open **Building A** details and go to the **Attributes** tab.
2. Click **+** and add each attribute from the table above.
3. Repeat for **Building B** using its own values.
</Steps>

**Appearance tab:**

- Set map provider to **CartoDB.Positron**.
- In **Label**, paste:
  ```html
  <div style="position: relative; white-space: nowrap; text-align: center; font-size: 12px; top: -5px;">
    <span style="border-radius: 10px; background-color: #fff; padding: 4px 12px;">${entityName}</span>
  </div>
  ```
- Enable **Use tooltip function** and paste the following:
  ```javascript
  var entityName = dsData[dsIndex].entityName;
  var address = data[0].address;
  var email = data[0].email;
  var phone = data[0].phone;
  var buildingImage = data[0].buildingImage;

  var html = '<div style="min-width: 240px; padding: 8px;">';
  if (buildingImage) {
    html += '<img src="' + buildingImage + '" style="width: 100%; border-radius: 8px; margin-bottom: 8px;">';
  }
  html += '<p style="font-size: 14px; font-weight: 700; margin: 0 0 6px;">' + entityName + '</p>';
  html += '<p style="margin: 0 0 4px;">' + (address || '') + '</p>';
  html += '<p style="margin: 0 0 4px;"><a href="mailto:' + (email || '') + '">' + (email || '') + '</a></p>';
  html += '<p style="margin: 0;">' + (phone || '') + '</p>';
  html += '</div>';
  return html;
  ```
- In **Marker image**, disable **Use marker image function**, upload a custom marker image, and set size to **60 px**.

### Global Widget CSS

<Steps>
1. Click **Settings** on the dashboard toolbar.
2. In **Advanced settings → Dashboard CSS**, paste:
   ```css
   .tb-widget-container > .tb-widget {
     border-radius: 8px;
     box-shadow: 0px 4px 10px rgba(23, 33, 90, 0.5);
   }
   .tb-dashboard-page .tb-widget-container > .tb-widget .leaflet-popup a.tb-custom-action {
     font-family: 'Roboto';
     font-weight: 500;
     font-size: 14px;
     line-height: 20px;
     letter-spacing: 0.25px;
     border-bottom: none;
     color: #00695C;
   }
   ```
3. Click **Save**, then save the dashboard.
</Steps>

This CSS snippet applies styles to elements with the `.tb-widget` class that are direct children of the `.tb-widget-container`.
It specifies `border-radius: 8px`, which gives the element soft rounded corners, and `box-shadow: 0px 4px 10px rgba(23, 33, 90, 0.5)`, which creates a semi-transparent shadow with a subtle blur, offset by 4 pixels downward, adding visual depth, and specifies `opacity: 0.9` to make the element slightly transparent, allowing for some background visibility while maintaining the content&#39;s visibility.
