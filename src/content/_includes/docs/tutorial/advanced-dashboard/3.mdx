## Step 3.1 Simulate Device Telemetry

ThingsBoard <DocLink bold={false} product={props.product} path='user-guide/rule-engine-2-0/overview' target="_blank">rule chains</DocLink> can generate synthetic telemetry via **Generator** nodes — useful for demos, prototyping, and testing dashboards before real hardware is connected.

Create a dedicated rule chain with four generators — one per device — each posting telemetry every 10 minutes.

<Steps>
1. Go to **Rule Chains** and click **+ &#8702; Create new rule chain**.
2. Name it `Device Telemetry Emulators` and click **Add**.
3. Open the rule chain by clicking on it.
4. Find the **Generator** node in the node palette and drag it onto the canvas.
5. Name it `Indoor air quality data emulator`. Set **Message count** to `100` and **Period (seconds)** to `600`.
6. Set **Originator** to device **SD-001** (Indoor Air Quality Sensor).
7. Paste the generator script (see below), then click **Add**.
8. Add a second Generator node named `Power consumption data emulator`. Set count `100`, period `600`, originator **EM-002**.
9. Paste the energy script, then click **Add**.
10. Add a third Generator node named `Water consumption data emulator`. Set count `100`, period `600`, originator **WM-003**.
11. Paste the water script, then click **Add**.
12. Add a fourth Generator node named `IAQ data emulator`. Set count `100`, period `600`, originator **AM-307**.
13. Paste the IAQ script, then click **Add**.
14. Drag a **Save time series** node onto the canvas and click **Add**.
15. Connect the **Success** output of each Generator to the **Save time series** node.
16. Save the rule chain.
</Steps>

**Indoor Air Quality Sensor (SD-001) and IAQ Sensor (AM-307):**

```javascript
var temperature = toFixed(Math.random() * 10 + 18, 2);
var humidity = toFixed(Math.random() * 15 + 40, 2);
var co2 = toFixed(Math.random() * 70 + 440, 2);
var msg = { temperature: temperature, humidity: humidity, co2: co2 };
var metadata = { data: 40 };
var msgType = 'POST_TELEMETRY_REQUEST';
return { msg: msg, metadata: metadata, msgType: msgType };
```

**Energy Meter (EM-002):**

```javascript
var powerConsumption = toFixed(Math.random() * 2.2, 2);
var msg = { powerConsumption: powerConsumption };
var metadata = { data: 40 };
var msgType = 'POST_TELEMETRY_REQUEST';
return { msg: msg, metadata: metadata, msgType: msgType };
```

**Water Flow Meter (WM-003):**

```javascript
var waterConsumption = toFixed(Math.random() * 1.3, 2);
var batteryLevel = toFixed(Math.random() * 1 + 45, 2);
var msg = { waterConsumption: waterConsumption, batteryLevel: batteryLevel };
var metadata = { data: 40 };
var msgType = 'POST_TELEMETRY_REQUEST';
return { msg: msg, metadata: metadata, msgType: msgType };
```

<Aside type="note">
  `toFixed(value, digits)` is a ThingsBoard helper available inside Generator scripts — it rounds a number to the specified number of decimal places and returns a number (not a string).
</Aside>

```javascript
    '</div>';

    return '<div style="display:inline-block;padding: 4px 12px;background:#d3d3d3;border-radius:20px;">' + entity.powerConsumption.toFixed(1) + ' kW</div>';
} else if (entity.waterConsumption) {
    return '<div style="display:inline-block;padding: 4px 12px;background:#d3d3d3;border-radius:20px;">' + entity.waterConsumption.toFixed(1) + ' gal</div>';
}
return value;
```

5. Click **Save**.
</Steps>

<Steps>

## Step 3.3 Configure the Dashboard Time Window

Energy and water meters measure **cumulative consumption** — each telemetry point is a rate, not a total. To display meaningful totals, configure the dashboard's default <DocLink bold={false} product={props.product} path='user-guide/dashboards#time-window' target="_blank">time window</DocLink> to use **Sum** aggregation in history mode and appropriate grouping intervals in realtime mode.

<Steps>
1. Enter dashboard edit mode and click the **Edit time window** icon on the toolbar.
2. Click the **gear** icon to open the time window settings.
3. On the **Realtime** tab:
   - **Last** interval: hide this option from users.
   - **Relative** tab: click the **pencil** icon and uncheck all intervals except **Current day**, **Current week (Sun–Sat)**, and **Current week (Mon–Sun)**. Set their grouping interval and default grouping interval to **1 hour**.
   - Check **Current month** and set its grouping to **2 hours**.
   - Click **Apply**.
4. On the **History** tab:
   - Hide the **Last** and **Range** interval options.
   - Set **Aggregation** to **Sum** and hide the aggregation selector.
   - Set grouping interval to **1 hour** and lock it (prevent users from changing it).
   - Click **Apply**.
5. Click **Update** to apply the time window settings, then save the dashboard.
</Steps>

<Aside type="note">
  Sum aggregation adds up all telemetry points in each time bucket. Combined with hourly grouping, this converts per-reading rates (e.g. 0.8 kW per measurement) into hourly totals that users can interpret as consumption figures.
</Aside>

## Step 3.5 Sensors List — Navigation to Device States

The three device states have different IDs, so a simple "navigate to state" action is not enough — the correct target depends on the **device profile** of the clicked row. Use a **Custom action** that reads the device type and routes accordingly.

<Steps>
1. In the **office** state (edit mode), click the **pencil** icon on the **Office sensors list** widget.
2. Scroll to **Actions** and click **Add action &#8702; +**.
3. Set source to **On row click** and enter an action name (e.g. `go_to_device`).
4. Set action type to **Custom action**.
5. Paste the following function:
</Steps>

```javascript
const $injector = widgetContext.$injector;
const deviceService = $injector.get(widgetContext.servicesMap.get('deviceService'));

deviceService.getDevice(entityId.id).subscribe(device => {
    if (device.type === 'energy-sensor') {
        openDashboardState('energy_sensor');
    } else if (device.type === 'water-sensor') {
        openDashboardState('water_sensor');
    } else {
        openDashboardState('air_sensor');
    }
});

function openDashboardState(stateId) {
    const params = {
        entityId: entityId,
        entityName: entityName
    };
    widgetContext.stateController.openState(stateId, params, false);
}
```

<Steps>
6. Click **Add**, review the action, then click **Save**.
7. Click **Apply**, then save the dashboard.
</Steps>

<Aside type="note">
  `widgetContext.stateController.openState(stateId, params, false)` navigates to the specified state while passing the selected entity as the state parameter — this is how the device state knows which entity to display. The third argument `false` means the new state replaces the current one rather than opening in a new browser tab.
</Aside>

## Step 3.6 Image Map — Telemetry Tooltips and Navigation

Update the **Image Map** widget (office floor plan) to show live telemetry values in marker tooltips and add a "Details >" link that navigates to the correct device state — the same routing logic as the sensors list.

**Add telemetry keys to the Image Map:**

<Steps>
1. In the **office** state (edit mode), click the **pencil** icon on the **Image Map** widget.
2. Add the following data keys to the existing datasource: `temperature`, `humidity`, `co2`, `powerConsumption`, `waterConsumption`, `batteryLevel`, `deviceType`.
</Steps>

<Aside type="tip">
  `deviceType` is a built-in entity field (not a telemetry key). In ThingsBoard widget datasources it is available as a key of type **Entity field**, not Time series — select the correct key type when adding it.
</Aside>

**Configure the tooltip:**

<Steps>
3. Go to the **Appearance** tab, scroll to **Tooltip**, and enable **Use tooltip function**.
4. Paste the following tooltip function:
</Steps>

```javascript
let tooltip = '<h1 style="margin:8px 0;width:200px;border-bottom:1px solid #0000000d;padding-bottom:8px;padding-right:15px;font-size:16px;font-weight:600;line-height:24px;">${entityLabel}</h1>';
let typeTooltip = '';

if (data.deviceType == 'energy-sensor') {
    typeTooltip =
        '<div style="display:flex;flex-direction:row;justify-content:space-between;align-items:center;gap:10px;">' +
            '<span style="font-weight:600;font-size:12px;color:#0000008a;max-width:90px;">Power consumption: </span>' +
            '<span style="font-weight:600;font-size:13px;color:#000000de">${powerConsumption:1} kW</span>' +
        '</div>';
    typeTooltip += '<div style="margin-top:17px;text-align:center;background:var(--tb-primary-50,#87CEEB);border-radius:6px;"><link-act name="sensor_details">Details ></link-act></div>';

} else if (data.deviceType == 'air-sensor') {
    typeTooltip =
        '<div style="display:flex;flex-direction:column;">' +
            '<div style="display:flex;flex-direction:row;justify-content:space-between;align-items:center;gap:10px;">' +
                '<span style="font-weight:600;font-size:12px;color:#0000008a;max-width:90px;">Temperature: </span>' +
                '<span style="font-weight:600;font-size:13px;color:#000000de">${temperature:0} °C</span>' +
            '</div>' +
```
