Dashboard **states** let you build a multi-level navigation hierarchy within a single dashboard — each state acts as a separate page scoped to the entity selected in the previous level.
In this lesson you will add **building** and **office** states to the dashboard created in Lesson 1 and connect them with widget actions so users can drill down from buildings to offices to devices.

---

## Step 2.1 Add Dashboard States

A <DocLink bold={false} product={props.product} path='user-guide/dashboards#states' target="_blank">dashboard state</DocLink> is an independent view inside the same dashboard. States share the same aliases and URL parameters, but each one can display a different set of widgets. The special `${entityName}` variable makes the state name dynamic — it is replaced at runtime with the name of the entity that triggered the navigation.

<Steps>
1. Open the **Buildings** dashboard and enter edit mode.
2. Click the **Manage dashboard states** icon in the upper-left corner of the toolbar.
3. Click **+** to add a new state. Set the name to `${entityName}` and the ID to `building`, then click **Add**.
4. Click **+** again. Set name to `${entityName}` and ID to `office`, then click **Add**.
5. Click **Save**.
</Steps>

## Step 2.2 Default State — Navigation to Building

Connect the default state to the building state in two ways: a **Tooltip tag action** on the map widget (a clickable "Details" link inside the popup), and an **On row click** action on the Buildings list table.

### Map Widget Action

<Steps>
1. Enter edit mode and click the **pencil** icon on the **OpenStreet Map** widget.
2. Go to the **Actions** tab and click **+**.
3. Set action source to **Tooltip tag action** and name it `building_details`.
4. Set action type to **Navigate to new dashboard state** and select **building**.
5. Click **Add**.
6. Go to the **Appearance** tab, find the **Tooltip** section, and append the following lines at the end of the existing tooltip HTML value:
   ```html
   + '<div style="text-align:center;background:var(--tb-primary-50, #87ceeb);border-radius:6px">' +
       '<link-act name="building_details">Details ></link-act>' +
   '</div>'
   ```
7. Save the widget, then save the dashboard.
</Steps>

<Aside type="note">
  The `<link-act name="...">` element is a ThingsBoard-specific tag that renders as a styled clickable link inside map tooltips and triggers the named widget action when clicked.
</Aside>

The full tooltip function after modification (for reference):

```javascript

return '<div style="display:flex;flex-direction:column;font-family:\'Roboto\';font-weight:500;font-size:14px;line-height:24px;letter-spacing:0.25px;color:#29313C;margin-bottom:5px">' +
           (buildingImage ? `<img style="width: 300px; height: auto" src=${buildingImage}>` : '') +
           '<div style="margin-top:5px;">${entityName}</div>' +
```

### Office Info Card

The office info card shows two sets of data from two datasources: the office's own attributes (`floor`, `officeManager`, `phone`, `email`) and the parent building's `address`, fetched via a **Relation type** alias.

**Datasources:**

<Steps>
1. In the **office** state (edit mode), click **Add widget** and select **Markdown/HTML Card** from the **Cards** bundle.
2. Set the first datasource to the **Selected entity** alias. Add keys: `floor`, `officeManager`, `phone`, `email`.
3. Click **Add datasource**.
4. Type `Building` in the new alias field and click **Create a new one!**
5. Set filter type to **Relation type**, enable **Use dashboard state entity as root**.
6. Set direction to **To**, max relation level to **1**.
7. Add a relation filter: type **Contains**, entity type **Asset**. Click **Add**.
8. Add key `address` to this second datasource.
</Steps>

<Aside type="note">
  The **Relation type** alias with direction **To** and type **Contains** traverses the relation from the selected office back to its parent building asset — this is how the building's `address` is retrieved for the office info card.
</Aside>

**Appearance tab:**

### Image Map Widget

The **Image Map** widget places device markers on a custom image using percentage-based X/Y coordinates stored as entity attributes.

#### Add Device Coordinates

Add `xCoordinate` and `yCoordinate` attributes (type **Double**, values `0`–`100`) to each device to define its position on the floor plan image.

<Steps>
1. Go to **Entities &#8702; Devices** and open each device (SD-001, EM-002, WM-003, AM-307).
2. Go to the **Attributes** tab, click **+**, and add `xCoordinate` (Double) and `yCoordinate` (Double).
3. Repeat for all devices in both offices.
</Steps>

Suggested positions on the floor plan:

| Device | xCoordinate | yCoordinate |
|--------|-------------|-------------|
| SD-001 | `28` | `45` |
| EM-002 | `72` | `30` |
| WM-003 | `55` | `70` |
| AM-307 | `40` | `50` |

<Aside type="note">
  Values are percentages relative to the image width and height. `0 / 0` is the top-left corner; `100 / 100` is the bottom-right. Adjust coordinates to match where devices are physically located on your floor plan.
</Aside>
