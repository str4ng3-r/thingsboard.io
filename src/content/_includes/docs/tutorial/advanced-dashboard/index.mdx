<Lede>A hands-on series where you build a multi-level building monitoring dashboard — from entities and maps to alarms and customer access.</Lede>

By completing it, you will learn how to:

- Create assets, devices, and hierarchical relationships
- Visualize geolocation data on an OpenStreet Map widget
- Build multi-state dashboards with drill-down navigation
- Simulate and display telemetry from multiple device types
- Configure alarm rules with propagation across entity hierarchies
- Share the dashboard securely with customers

<Aside type="tip">
At the end of each lesson, you will find an interactive checkbox to track your progress, along with navigation buttons.
Using the tracker is optional, but it can help you remember where you left off.

All data is stored locally in your browser and is not sent to or stored by ThingsBoard.
</Aside>

<Aside type="note">
Lessons 2–5 use features such as entity groups, custom ownership, and role-based access control that require **ThingsBoard Professional Edition** or **ThingsBoard Cloud**.
</Aside>

---

## What will I build?

Over the course of five lessons, you will build a **Buildings monitoring dashboard** that covers a realistic IoT scenario: two buildings, each containing offices with connected IoT sensors.

| Lesson | Topic |
|--------|-------|
| 1 | Set up entities, relationships, and add an interactive map with a buildings list |
| 2 | Add dashboard states for buildings and offices with drill-down navigation |
| 3 | Add device-level states and simulate sensor telemetry |
| 4 | Configure alarm rules and add alarm monitoring widgets |
| 5 | Share the dashboard with customers scoped to their own data |

---

## Prerequisites

Before starting, make sure you have a ThingsBoard server up and running:

- **ThingsBoard Cloud:** The fastest way to start. Click for the [North America](https://thingsboard.cloud/) or [Europe](https://eu.thingsboard.cloud/) region.
- **Self-managed:** Install using the guides for <DocLink bold={false} product={props.product} path='installation/docker-windows'>Windows (Docker)</DocLink> or <DocLink bold={false} product={props.product} path='installation/docker'>Linux and macOS (Docker)</DocLink>.

Familiarity with the basics — devices, dashboards, and widgets — is recommended. If you are new to ThingsBoard, complete the <DocLink bold={false} product={props.product} path='getting-started'>Getting Started guide</DocLink> first.
