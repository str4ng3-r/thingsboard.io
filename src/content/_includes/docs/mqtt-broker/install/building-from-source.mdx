This guide covers downloading and building TBMQ from source. Instructions are tested on Ubuntu 20.04 LTS, 22.04 LTS, and 24.04 LTS.

## Required tools

This section describes the build tools required to compile TBMQ from source.

### Java

TBMQ is built using Java 17. Install OpenJDK 17:

```bash
sudo apt update
sudo apt install openjdk-17-jdk-headless
```

Configure your operating system to use OpenJDK 17 by default. Set the default version with the following command:

```bash
sudo update-alternatives --config java
```

Verify the installation:

```bash
java -version
```

Expected output:

```text
openjdk version "17.0.xx"
OpenJDK Runtime Environment (...)
OpenJDK 64-Bit Server VM (build ...)
```

### Maven

TBMQ requires Maven 3.6.3 or later:

```bash
sudo apt install maven
```

<Aside type="caution">
  Maven installation may set Java 7 as the default JVM on some Linux systems. If that happens, follow the <a href="#java">Java instructions</a> above to restore Java 17 as the default.
</Aside>

## Build

From the `tbmq` directory, build the project:

```bash
mvn clean install -DskipTests
```

## Build local Docker images

Make sure <a href="https://docs.docker.com/engine/install/" target="_blank" rel="noopener noreferrer">Docker</a> is installed, then run:

```bash
mvn clean install -DskipTests -Ddockerfile.skip=false
```

## Build artifacts

Debian, RPM, and Windows packages are located in the target folder:

```bash
cd application/target
```

## Running tests

<a href="https://docs.docker.com/engine/install/" target="_blank" rel="noopener noreferrer">Docker</a> is required to run integration and black-box tests.
Make sure your user can <a href="https://docs.docker.com/engine/install/linux-postinstall/#manage-docker-as-a-non-root-user" target="_blank" rel="noopener noreferrer">run Docker without root</a>.

<Aside type="tip">
  Main and release branches are already tested in CI — you can skip tests to avoid requiring Docker locally.
</Aside>

Run all unit and integration tests:

```bash
mvn clean install
```

## Tips and tricks

The following tips can help with common build issues:

- **Clean <a href="https://www.baeldung.com/maven-clear-cache" target="_blank" rel="noopener noreferrer">Maven cache</a>:**
  ```bash
  rm -rf ~/.m2/repository
  ```

- **Clean Gradle cache:**
  ```bash
  rm -rf ~/.gradle/caches/
  ```

- **Clean node modules:**
  ```bash
  rm -rf ui-ngx/node_modules
  ```

- **Build in parallel, format license headers, and build Docker images:**

  ```bash
  mvn -T 0.8C license:format clean install -DskipTests -Ddockerfile.skip=false
  ```
