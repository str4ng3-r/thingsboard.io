**Assets** are abstract entities in ThingsBoard that represent physical objects, spaces, or organizational units — buildings, rooms, production areas, fields, streetlights, substations, or vehicles. Unlike devices, assets don't generate telemetry on their own but can aggregate data from related devices and participate in rule chains, alarms, and dashboards.

## Assets vs. Devices

Use a <DocLink product={props.product} path='user-guide/devices' target="_blank">Device</DocLink> for anything that connects to ThingsBoard and sends its own telemetry — a sensor, a controller, a gateway. Use an **Asset** for something that doesn't connect directly but still needs to exist as an entity — a building, a room, a vehicle, a production line — so it can group related devices into a hierarchy or aggregate their data.

A common pattern: model the physical structure with assets (`Building → Floor → Room`), then relate each room to the devices physically inside it via <DocLink product={props.product} path='user-guide/digital-twins/relations' target="_blank">Relations</DocLink>. Telemetry rolls up to the asset through <DocLink product={props.product} path='user-guide/calculated-fields' target="_blank">Calculated Fields</DocLink> or is queried across the whole hierarchy on a dashboard.

## Managing Assets

### Create an Asset

<Steps>
1. Navigate to **Entities &#8702; Assets**.
2. Click **+ Add asset** and select **Add new asset** — or **Import assets** to create many at once from a CSV file (see <DocLink product={props.product} path='user-guide/provisioning/#bulk-provisioning' target="_blank">Bulk Provisioning</DocLink>).
3. Enter a **Name** (must be unique within the tenant).
4. Optionally select an <DocLink product={props.product} path='user-guide/asset-profiles' target="_blank">Asset profile</DocLink> (defaults to `default`), **Label**, or **Description**.
5. Click **Add**.
</Steps>

</Aside>
)}

<Aside type="tip">
  **Example:** an asset hierarchy `Building A → Room 101, Room 102`, each room related to its temperature sensor via a **Contains** relation — the building's dashboard then shows data aggregated from every room.
</Aside>

### Edit an Asset

<Steps>
1. Click the asset in the list to open its details.
2. Click the **pencil** icon.
3. Update the name, label, profile, or description.
4. Click **Apply changes**.
</Steps>

### Delete an Asset

Click the **trash** icon next to the asset and confirm, or open the asset details page and click **Delete asset**.

<Aside type="note">
  Deleting an asset does **not** delete related devices or other entities, but it **breaks all existing relations** (links to devices, alarms, dashboards, etc.) — any dashboard widget or entity alias built on those relations will need to be updated.
</Aside>

## Asset Details

| Action | Description |
|--------|-------------|
| **Open details page** | Opens the asset on its own full page instead of a side panel. |
| **Manage owner and groups** | Change the asset's owner and group membership (PE/Cloud only). |
| **Delete asset** | Deletes the asset — see **[Delete an Asset](#delete-an-asset)**. |
| **Copy asset Id** | Copies the asset's unique identifier, e.g. for use with the <DocLink product={props.product} path='reference/rest-api' target="_blank">REST API</DocLink>. |

## Managing Relations

You can create, view, edit, and delete <DocLink product={props.product} path='user-guide/digital-twins/relations' target="_blank">relations</DocLink> between an asset and other entities from the **Relations** tab of the asset details page.

**Add a relation:**
<Steps>
1. Open the asset details → **Relations** tab.
2. Select the direction: **From** (outbound) or **To** (inbound).
3. Click **+**.
4. Set the **Relation type** and choose the target entity — see <DocLink product={props.product} path='user-guide/digital-twins/relations/#relation-types' target="_blank">Relation Types</DocLink> for the full list (Contains, Manages, Uses, Supports).
5. Click **Add**.
</Steps>

<Aside type="tip">
  Relations are directional. Use "From" to define relations originating from the current asset, or "To" to show relations pointing to this asset.
</Aside>

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Dashboard widget shows no data for an asset | The widget's entity alias doesn't resolve to the asset, or the asset has no related device sending data | Check the alias in <DocLink product={props.product} path="user-guide/dashboards/#entity-aliases" target="_blank">Entity Aliases</DocLink>, and confirm the asset has a relation to a device that's actively sending telemetry |
| Calculated field on an asset never produces a value | The source device isn't related to the asset yet, or the calculated field was defined at the wrong level | Check the asset's **Relations** tab, and confirm whether the calculated field belongs on the asset or on its <DocLink product={props.product} path="user-guide/asset-profiles" target="_blank">Asset Profile</DocLink> |
| Can't find a way to share a single asset | {props.product === Products.CE ? "Individual assets are made public or assigned to one customer, not \"shared\"" : "Assets aren't shareable individually"} | {props.product === Products.CE ? <>Assign it to a customer, or make it public — see <DocLink product={props.product} path="user-guide/assets/#assign-to-customer">Assign to Customer</DocLink></> : <>Share the asset group that contains it instead — see <DocLink product={props.product} path="user-guide/groups" target="_blank">Entity Groups</DocLink></>} |
| Dashboard still references a deleted asset | Deleting an asset breaks its relations but not the dashboard widgets built on them | Update the affected entity aliases or widgets to point at a different asset |

## Where Assets Are Used

- <DocLink product={props.product} path='user-guide/dashboards'>Dashboards</DocLink> — visualize aggregated data, attributes, and alarms via widgets
- <DocLink product={props.product} path='reference/widgets/widget-library/maps' target="_blank">Maps</DocLink> — plot asset locations (buildings, vehicles, fields) and show related device data directly on the map
- <DocLink product={props.product} path='user-guide/rule-engine'>Rule Engine</DocLink> — process attribute updates, trigger alarms, route messages
- <DocLink product={props.product} path='user-guide/calculated-fields'>Calculated Fields</DocLink> — compute derived values by aggregating data from related devices
- <DocLink product={props.product} path='user-guide/digital-twins/relations'>Relations</DocLink> — model hierarchies (e.g., Building → Contains → Room → Contains → Sensor)
- <DocLink product={props.product} path='user-guide/entity-views' target="_blank">Entity Views</DocLink> — expose a subset of an asset's telemetry and attributes to a Customer without granting full access
- <DocLink product={props.product} path='reference/rest-api'>REST API</DocLink> — query, create, and manage assets programmatically
