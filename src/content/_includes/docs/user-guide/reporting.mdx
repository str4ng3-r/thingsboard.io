<Aside type="note">
  Reporting is available in **ThingsBoard Professional Edition** starting from version 4.2.
</Aside>

**Reporting** generates PDF or CSV files from live platform data and delivers them automatically — on a schedule or when triggered by a system event such as an alarm.

A report is composed of two things: a **report template** that defines the content and layout, and a **scheduler event** (or notification rule) that triggers generation and delivery.

---

## Report templates

A report template defines what appears in the generated file. Templates are reusable — the same template can be scheduled for multiple customer groups or attached to multiple notification rules.

### Format and type

| Setting | Options |
|---------|---------|
| **Format** | PDF, CSV |
| **Type** | **Report** — standalone, scheduled or triggered by a notification; **Subreport** — embedded inside a master report |

### Create a template

<Steps>
1. Navigate to **Reporting**.
2. Click **+ Add report template**.
3. Enter a **name**, select **format** and **type**.
4. Click **Add**.
</Steps>

The template opens in the **Report Builder** — a drag-and-drop editor where you compose the layout.

### Import a template

<Steps>
1. Navigate to **Reporting**.
2. Click **Import report template**.
3. Upload the `.json` file and click **Import**.
</Steps>

## Report Builder

The Report Builder is a drag-and-drop editor. Components are dragged from the **Components library** panel into the **content area**. Each component has two configuration tabs: **Data** (data source, filters, series) and **Layout** (sizing, margins, alignment, background).

### Component categories

| Category | What it provides |
|----------|-----------------|
| **Text & Images** | Headings, rich text, images, dividers, page breaks |
| **Tables** | Entity table (entity data and attributes), Alarm table (alarm history) |
| **Charts** | Line, Bar, Pie, State, Range, and other chart types |
| **Dashboard** | Snapshot of a selected ThingsBoard dashboard |
| **Subreport** | Embeds a subreport template — one instance per entity |
| **Branding** | Logo and color theme elements |
| **Layout & Info** | Page number, total pages, report creation timestamp |

---

## Variables

Type `$` in any text field to open the variable picker and insert a dynamic value.

### System variables

| Variable | Description |
|----------|-------------|
| `${pageNumber}` | Current page number |
| `${totalPages}` | Total page count |
| `${reportCreatedTime}` | Report generation timestamp |
| `${entityName}` | Name of the current entity in context |
| `${entityLabel}` | Label of the current entity in context |
| `${rowCount}` | Row count in the current table |

### Data source variables

Any telemetry key or attribute returned by a component's data source is available as `${keyName}`. For example, a device sending `temperature` exposes `${temperature}`.

---

## Entity aliases

Report templates support all standard ThingsBoard entity aliases, plus two additional ones for notification-triggered reports:

| Alias | Resolves to |
|-------|------------|
| **Originator entity** | The entity that triggered the notification (e.g. the device that raised an alarm) |
| **Owner of originator entity** | The customer or tenant that owns the originator entity |
