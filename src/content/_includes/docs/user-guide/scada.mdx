SCADA dashboards provide a specialized layout for building interactive supervisory control interfaces. Available since ThingsBoard 3.8.0, the SCADA layout uses absolute positioning and fixed-pixel sizing so that every symbol stays exactly where you place it — ideal for process diagrams, factory floor plans, and energy system schematics.

## SCADA dashboard layout

The key difference between the SCADA layout and the default layout is the way widgets are positioned. In a default layout, widgets are placed in a grid. In a SCADA layout, widgets use absolute positioning with no grid snapping:

- **Margin** between the edge of the dashboard and the SCADA area (in pixels).
- **Widget appearance** — each SCADA symbol widget is placed at an exact pixel position and sized in pixels, not grid cells.

## Building a SCADA dashboard

<Steps>
1. Navigate to the **Dashboards** page and click the **+** icon. Select **Create new dashboard**.
2. On the new dashboard, enter **Edit mode**. Open the **Layout settings** dialog and switch the layout to **SCADA**.
3. Save the dashboard.
</Steps>

## SCADA symbol

A SCADA symbol is an SVG element that represents a piece of equipment, a valve, a pump, or any other process component. Symbols include built-in behavior tags that let them interact with device data.

ThingsBoard provides 100+ SCADA symbols out of the box, with plans to increase to 500+. You can also create <DocLink product={props.product} path="user-guide/scada-symbol-dev">custom SCADA symbols</DocLink>.

Starting with ThingsBoard 3.9, **high-performance SCADA symbols** are available — optimized graphical elements designed for large-scale SCADA dashboards:

- **Simplified design** — clean, standardized visuals that reduce rendering overhead.
- **Color coding** — status communicated through color changes.
- **Dynamic changes** — real-time visual updates reflecting process state.
- **Standardization** — consistent look and feel across all diagrams.
- **Compatibility** — work seamlessly on different devices and screen sizes.

## SCADA symbol widget

The **SCADA symbol** widget displays an SVG-based SCADA symbol on the dashboard and connects it to a target entity for live data.

<Steps>
1. In dashboard edit mode, click **+ Add widget** and select the **SCADA symbol** widget.
2. Choose a SCADA symbol from the library.
3. Configure the target entity, behavior, and appearance. Click **Add**.
</Steps>

### Behavior

Behavior settings define how the symbol retrieves and sends data. Each behavior parameter has an **action type** that determines the data source or target.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/user-guide/scada/scada-behavior-settings-1.png', alt: 'Behavior settings', caption: 'Configure behavior settings for the SCADA symbol.' },
]} />

**Action types to retrieve value:**

| Action type | Description |
|-------------|-------------|
| **Do nothing** | The parameter uses a fixed default value |
| **Execute RPC** | Sends an RPC request and uses the response |
| **Get attribute** | Reads an entity attribute |
| **Get time series** | Reads the latest time-series value |
| **Get alarm status** | Reads the alarm status of the entity |
| **Get dashboard state** | Reads a value from the current dashboard state |
