The **Applications** tab in the <DocLink product={props.product} path="user-guide/mobile-app-center">Mobile App Center</DocLink> is a control center for your mobile applications. It allows you to:

- **Add** new mobile application configurations to integrate with ThingsBoard.
- **Edit** existing application details — package names, statuses, versions, or store information.
- **Delete** obsolete or unused application configurations.

## Add application

<Steps>
1. Go to the **Applications** tab on the **Mobile center** page, and click **+ Add application** in the upper-right corner.
2. Enter the **application package** name.
3. Select the **platform type**: Android or iOS.
4. An **Application Secret** will be generated automatically. Save it or replace with your own.
5. Set the **application status**:
   - **Draft** — for applications still in development. Store information can be added later.
   - **Published** — for applications ready to be launched and used by end users.
   - **Deprecated** — for older applications that are no longer actively supported but still available.
   - **Suspended** — for applications that are temporarily disabled or restricted.
6. (Optional) Specify minimum/latest app versions and release notes.
7. Add **store information**:
   - **Android**: Google Play Store link and SHA256 certificate fingerprints.
   - **iOS**: App Store link and App ID.
8. Click **Add** to save the application.
</Steps>

## Edit application

You can edit all fields of the mobile application except for the platform type.

<Steps>
1. Open the application details by clicking on it.
2. Enter edit mode by clicking the **pencil** button.
3. Make the necessary changes.
4. Save the changes.
</Steps>
