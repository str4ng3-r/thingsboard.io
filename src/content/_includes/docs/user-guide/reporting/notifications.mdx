A report can be attached to any notification template and delivered automatically when a system event occurs — such as when an alarm is created or cleared.

This is useful for combining platform alerts with analytical context: instead of receiving a bare alarm notification, recipients get an email with a PDF summarising the event and related data.

**Prerequisites:** Familiarity with <DocLink product={props.product} path="user-guide/reporting">Reporting concepts</DocLink> and <DocLink product={props.product} path="user-guide/notifications">Notifications</DocLink>.

<Aside type="note">
  Customer users can only receive report-bearing notifications via **Email** or **Slack** — not the Web UI.
</Aside>

---

## How it works

When a notification rule fires, ThingsBoard:
1. Generates the specified report using the configured user account's permissions.
2. Attaches the resulting PDF (or CSV) to the outgoing notification.
3. Delivers the notification via the channels configured in the notification template.

The report is generated fresh each time the notification fires — it reflects data at the moment the event occurred.

---

## Step 1. Create the report template

Create a report template designed for the event type. For alarm notifications, the **Originator entity** alias is particularly useful — it automatically resolves to the entity that triggered the alarm.

<Steps>
1. Navigate to **Reporting**.
2. Click **+ Add report template** — **Format:** *PDF*, **Type:** *Report*.
3. Add an **Alarm table** component:
   - **Entity alias:** *Originator entity*
   - **Alarm filter:** *Active alarms*
4. Add any additional components (charts, device info, etc.).
5. Save the template.
</Steps>

---

## Step 2. Attach the report to a notification template

<Steps>
1. Navigate to **Notifications &#8702; Templates**.
2. Open or create the notification template (e.g. *New alarm notification*).
3. Add or edit the **Email** delivery configuration.
4. Enable **Attach report**.
5. Select the report template created in Step 1.
6. **User:** specify the account whose data access permissions are used to generate the report (e.g. a Tenant Administrator).
7. Save the template.
</Steps>

---

## Step 3. Test the setup

<Steps>
1. Configure an alarm rule on a device (e.g. trigger when `temperature > 40`).
2. Publish telemetry that exceeds the threshold to create an alarm.
3. Verify that a notification appears in the Web UI.
4. Check the recipient's email inbox — the message should include the PDF attachment.
</Steps>

## Example: New alarm with PDF summary

| Component | Configuration |
|-----------|--------------|
| Notification template | *New alarm notification* — Email delivery, *Attach report* enabled |
| Report template | Alarm table using *Originator entity* alias, active alarm filter |
| User account | Tenant Administrator (full data access) |
| Delivery | Email with PDF attached, Web UI notification (no attachment) |

Use this pattern for any event-driven report: replace the alarm rule with any notification trigger supported by the <DocLink product={props.product} path="user-guide/notifications">Notification system</DocLink>.
