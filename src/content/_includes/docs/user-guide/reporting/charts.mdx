Chart components visualise time-series telemetry directly in a report. ThingsBoard supports 11 chart types, each configurable with its own data source, series, axes, and legend.

**Prerequisites:** A <DocLink product={props.product} path="user-guide/reporting">report template</DocLink> must exist before adding a chart.

---

## Chart types

| Chart type | Best for |
|------------|---------|
| **Time series chart** | Multiple telemetry keys over time, shared time axis |
| **Line chart** | Continuous trends, comparing entities or keys |
| **Bar chart** | Aggregated totals or side-by-side comparisons |
| **Point chart** | Scattered individual measurements |
| **State chart** | Boolean or discrete-state values over time |
| **Bar chart with labels** | Bar chart with value labels rendered on bars |
| **Range chart** | Min/max range band around a central series |
| **Bars** | Stacked or grouped bar visualization |
| **Pie** | Proportional breakdown of a single snapshot value |
| **Doughnut** | Same as Pie with a hollow center |
| **Horizontal doughnut** | Horizontal layout doughnut |

## Add a chart to a template

<Steps>
1. Open the report template in the Report Builder.
2. Locate the desired chart type in the **Components library** panel.
3. Drag it into the content area.
4. Configure the **Data** and **Layout** tabs (see below).
5. Click **Save**.
</Steps>

---

## Configuration

### Data tab

| Setting | Description |
|---------|-------------|
| **Time window** | Relative or absolute time range for the data (e.g. *Last 24 hours*, *Last 7 days*). Also sets the aggregation interval. |
| **Datasource** | *Device* (fixed device) or *Entity alias* (dynamic, resolves at runtime) |
| **Entity filter** | Filters applied to the entity alias result |
| **Series** | One entry per telemetry key. Each series has a **key** (telemetry key name) and a **label** (displayed in the legend; supports variables such as `${entityName}`). |

### Layout tab

| Setting | Description |
|---------|-------------|
| **Width / height** | Component dimensions in the report |
| **Alignment** | Left, center, or right |
| **Padding / margin** | Spacing around the component |
| **Background** | Fill color |
| **Border** | Border style and color |

### Y axis

| Setting | Description |
|---------|-------------|
| **Min / Max** | Fixed range, or leave empty for auto-scale |
| **Label** | Axis label text |
| **Tick count** | Number of grid lines |

### Thresholds

Add one or more horizontal threshold lines with a fixed value and color. Useful for highlighting acceptable ranges or alert levels.

### X axis

Controls the time axis: label format, tick density, and whether to show/hide the axis.

### Legend

| Setting | Description |
|---------|-------------|
| **Position** | Bottom, right, or hidden |
| **Show Min / Max / Avg / Total** | Aggregate values displayed per series in the legend |

## Example: Line chart comparing temperature across devices

**Goal:** Show temperature readings from 3 devices over the last 24 hours.

<Steps>
1. Open the report template in the Report Builder.
2. Drag a **Line chart** into the content area.
3. **Data tab:**
   - **Time window:** *Last 24 hours*, aggregation *1 hour*.
   - **Entity alias:** select an alias returning 3 devices (e.g. *My devices*).
   - **Series:** key `temperature`, label `${entityName} Temperature`.
4. **Y axis:** min *15*, max *35*.
5. **Legend:** position *Bottom*, enable *Min*, *Max*, *Avg*.
6. Title: *Temperature — Last 24 Hours*.
7. Save the template and click **Generate test report** to preview.
</Steps>

The generated PDF contains a line chart with one series per device, each labelled with the device name.

<Aside type="tip">
  If your entity alias returns a variable number of devices, the chart automatically adds one series per resolved entity at report generation time.
</Aside>
