The **Scheduler** automates report generation and delivery based on a pre-defined template. A single template can be scheduled multiple times — for different customers, time zones, or recipient groups — without duplication.

**Prerequisites:** At least one <DocLink product={props.product} path="user-guide/reporting">report template</DocLink> must exist before creating a scheduler event.

---

## Create a scheduled report

<Steps>
1. Navigate to **Reporting &#8702; Scheduling**.
2. Click **+ Scheduled report**.
3. Configure the event (see fields below).
4. Click **Add**.
</Steps>

### Scheduler event fields

| Field | Description |
|-------|-------------|
| **Name** | Unique name for this scheduler event |
| **Event type** | Always *Generate Report* |
| **Report template** | The template to use for generation |
| **User** | The user account whose data access permissions are applied when the report runs. The report only includes data this user can see. |
| **Time zone** | Time zone used for interpreting the schedule and for relative time windows in the report |
| **Recipients** | One or more user groups or individual users. Configure delivery channels (Email, Web, Slack) per recipient. |
| **Notification template** | The notification template used to wrap the report delivery message |
| **Start time** | When the first report generation should occur |
| **Repeat** | *Daily*, *Weekly*, or *Monthly* |

<Aside type="note">
  Web UI delivery is available to tenant users only. Customer users receive reports via Email or Slack.
</Aside>

---

## View generated reports

All reports produced by the scheduler are stored and accessible on the **Reports** tab.

<Steps>
1. Navigate to **Reporting &#8702; Reports**.
</Steps>

Available actions per report:

| Action | Description |
|--------|-------------|
| **Download** | Download the generated PDF or CSV file |
| **Delete** | Remove the report from storage |
| **Filter by template** | Show only reports from a specific template |
| **Filter by user** | Show only reports generated on behalf of a specific user |

<Aside type="note">
  The maximum number of stored reports per tenant can be configured by the system administrator in the Tenant Profile settings.
</Aside>
