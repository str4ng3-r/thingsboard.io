The **Dashboard** component captures a snapshot of a ThingsBoard dashboard and embeds it as an image in the report. The snapshot uses the same data and widget configuration as the live dashboard, rendered at a fixed point in time.

This is useful for scheduled reports, data archiving, or sharing dashboard views with recipients who do not have direct platform access.

**Prerequisites:** A <DocLink product={props.product} path="user-guide/reporting">report template</DocLink> and at least one <DocLink product={props.product} path="user-guide/dashboards">dashboard</DocLink> must exist.

---

## Add a Dashboard component

<Steps>
1. Open the report template in the Report Builder.
2. Drag the **Dashboard** component from the Components library into the content area.
3. Configure the **Data** and **Layout** tabs (see below).
4. Save the template and click **Generate test report** to preview.
</Steps>

---

## Configuration

### Data tab

| Setting | Description |
|---------|-------------|
| **Datasource** | *Device* (fixed) or *Entity alias* (dynamic). If the alias returns multiple entities, a separate dashboard snapshot is generated for each entity in the report. |
| **Target dashboard** | The dashboard to snapshot |
| **Dashboard state** | Optional — open the dashboard in a specific state |
| **Time window** | Override the dashboard's default time window for the snapshot |
| **Time zone** | Time zone used for rendering the snapshot |

<Aside type="tip">
  When the datasource resolves to multiple entities, the report contains one dashboard snapshot per entity — useful for generating per-device or per-customer sections in a single run.
</Aside>

### Layout tab

| Setting | Description |
|---------|-------------|
| **Width** | Component width in the report |
| **Alignment** | Left, center, or right |
| **Padding / margin** | Spacing around the image |
| **Background** | Fill color behind the image |
| **Border** | Border style and color |

## Example: Embed the "Environmental Monitor" dashboard

**Goal:** Add a per-device dashboard snapshot showing temperature and humidity to a scheduled report.

<Steps>
1. Ensure a device is sending `temperature` and `humidity` telemetry.
2. Import or create the *Environmental Monitor* dashboard.
3. Open the report template in the Report Builder.
4. Drag a **Dashboard** component into the content area.
5. **Data tab:**
   - **Datasource:** the device (or an alias that returns it).
   - **Target dashboard:** *Environmental Monitor*.
6. Save the template and click **Generate test report**.
</Steps>

The generated PDF contains a rendered image of the dashboard showing live data from the selected device.
