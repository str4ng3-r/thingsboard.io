This guide walks you through creating a **Daily Device Alarm Report** — a PDF with a device table, alarm table, and two line charts — then scheduling it for two separate customer groups.

**Prerequisites:** Familiarity with <DocLink product={props.product} path="user-guide/reporting">Reporting concepts</DocLink> and <DocLink product={props.product} path="user-guide/notifications">Notifications</DocLink>.

---

## Step 1. Create the report template

<Steps>
1. Navigate to **Reporting**.
2. Click **+ Add report template**.
3. **Name:** *Daily Device Alarm Report* — **Format:** *PDF* — **Type:** *Report*.
4. Click **Add**.
</Steps>

The template opens in the Report Builder.

---

## Step 2. Build the template

### 2.1 Add a heading

Drag a **Heading** component into the content area. Set the text to *Daily Device Alarm Report*.

### 2.2 Add introductory text

Drag a **Rich text** component below the heading. Enter a brief description, for example:
*This report contains the current device list, alarms from the last 24 hours, and telemetry charts.*

### 2.3 Add a device table

Drag an **Entity table** component.

- **Data tab:**
  - **Entity alias:** select or create an alias that returns the target devices.
  - **Columns:** add *Name*, *Temperature*, *Humidity*, *Active* (map each to the corresponding telemetry key or attribute).
- **Layout tab:** adjust column widths as needed.

### 2.4 Add an alarm table

Drag an **Alarm table** component.

- **Data tab:**
  - **Entity alias:** same alias as the device table.
  - **Time window:** *Last 24 hours*.
  - **Alarm filter:** *Active alarms*.
- Update the component title to *Active Alarms — Last 24 Hours*.

### 2.5 Add a temperature line chart

Drag a **Line chart** component.

- **Data tab:**
  - **Time window:** *Last 24 hours*, aggregation *1 hour*.
  - **Entity alias:** same device alias.
  - **Series:** add key `temperature`, label `${entityName} Temperature`.
- **Layout tab:**
  - **Y axis:** min *15*, max *35* (°C).
  - **Legend:** position *Bottom*, show *Min*, *Max*, *Avg*.
- Title: *Temperature — Last 24 Hours*.

### 2.6 Add a humidity line chart

Repeat step 2.5 for `humidity`. Title: *Humidity — Last 24 Hours*.

### 2.7 Set the file name pattern and save

At the top of the Report Builder, set the **File name** to a pattern such as `daily-alarm-report-${reportCreatedTime}`. Click **Save**.

<Aside type="tip">
  Click **Generate test report** (top right of the Report Builder) to preview the output before scheduling.
</Aside>

---

## Step 3. Schedule the report

### 3.1 Schedule for Customer A

<Steps>
1. Navigate to **Reporting &#8702; Scheduling**.
2. Click **+ Scheduled report**.
3. **Name:** *Daily Alarm Report — Customer A*.
4. **Report template:** *Daily Device Alarm Report*.
5. **User:** select a user from Customer A (e.g. *janesmith@thingsboard.io*). Reports are generated using this user's data access permissions.
6. **Time zone:** choose the customer's local time zone.
7. **Recipients:** add the *Customer users* group for Customer A. Select **Email** and **Web** as delivery channels.
8. **Schedule:** *Daily*, set the desired delivery time.
9. Click **Add**.
</Steps>

### 3.2 Schedule for Customer B

Repeat step 3.1 with a Customer B user and Customer B recipient group. The same report template is reused — no duplication required.

---

## Result

Both customer groups receive a daily notification:
- **Web UI** — the notification appears in the platform notification center (visible to tenant users).
- **Email** — a message with the PDF report attached is delivered to each recipient.

Each report reflects data accessible to the specified user at the time of generation.

---

## Import the example template

You can skip building manually by importing a pre-built template:

<Steps>
1. Download the `daily-alarm-report.json` template file.
2. Navigate to **Reporting**.
3. Click **Import report template**, upload the file, and click **Import**.
</Steps>

Edit the imported template's entity aliases to match your own devices before scheduling.
