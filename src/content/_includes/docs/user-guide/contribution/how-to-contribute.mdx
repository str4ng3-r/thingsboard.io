## Prerequisites

Install the following tools before proceeding:

| Tool | Version | Notes |
|------|---------|-------|
| Java | 17 | <a href="https://adoptium.net/" target="_blank" rel="noopener noreferrer">Eclipse Temurin</a> or <a href="https://openjdk.org/" target="_blank" rel="noopener noreferrer">OpenJDK</a> |
| Maven | 3.6+ | <a href="https://maven.apache.org/install.html" target="_blank" rel="noopener noreferrer">Apache Maven</a> |
| Docker + Docker Compose | Latest | Required for integration and black-box tests |

## Fork and build

Fork the <a href="https://github.com/thingsboard/thingsboard" target="_blank" rel="noopener noreferrer">thingsboard/thingsboard</a>
repository on GitHub and clone it locally. The clone path is referred to as `TB_WORK_DIR` throughout this guide.

<Aside type="note">
**Windows:** Before building for the first time, install the required npm globals:

```bat
npm install -g cross-env
npm install -g webpack
```
</Aside>

Build from the root folder to generate the protobuf files required for IDE compilation:

```bash
cd $TB_WORK_DIR
mvn clean install -DskipTests
```

Import the project into your IDE as a Maven project. See instructions for
<a href="https://www.jetbrains.com/help/idea/importing-project-from-maven-model.html" target="_blank" rel="noopener noreferrer">IntelliJ IDEA</a>
and <a href="https://www.baeldung.com/maven-import-eclipse/" target="_blank" rel="noopener noreferrer">Eclipse</a>.

<Aside type="tip">
**Eclipse:** Disable Maven Project Builder on the `ui-ngx` project to prevent it from scanning `node_modules`.
Right-click `ui-ngx` → **Properties > Builders**, uncheck **Maven Project Builder**, and click **OK**.
</Aside>

## Database

ThingsBoard uses PostgreSQL by default for both entities and timeseries data. Optionally, you can use Cassandra for
timeseries data instead.

### PostgreSQL

Install PostgreSQL using the <a href="https://wiki.postgresql.org/wiki/Detailed_installation_guides" target="_blank" rel="noopener noreferrer">official installation guide</a>.

Connect to PostgreSQL and create the ThingsBoard database:

```sql
psql -U postgres -d postgres -h 127.0.0.1 -W
CREATE DATABASE thingsboard;
\q
```

### Cassandra (optional)

Install <a href="https://cassandra.apache.org/_/download.html" target="_blank" rel="noopener noreferrer">Apache Cassandra</a>,
then set the timeseries database type in `application/src/main/resources/thingsboard.yml`:

```yaml
database:
  ts:
    type: "${DATABASE_TS_TYPE:cassandra}"
```

If Cassandra runs on a remote host or a non-default port, update the connection settings in the same file.
See the <DocLink product={props.product} path="reference/configuration/core-rule-engine-config/" target="_blank">configuration guide</DocLink> for the full property reference.

Rebuild the application module after updating the config:

```bash
cd $TB_WORK_DIR/application
mvn clean install -DskipTests
```

### Initialize the schema

Run the install script to create database tables and populate demo data. The script is generated during the Maven build.

On Linux:

```bash
cd $TB_WORK_DIR/application/target/bin/install
chmod +x install_dev_db.sh
./install_dev_db.sh
```

On Windows:

```bat
cd %TB_WORK_DIR%\application\target\windows
install_dev_db.bat
```

## Run the dev environment

### UI (hot reload)

Skip this step if you are not modifying the UI.

Start the UI dev server on port 4200. REST API and WebSocket requests are forwarded to port 8080:

```bash
cd $TB_WORK_DIR/ui-ngx
mvn clean install -P yarn-start
```

### Server

Start the server using one of these two options.

**IDE:** Run the main method of `org.thingsboard.server.ThingsboardServerApplication` in the `application` module.

**Command line:**

<Code code={jarCmd} lang="bash" />

### Verify the setup

Open `http://localhost:4200` (with UI hot reload) or `http://localhost:8080` and log in with the demo credentials:

| Field | Value |
|-------|-------|
| Login | tenant@thingsboard.org |
| Password | tenant |

## Submit changes

### Verify the build

Before committing, run the full build with tests:

```bash
mvn clean install
```

Also run the <a href="https://github.com/thingsboard/thingsboard/tree/master/msa/black-box-tests" target="_blank" rel="noopener noreferrer">black-box tests</a>.

### Push and create a pull request

Commit and push your changes to your fork:

```bash
git commit -m 'Descriptive commit message'
git push origin master
```

Create a pull request against the **master** branch. If conflicts exist, resolve them first.

The GitHub CLA bot prompts you to sign the Contributor License Agreement (CLA) when you open the PR.
