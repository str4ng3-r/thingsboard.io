Simple calculated fields compute a real-time value using a single **mathematical expression** —
no scripting required. For example, given `temperature` and `humidity` telemetry, the expression
`243.04 * (ln(humidity/100) + 17.625 * temperature / (243.04 + temperature)) / (17.625 - (ln(humidity/100) + 17.625 * temperature / (243.04 + temperature)))`
computes dew point and stores it as `dewPoint` the moment new data arrives.

Use Simple when you need:

- Derived sensor metrics (dew point, heat index, power = voltage × current)
- Normalization and scaling (calibration factors, unit conversions)
- Threshold-based values (clamping, simple scoring, deltas)
- Cross-entity calculations (combine device telemetry with asset attributes)

## Configuration

<Steps>
1. Open the **Calculated fields** tab on the device, asset, or profile.
2. Click **+** → **Create new calculated field**.
3. In **General**, enter a title and select **Simple** as the type.
4. Add [Arguments](#arguments), write the [Expression](#expression), and configure [Output](#output).
5. Click **Add** to save.
</Steps>

### Arguments

Simple fields support **Latest telemetry** and **Attribute** argument types.
The data source can be the current entity, another Device/Asset, the Customer, the Current tenant,
or the Current owner.

For full argument configuration details, see
<DocLink product={props.product} path="user-guide/calculated-fields">Calculated Fields — Arguments</DocLink>.

### Expression

In the **Expression** field, write a single mathematical formula that references your argument
variables by name (e.g., `temperature`, `humidity`).

Supported operations and functions:

| Category | Supported |
|----------|-----------|
| Arithmetic | `+`, `-`, `*`, `/` |
| Functions | `abs`, `sqrt`, `pow`, `min`, `max`, `ln`, `log`, `ceil`, `floor`, `round` |

The expression produces **one result value** saved under the configured output key.

### Output

| Field | Description |
|-------|-------------|
| **Output type** | Time series or Attribute |
| **Key** | The name under which the computed value is saved |
| **Decimals** | Optional decimal rounding |
| **Use latest timestamp** | (Time series only) Store the result using the most recent argument timestamp |

For output strategy options, see
<DocLink product={props.product} path="user-guide/calculated-fields">Calculated Fields — Output Strategy</DocLink>.

## Example: Dew Point Calculation

A Smart Device sends `temperature` (°C) and `humidity` (%) as telemetry in real time.
Goal: compute `dewPoint` and store it as a time series key on the same device.

### Configuration steps

<Steps>
1. Open the **Calculated fields** tab on the Smart Device (or its device profile).
2. Click **+** → **Create new calculated field**. Enter title **Dew Point** and select type **Simple**.
3. Click **Add argument**:
   - Entity type: **Current entity**
   - Argument type: **Latest telemetry**
   - Time series key: `humidity`
   - Argument name: `humidity`
   Add a second argument for `temperature` with the same settings.
4. In **Expression**, enter:
   ```
   243.04 * (ln(humidity / 100) + 17.625 * temperature / (243.04 + temperature)) /
   (17.625 - (ln(humidity / 100) + 17.625 * temperature / (243.04 + temperature)))
   ```
5. In **Output**:
   - Output type: **Time series**
   - Key: `dewPoint`
6. Click **Add**.
</Steps>

**Result:** Each time `temperature` or `humidity` is updated, `dewPoint` is recalculated and stored
as a new telemetry entry on the Smart Device.
