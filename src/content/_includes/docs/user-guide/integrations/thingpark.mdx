The <a href="https://www.actility.com/" target="_blank" rel="noopener noreferrer">Actility ThingPark</a> integration connects ThingsBoard to the ThingPark LoRaWAN network server. ThingPark pushes uplink messages from LoRaWAN devices to a ThingsBoard HTTP endpoint, where an uplink converter decodes them into device telemetry and attributes for real-time monitoring and visualization on <DocLink product={props.product} path="user-guide/dashboards" bold={false} target="_blank">dashboards</DocLink>. In the reverse direction, ThingsBoard encodes Rule Engine messages via a downlink converter and delivers them to devices through Actility.

## Register a ThingPark account

<Steps>
1. Go to the <a href="https://www.actility.com/thingpark/" target="_blank" rel="noopener noreferrer">ThingPark website</a> and sign up for an account, or log in to your existing <a href="https://thingparkenterprise.eu.actility.com/tpe/#/login" target="_blank" rel="noopener noreferrer">ThingPark Wireless OSS</a> instance.
2. Review the <a href="https://www.actility.com/thingpark-documentation-portal/" target="_blank" rel="noopener noreferrer">ThingPark documentation</a> to familiarize yourself with the platform.
</Steps>

## Register your device in ThingPark

<Steps>
1. Log in to your ThingPark Wireless OSS instance and create an application (e.g. *My Sensors*).
2. Add a new device to the application. Enter the **DevEUI** and **AppEUI** from the device label or documentation. Select the appropriate device profile.
3. The **DevEUI** becomes the device name in ThingsBoard automatically after the first uplink.
</Steps>

## Verify device connectivity

<Steps>
1. Open the <a href="https://thingparkenterprise.eu.actility.com/thingpark/wlogger/gui/" target="_blank" rel="noopener noreferrer">ThingPark Wireless Logger</a> and confirm your device is transmitting uplinks.
2. Note the **DevEUI** — you will see this value as the device name in ThingsBoard after integration.
</Steps>

## ThingsBoard integration setup

### Create an uplink converter

The uplink converter decodes the incoming ThingPark message and maps it to the ThingsBoard data model. For the full decoder function reference, see <DocLink product={props.product} path="user-guide/integrations/uplink-data-converter" bold={false} target="_blank">Uplink data converter</DocLink>.

From ThingsBoard 4.0, you can map message fields to attributes or telemetry directly in the UI — no scripting needed for structured JSON payloads. For binary payloads, add a decoder script in the same wizard.

If your device is in the built-in catalog, use the **Library** tab — see <DocLink product={props.product} path="user-guide/integrations/uplink-data-converter/#converters-library" bold={false} target="_blank">Library</DocLink> for a built-in catalog of ready-to-use decoder functions for over 100 devices.

<Aside type="tip" title="TBEL">
  Use <DocLink product={props.product} path="user-guide/tbel" bold={false} target="_blank">TBEL</DocLink> for converter scripts — it executes more efficiently than JavaScript inside ThingsBoard.
</Aside>

Adjust if needed; see <DocLink product={props.product} path="user-guide/integrations/uplink-data-converter/#advanced-decoding-parameters" bold={false} target="_blank">Advanced decoding parameters</DocLink>.

  // Initialize the output object with empty attributes and telemetry for clarity.
  var result = { attributes: {}, telemetry: {}};

  // Extract the timestamp from metadata (represented in milliseconds).
  var timestamp = metadata.ts; // ts is the timestamp parsed from the incoming message's time, or returns the current time if it cannot be parsed.

  if (metadata.payloadFormat == 'JSON') {
  // payloadFormat is 'JSON' when ThingPark sends a pre-decoded JSON payload
  // (DevEUI_uplink contains a "payload" object).
  // All fields from the decoded payload object are put directly into telemetry.
  var decoded = decodeToJson(input);
  result.telemetry = {
  ts: timestamp,
  values: decoded
};
  return result;
}

  // payloadFormat is 'BINARY' when raw hex payload is received (payload_hex field).

### Create the integration

<Steps>
  <ol>
    <li>Go to <strong>Integrations center &#8702; Integrations</strong> and click <strong>+ Add integration</strong>.</li>
    <li>
      <strong>Basic settings:</strong>
      <ul>
        <li>Set <strong>Integration type</strong> to <strong>ThingPark</strong>.</li>
        <li>Enter a name, e.g. <code>ThingPark integration</code>.</li>
        <li>Leave <strong>Enable integration</strong> and <strong>Allow create devices or assets</strong> on so the device is created automatically on first data.</li>
        <li>Click <strong>Next</strong>.</li>
      </ul>
    </li>
    <li>
      <strong>Uplink data converter:</strong>
      <ul>
        <li>Click <strong>Select existing</strong> and choose the <a href="#create-an-uplink-converter">previously created</a> <strong>ThingPark Uplink Converter</strong>.</li>
        <li>Click <strong>Next</strong>.</li>
      </ul>
    </li>
    <li>
      <strong>Downlink data converter:</strong>
      <ul>
        <li>Click <strong>Skip</strong> — only needed for sending commands to devices; can be added later.</li>
      </ul>
    </li>
    <li>
      <strong>Connection:</strong>
      <ul>
        <li>Copy the <strong>HTTP endpoint URL</strong> — paste this into your ThingPark application as the destination routing address so Actility forwards uplinks to ThingsBoard.</li>
        <li>Click <strong>Add</strong>.</li>
      </ul>
    </li>
  </ol>
</Steps>

#### Connection settings

**Base URL** — the base address of the ThingsBoard server used to construct the HTTP endpoint URL. For ThingsBoard Cloud, this is `https://thingsboard.cloud`.

**HTTP endpoint URL** — the auto-generated webhook address for this integration. Paste this URL into your ThingPark application server routing profile as the destination so Actility forwards device uplink messages to ThingsBoard.

**Enable security (Headers filter)** — restrict incoming requests to those carrying a specific HTTP header and value. Click **Add** to define one or more **Header** / **Value** pairs. Requests missing a listed header or carrying an incorrect value are rejected.

**Execute remotely** — route integration processing to a <DocLink product={props.product} path="user-guide/integrations/remote" bold={false} target="_blank">remote integration</DocLink> instance running inside your private network, rather than on the ThingsBoard server.

## Visualize data with dashboards

Once devices send uplinks through the ThingPark integration, their telemetry appears under **Entities &#8702; Devices** in ThingsBoard. Use <DocLink product={props.product} path="user-guide/dashboards" bold={false} target="_blank">Dashboards</DocLink> to build real-time visualizations: add widgets for temperature, battery level, GPS location, and any other fields decoded by your uplink converter. Dashboards support time-series charts, maps, gauges, and tables, and can be shared with customers or embedded in external applications.
