A widget is a dashboard element that displays data or provides functionality. Widgets can visualize device data (telemetry and attributes), control devices via RPC, manage alarms, or display static HTML content.

## Widget types

| Type | Description | Example |
|------|-------------|---------|
| **Time Series** | Displays data over a time window (real-time or historical) | Line chart, Bar chart |
| **Latest Values** | Shows the most recent attribute or telemetry value | Gauge, Value card |
| **Control** | Sends RPC commands to devices | Knob, Switch |
| **Alarm** | Displays and manages alarms | Alarms table |
| **Static** | Shows static HTML content (no data source) | Floor plan, HTML card |

See <DocLink product={props.product} path="user-guide/widget-library">Widgets Library</DocLink> for the complete list of widget bundles.

## Adding a widget to the dashboard

<Steps>
1. Open a dashboard and enter **Edit mode**.
2. Click **+ Add widget**.
3. Select a widget bundle and choose the specific widget.
4. Configure the data source (entity alias and keys).
5. Click **Add** and apply changes.
</Steps>

#### Data keys

A data key identifies the telemetry, attribute, or entity field a widget should display.

**Key sources:**
- **Attributes** — client, server, and shared attributes combined.
- **Time series** — telemetry keys reported by devices or Rule Engine.
- **Entity fields** — metadata fields (created time, entity type, name, label).
