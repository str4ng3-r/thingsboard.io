The **Login** tab on the White labeling page lets you customize the login page that your clients see when accessing your ThingsBoard instance via a custom domain.

## Configure the login page

<Steps>

1. Navigate to the **White labeling** page and open the **Login** tab.

2. Enter your registered domain name, or follow the <DocLink product={props.product} path="user-guide/security/domains">domain registration</DocLink> guide to set one up.

3. Enable the **Prohibit to use hostnames from header** option (recommended for security).

4. Set a custom application title, replace the default favicon and logo with your own, define the primary and accent color palettes, and set the page background color.

5. Click **Save**.

</Steps>

## Self-registration

ThingsBoard supports customer self-registration — a sign-up page where your customers can create their own accounts and log in with predefined roles, dashboards, and permissions. The login page appearance configured above (logo, colors, background) is applied to the sign-up form as well.

<Aside type="tip">
Self-registration is configured under **Security** > **Self registration**. See the <DocLink product={props.product} path="user-guide/security/self-registration">Self-Registration</DocLink> guide for the full setup — domain, CAPTCHA, user roles, default dashboard, privacy policy, and terms of use.
</Aside>
