Dashboard layouts organize how widgets appear on the dashboard grid. Each <DocLink product={props.product} path="user-guide/dashboards">dashboard</DocLink> state can have its own layout type, settings, and breakpoints — enabling tailored visual presentations for different IoT use cases and screen resolutions.

<Steps>
1. Enter dashboard edit mode by clicking the **Edit mode** button in the upper right corner.
2. Click the **Layouts** button in the upper left corner to open the layout management window.
</Steps>

## Layout types

The platform supports three layout types: Default, SCADA, and Divider. Each dashboard state has exactly one layout type configured.

### Default layout

The Default layout is a general-purpose layout that fits most dashboards. It supports screen breakpoints for responsive design.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/user-guide/layouts/layout-default-1.png', alt: 'Default layout', caption: 'The Default layout is a general-purpose layout that supports screen breakpoints for responsive design.' },
  { src: '/src/assets/images/user-guide/layouts/layout-default-2.png', alt: 'Default layout example' },
]} />

### SCADA

The SCADA layout is designed for creating SCADA HMIs (Human-Machine Interfaces). SCADA HMIs typically consist of scalable widgets called <DocLink product={props.product} path="user-guide/scada">SCADA symbols</DocLink>. You can also combine SCADA symbols with other ThingsBoard widgets on the same layout. This layout type also supports screen breakpoints.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/user-guide/layouts/layout-scada-1.png', alt: 'SCADA layout', caption: 'The SCADA layout is designed for creating SCADA HMIs consisting of scalable SCADA symbols.' },
  { src: '/src/assets/images/user-guide/layouts/layout-scada-2.png', alt: 'SCADA layout example' },
]} />

### Divider

The legacy Divider layout splits the dashboard into two sub-layouts: left and right. You can configure the width of these sub-layouts using either a fixed pixel value or a percentage of the screen width. Most layout parameters available for the Default layout are also supported by sub-layouts. However, sub-layouts do not support breakpoints.

Set up both layouts independently — add a background image to one and a new background color to the other, then resize windows in a percentage ratio to each other.

## Breakpoints

Breakpoints allow you to define different layout settings and widgets based on the screen width of the end user.

Each dashboard state has a **Default** breakpoint — used when no other breakpoints match the current screen resolution. You can define the following breakpoints:

| Breakpoint | Min width | Max width |
|-----------|-----------|-----------|
| **Desktop (xl)** | 1920 px | 5000 px |
| **Desktop (lg)** | 1280 px | 1919 px |
| **Laptop (md)** | 960 px | 1279 px |
| **Tablet (sm)** | 600 px | 959 px |
| **Mobile (xs)** | — | 599 px |

### Adding a breakpoint

<Steps>
1. Open the **Manage layouts** window and click the **+** button in the breakpoints table.
2. Select the breakpoint from the dropdown list.
3. Select the screen resolution and optionally copy widgets and layout settings from another breakpoint. Click **Add**.
</Steps>

### Widget references vs widget copies

#### Widget references

When a new breakpoint layout is created, all widgets from the source layout are copied **by reference**. Changes to the widget configuration are synchronized between breakpoints, but position and size are independent per layout.

<Steps>
1. Right-click on the widget and select **Copy reference** (Ctrl+R).
2. In another breakpoint layout, right-click on empty space and select **Paste reference** (Ctrl+I).
</Steps>

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/user-guide/layouts/copy-widget-by-reference-1.png', alt: 'Copy widget reference', caption: 'Right-click on the widget and select "Copy reference" (Ctrl+R).' },
  { src: '/src/assets/images/user-guide/layouts/copy-widget-by-reference-2.png', alt: 'Paste widget reference', caption: 'Right-click on empty space and select "Paste reference" (Ctrl+I).' },
  { src: '/src/assets/images/user-guide/layouts/copy-widget-by-reference-3.png', alt: 'Widget pasted as reference', caption: 'The widget is pasted as a reference.' },
]} />

In edit mode, identify a referenced widget by the **Reference** button in its edit panel.

<MultiProductImageGallery product={props.product} images={[
  { src: '/src/assets/images/user-guide/layouts/identify-referenced-widget-1.png', alt: 'Identify referenced widget', caption: 'A referenced widget has a "Reference" button in its edit panel.' },

To replace a reference with an independent copy, click the **Reference** button or right-click the widget and select **Replace reference with widget copy**.

#### Widget copies

A widget copy is a complete, independent copy. Changes to the copied widget do not affect the original.

<Steps>
1. Right-click on the widget and select **Copy** (Ctrl+C).
2. In another breakpoint layout, right-click on empty space and select **Paste** (Ctrl+V).
</Steps>
