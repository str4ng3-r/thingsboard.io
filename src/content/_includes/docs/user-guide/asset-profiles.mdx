**Asset profiles** let you centrally define common settings for a group of similar assets — rule chain, queue, alarm rules, and calculated fields. Instead of configuring each asset individually, you define the profile once and assign it to as many assets as needed.

## Managing Profiles

### Create a Profile

<Steps>
1. Navigate to **Profiles &#8702; Asset profiles**.
2. Click **+ Add asset profile** and select **Create new asset profile**.
3. Enter a **Name** (the only required field).
4. Optionally configure the rule chain, queue, alarm rules, and calculated fields.
5. Click **Add**.
</Steps>

<Aside type="tip">
  **Example:** a `Building` profile with a dedicated rule chain for occupancy events and a Critical alarm rule for "power consumption exceeds capacity," assigned to every building asset in the tenant — one configuration, applied to all of them at once.
</Aside>

### Edit a Profile

<Steps>
1. Click the profile name or the **pencil** icon.
2. Update the desired settings across any tab.
3. Click **Apply changes**.
</Steps>

### Delete a Profile

Click the **trash** icon next to the profile and confirm deletion.

## Profile Details

Clicking a profile opens a details panel with the following tabs:

| Tab | Contents |
|-----|---------|
| **Details** | Name, default rule chain, mobile dashboard, queue, default edge rule chain, profile image, and description |
| <DocLink product={props.product} path='user-guide/calculated-fields' target="_blank">Calculated Fields</DocLink> | Formulas for computing derived metrics from asset telemetry or attributes |
| <DocLink product={props.product} path='user-guide/alarm-rules' target="_blank">Alarm Rules</DocLink> | Conditions, severity levels, and actions applied to all assets in this profile |
| <DocLink product={props.product} path='user-guide/security/audit-log' target="_blank">Audit Logs</DocLink> | History of user actions performed on this profile |
| <DocLink product={props.product} path='user-guide/version-control' target="_blank">Version Control</DocLink> | Git-based export and restore for backup and collaboration |

A few Details fields are easy to misread:
- **Mobile dashboard** — the dashboard the mobile app uses as the device details screen for assets on this profile. Unrelated to the per-user login dashboard covered in <DocLink product={props.product} path='user-guide/dashboards' target="_blank">Dashboards</DocLink>.
- **Default edge rule chain** — only relevant if the tenant also runs ThingsBoard Edge; it's the rule chain Edge instances use to process incoming data for assets on this profile.

## Key Settings

### Default Rule Chain

By default, the Root Rule Chain processes all incoming messages and events for every asset. As your system grows and you introduce different asset types, the Root Rule Chain can become complex.

Assign a dedicated <DocLink product={props.product} path='user-guide/rule-engine/#rule-chain' target="_blank">rule chain</DocLink> to an asset profile to isolate processing logic — attribute updates and lifecycle events (Created/Updated/Deleted) — for that asset type without overloading the Root Rule Chain.

### Default Queue

By default, all messages use the **main** queue. Assign a custom queue to isolate high-priority assets from high-volume ones, or to apply different submit and processing strategies. See <DocLink product={props.product} path='user-guide/rule-engine/queues' target="_blank">Queues</DocLink> for available delivery guarantees and configuration.

<Aside type="note">
  Custom queues must be configured by a System Administrator before they can be assigned to a profile.
</Aside>

### Alarm Rules

Define conditions that apply to all assets in this profile — for example, trigger a Critical alarm when a computed value exceeds a threshold. Rules defined here are shared across every asset using the profile. See <DocLink product={props.product} path='user-guide/alarm-rules' target="_blank">Alarm Rules</DocLink>.

### Calculated Fields

Define formulas that compute derived metrics from telemetry or attributes — for example, aggregating temperature readings from every related device into a single building-level average. Like Alarm Rules, calculated fields defined here are inherited by every asset using the profile; an asset-level calculated field runs alongside any profile-level one, not instead of it. See <DocLink product={props.product} path='user-guide/calculated-fields' target="_blank">Calculated Fields</DocLink>.

### Export and Import Asset Profile

**Export asset profile** downloads the profile's configuration as a JSON file — useful for backup or for migrating a profile between ThingsBoard instances. **Import asset profile** (from the **+ Add asset profile** menu) creates a new profile from a previously exported file.

### Make Asset Profile Default

Sets this profile as the tenant's default — the one pre-selected in the **Asset profile** field when creating a new asset without explicitly choosing one. Only one profile can be the default at a time.

## Profile Changes and Existing Assets

Editing a profile's **Rule Chain**, **Queue**, **Alarm Rules**, or **Calculated Fields** immediately affects every asset already assigned to that profile — there's no need to touch individual assets. This is the main advantage of configuring these settings at the profile level instead of per asset.

Moving an asset to a *different* profile (in <DocLink product={props.product} path='user-guide/assets/#edit-an-asset' target="_blank">Edit an Asset</DocLink>) applies that profile's rule chain, queue, alarm rules, and calculated fields going forward — it does not retroactively change alarms or calculated field results already generated under the previous profile.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Alarm rule or calculated field doesn't seem to run | It was added on the asset instead of the profile (or vice versa) | Confirm which level the rule was created at — profile-level rules apply to every asset on that profile, asset-level rules apply only to that one asset |
| Custom queue isn't selectable when editing a profile | The queue hasn't been created yet | Ask a System Administrator to configure the queue first — see <DocLink product={props.product} path='user-guide/rule-engine/queues' target="_blank">Queues</DocLink> |
| New asset doesn't get the profile you expected | No profile was selected at creation, so the tenant's default profile was applied | Check which profile is marked default — see **[Make Asset Profile Default](#make-asset-profile-default)** — or select the profile explicitly when creating the asset |
| Imported asset profile fails or looks incomplete | The JSON file was edited by hand or exported from an incompatible ThingsBoard version | Re-export the original profile and import that file unmodified |
