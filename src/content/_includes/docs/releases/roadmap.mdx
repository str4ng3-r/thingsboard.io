## ThingsBoard AI Ecosystem

ThingsBoard AI features help you move from a real-world problem to a running IoT system, using natural language.

### Available now

- <DocLink product={props.product} path='user-guide/ai-assistant'>AI Assistant</DocLink>: Creates and modifies calculated fields, alarm rules, dashboards, and notifications from a plain-language description, with no manual setup.
- **Device Dashboard Assistant:** Generates a complete dashboard for a single device with one click.
- <DocLink product={props.product} path='user-guide/ai-solution-creator'>AI Solution Creator</DocLink>: Turns a described scenario into a complete solution, with entities, relations, calculated fields, alarm rules, dashboards, customers, users, and roles.
- <DocLink product={props.product} path='user-guide/cli-solutions'>ThingsBoard CLI</DocLink>: Builds and customizes your system from the terminal, with help from an AI agent like Claude.

### Coming soon

- **Extensions via ThingsBoard CLI:** Extend the platform with your own logic.
- **Device connectivity and provisioning via AI Assistant:** Connect and provision devices through natural language.
- **RBAC via AI Assistant:** Configure roles and permissions with the help of the assistant.

## Major improvements

### Available now

- <DocLink product={Products.CE} path='iot-hub/user-guides'>ThingsBoard IoT Hub</DocLink>: A marketplace where the community and ThingsBoard team share free, ready-to-use calculated fields, alarm rules, devices, solution templates, rule chains, and widgets.
- **Device Emulators:** Simulate overload events, meter failures, and edge cases to stress-test your system before it goes live.

### Coming soon

- **New sidebar and navigation:** Navigate the platform with a redesigned sidebar.
- **Citus database support:** Scale the database layer horizontally with Citus.
- **New onboarding experience:** Guided setup that helps you start with the platform, create your first entities, and connect devices.
