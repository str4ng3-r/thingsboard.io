Route and direct messages based on various criteria without modifying them. Filter nodes typically
produce `True`/`False` outputs for boolean evaluations, or named connections for switch-style routing.

| Node | Description |
|------|-------------|
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/alarm-status-filter" bold={false}>Alarm status filter</DocLink> | Routes alarm messages based on their status (`ACTIVE_UNACK`, `ACTIVE_ACK`, `CLEARED_UNACK`, `CLEARED_ACK`, `CLEARED`) |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/asset-profile-switch" bold={false}>Asset profile switch</DocLink> | Routes to a connection named after the originator asset's profile |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/check-fields-presence" bold={false}>Check fields presence</DocLink> | Routes based on whether specified fields exist in message data or metadata |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/check-relation-presence" bold={false}>Check relation presence</DocLink> | Routes based on whether a relation exists between the originator and a target entity |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/device-profile-switch" bold={false}>Device profile switch</DocLink> | Routes to a connection named after the originator device's profile |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/entity-type-filter" bold={false}>Entity type filter</DocLink> | Passes or blocks messages based on the originator entity type |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/entity-type-switch" bold={false}>Entity type switch</DocLink> | Routes to a connection named after the originator entity type |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/gps-geofencing-filter" bold={false}>GPS geofencing filter</DocLink> | Routes based on whether GPS coordinates fall inside a configured geofence |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/message-type-filter" bold={false}>Message type filter</DocLink> | Passes or blocks messages based on the message type |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/message-type-switch" bold={false}>Message type switch</DocLink> | Routes to a connection named after the message type |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/script" bold={false}>Script</DocLink> | Routes using a boolean TBEL/JS function — `True` or `False` output |
| <DocLink product={props.product} path="reference/rule-engine/nodes/filter/switch" bold={false}>Switch</DocLink> | Routes to one or more connections returned by a TBEL/JS function |
