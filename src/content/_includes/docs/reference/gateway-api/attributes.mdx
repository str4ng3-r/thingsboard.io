## Publish Client-Side Attributes

Upload client-side attributes for one or more devices.

**Topic:** `v1/gateway/attributes`

```json
{
  "Device A": {"fw_version": "1.0", "battery": 87},
  "Device B": {"fw_version": "2.1", "battery": 92}
}
```

Replace `$ACCESS_TOKEN` in the commands below with the gateway device's access token.

<HostCode product={props.product} lang="bash" code={`mosquitto_pub -h "{MQTT_HOST}" -t "v1/gateway/attributes" -u "$ACCESS_TOKEN" -m '{"Device A":{"fw_version":"1.0","battery":87}}'`} />

## Request Attribute Values

Subscribe to the response topic first, then publish the request.

**Subscribe:** `v1/gateway/attributes/response`

**Publish:** `v1/gateway/attributes/request`

| Field | Required | Description |
|-------|----------|-------------|
| `id` | Yes | Integer request identifier — echoed in the response |
| `device` | Yes | Device name |
| `keys` | Yes | Array of attribute keys to return |
| `client` | Yes | Boolean flag: set to `true` to request client-side attributes, or `false` to request shared attributes |

<Tabs>
<TabItem label="Client">

```json
{
  "id": 1,
  "device": "Device A",
  "keys": ["fw_version", "battery"],
  "client": true
}
```

</TabItem>
<TabItem label="Shared">

```json
{
  "id": 1,
  "device": "Device A",
  "keys": ["state", "level"],
  "client": false
}
```

</TabItem>
</Tabs>

Replace `$ACCESS_TOKEN` in the commands below with the gateway device's access token.

<HostCode product={props.product} lang="bash" code={`mosquitto_sub -h "{MQTT_HOST}" -t "v1/gateway/attributes/response" -u "$ACCESS_TOKEN"`} />

Then, in a new terminal window, publish the request.

<Tabs>
<TabItem label="Requesting client attributes">

<HostCode product={props.product} lang="bash" code={`mosquitto_pub -h "{MQTT_HOST}" -t "v1/gateway/attributes/request" -u "$ACCESS_TOKEN" -m '{"id": 1, "device": "Device A", "keys": ["fw_version", "battery"], "client": true}'`} />

</TabItem>
<TabItem label="Requesting shared attributes">

<HostCode product={props.product} lang="bash" code={`mosquitto_pub -h "{MQTT_HOST}" -t "v1/gateway/attributes/request" -u "$ACCESS_TOKEN" -m '{"id": 1, "device": "Device A", "keys": ["state", "level"], "client": false}'`} />

</TabItem>
</Tabs>

## Subscribe to Shared Attribute Updates

Subscribe to receive push notifications when shared attributes change for any downstream device.

**Topic:** `v1/gateway/attributes` (subscribe)

<HostCode product={props.product} lang="bash" code={`mosquitto_sub -h "{MQTT_HOST}" -t "v1/gateway/attributes" -u "$ACCESS_TOKEN"`} />

ThingsBoard delivers updates in this format:

```json
{
  "device": "Device A",
  "data": {"targetFwVersion": "2.0", "enabled": true}
}
```
