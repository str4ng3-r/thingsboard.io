Once a Trendz view is <DocLink product={props.product} path="embed-visuals">embedded on a ThingsBoard dashboard</DocLink>, the widget can trigger actions when a user interacts with it — letting you connect Trendz views to the rest of the dashboard.

This page covers the following actions:
- **Date selected** — clicking a date on a Calendar view updates the dashboard time window.
- **On row click (Table)** — clicking a table row can select an entity, navigate to a dashboard state, or run any custom script.
- **On row click (Anomaly table)** — clicking a row selects an entity and updates the time window simultaneously.
- **On row click — Pass related entities into alias** — clicking a row populates a dashboard alias with all related entities of that row.

## Date selected action

The **Calendar** and **Anomaly table** views support a **date-selected** action. When a user clicks a date or a date range on a Calendar view, the action fires and you can use it to update the ThingsBoard dashboard time window — so that all other widgets on the dashboard instantly zoom into the selected period.

<ImageGallery images={[
  { src: '/src/assets/images/trendz/widget-actions/date-selected/date-selected-1.png', alt: 'Heat Consumption Report dashboard showing a Calendar heatmap and a line chart below it', caption: 'Goal: clicking a date on the Calendar view updates the time window and all other widgets refresh to show data for that day.' },
]} />

<Aside type="tip">
  For other Trendz views on the same dashboard — such as a line chart — enable <DocLink product={props.product} path="widget-settings#use-the-dashboard-time-window">Use dashboard time window</DocLink> so they automatically reload when the time window changes.
</Aside>

### Configure the action

<Steps>
1. Open the dashboard in edit mode and click **Edit widget** on the Trendz Calendar view.

2. Switch to the **Actions** tab.

3. Click **Add action**.

4. In the **Add action** dialog:
   - Set **Action source** to `date-selected`.
   - Enter a **Name** (e.g. `Select Date`).
   - Set **Action** to **Custom action**.
   - Paste the following code into the custom action function:

   ```js
   widgetContext.dashboard.onUpdateTimewindow(
     additionalParams.startTs,
     additionalParams.endTs
   );
   ```

5. Click **Add**.
</Steps>

## On row click — Table view

The **On row click** action is available for the **Table** and **Anomaly table** view types. When a user clicks a row, the action fires and you can use it to navigate to a dashboard state, select an entity, or run any custom script.

**Example:** A dashboard shows buildings with aggregated energy consumption. Clicking a building row filters all other widgets — the line chart and pie chart update instantly to show data only for that building.

<ImageGallery images={[
  { src: '/src/assets/images/trendz/widget-actions/row-click/row-click-1.png', alt: 'Energy Consumption By Floor dashboard — table with buildings, line chart, and pie chart', caption: 'Goal: clicking a building row in the table updates all other widgets to show data for that building.' },
]} />

<Aside type="tip">
  In this example, clicking a row will update a filter alias that drives the other Trendz views on the dashboard. To set up this alias connection first, see <DocLink product={props.product} path="widget-filter-by-alias">Filter Trendz Widget by Dashboard Alias</DocLink>.
</Aside>

### Step 1 — Set the Row click entity in Trendz

Each row in a Trendz table can aggregate data from multiple entity types. You need to tell Trendz which entity type represents the selected row — this entity will be passed to the ThingsBoard action.

<Steps>
1. Open the view in Trendz and click **View Settings** (the gear icon in the top-right corner).

2. Expand the **View Mode fields** section.

3. In the **Row click entity** field, enter the entity type you want to select on row click (e.g. **EM building**).

4. Click **Save changes**.
</Steps>

### Step 2 — Configure the On row click action

<Steps>
1. Open the dashboard in edit mode and click **Edit widget** on the Trendz Table widget.

2. Switch to the **Actions** tab.

3. Click **Add action**.

4. In the **Add action** dialog, configure the action:
   - Set **Action source** to **On row click**.
   - Enter a **Name** (e.g. `On row click select Building`).
   - Choose any **Action** type. In this example, use **Update current dashboard state**.
   - Set **Target dashboard state** to the state you want to update (e.g. **default**).
   - Set **State entity parameter name** to the alias name used as the datasource by the other Trendz views on the dashboard (e.g. the alias that drives their filters).

5. Click **Add**.

6. Click **Apply**.
</Steps>

## On row click — Pass related entities into alias

When a user clicks a row in a Trendz table, you can push all entities related to that row into a dashboard alias. Any widget using that alias — a map, a chart, an entities table — instantly updates to show only those entities.

**Example:** A table shows buildings with aggregated energy consumption. Clicking a building row sends all its apartments into an alias. A widget on the right side of the dashboard refreshes to show only those apartments.

<ImageGallery images={[
  { src: '/src/assets/images/trendz/widget-actions/related-entities/related-entities-18.png', alt: 'Apartments List Dashboard — Feline row clicked in Trendz table, Entities table on the right showing all apartments in Feline', caption: 'Goal: clicking a building row populates an alias with all apartments in that building. Any widget can use this alias — an Entities table is used here just to verify it works.' },
]} />

<Aside type="tip">
  The Entities table in this example is only used to verify that the alias is populated correctly. In a real dashboard you can replace it with any ThingsBoard widget — for example a **Map** widget to see the locations of all related entities on the map.
</Aside>

### Step 1 — Set the Row click entity in Trendz

<Steps>
1. Open the view in Trendz and click **View Settings** (the gear icon in the top-right corner).

2. Expand the **View Mode fields** section.

3. In the **Row click entity** field, enter the entity type whose IDs you want to pass to the alias (e.g. **EM apartment**).

4. Click **Save changes**.
</Steps>

### Step 2 — Create the alias on the dashboard

<Steps>
1. Open the dashboard in edit mode and click **Entity aliases** in the toolbar.

2. Click **Add alias**.

3. In the **Add alias** dialog:
   - Enter a name (e.g. **Selected Entities** — you will reference this exact name in the action code).
   - Enable **Resolve as multiple entities**.
   - Set **Filter type** to **Entity list**.
   - Set **Type** to the entity type that will be passed on row click.
   - Optionally add default entities to show before any row is clicked.

4. Click **Add**, then **Save**.
</Steps>

### Step 3 — Configure the custom action

<Steps>
1. Open the dashboard in edit mode and click **Edit widget** on the Trendz Table widget.

2. Switch to the **Actions** tab.

3. Click **Add action**.

4. In the **Add action** dialog:
   - Set **Action source** to **On row click**.
   - Enter a **Name** (e.g. `Populate Selected Entities alias`).
   - Set **Action** to **Custom action**.
   - Paste the following code — it finds the alias by name, extracts the entity IDs from the clicked row, and updates the alias:

   ```js
   // Find the alias by name
   let entityAlias = Object.values(widgetContext.aliasController.entityAliases)
     .find(el => el.alias === 'Selected Entities');

   // Update alias with entity IDs from the clicked row
   if (entityAlias && additionalParams.entityIds?.length) {
     entityAlias.filter.entityList = additionalParams.entityIds
       .map(entity => entity.entityId.id);
     widgetContext.updateAliases([entityAlias.id]);
   }
   ```

   Replace `'Selected Entities'` with the exact alias name you created in Step 2.

5. Click **Save**.

6. Click **Apply**.
</Steps>

<Steps>

2. Select the **Tables** bundle and choose **Entities table**.

3. Switch to the **Entity alias** datasource tab and select **Selected Entities**.

4. Click **Add**, then **Save** the dashboard.
</Steps>

## On row click — Anomaly table

If you are using the **Anomaly Table** on a dashboard and want the row click to both select an entity and update the dashboard time window:

<Steps>
1. In the Trendz chart settings, set the **Row click entity** (see step above).

   <ImageGallery images={[
     {
       src: '/src/assets/images/trendz/row-click-anomaly.png',
       alt: 'Row click entity setting in Trendz anomaly view',
       caption: 'Set the Row click entity in Trendz chart settings',
     },
   ]} />

2. On the ThingsBoard dashboard, open the Anomaly widget in edit mode, switch to the **Actions** tab, and add a new action.

3. Set **Action source** to **On row click** and **Action type** to **Custom action**.

4. Use the following code to update both the selected entity and the dashboard time window:

   ```js
   let stateId = widgetContext.stateController.getStateId();
   let stateParams = widgetContext.stateController.getStateParams();
   stateParams.selectedDevice = {
     entityId: entityId,
     entityName: entityName,
   };

   widgetContext.dashboard.onUpdateTimewindow(
     additionalParams.startTs,
     additionalParams.endTs
   );
   widgetContext.stateController.updateState(stateId, stateParams);
   ```

   Replace `selectedDevice` with the alias name used in your dashboard state.
</Steps>
