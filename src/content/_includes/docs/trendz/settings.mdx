## System

### Import / Export

Use the built-in Import/Export mechanism to migrate configuration from one Trendz instance to another.

The following items can be exported and imported:

- Business Entities and their Fields
- View Configurations and View Collections
- Calculation Fields
- Prediction Models
- Anomaly Models and Anomalies
- AI Assistant Chats and Messages
- Prompts
- Tasks

**Exporting**

Click the **Export** button — the configuration file downloads automatically.

**Importing**

<Steps>
1. Click the **Import** button and select the previously exported file.
2. Configure the **Skip duplicates** option:
   - **True** — duplicate items in the import file are silently ignored.
   - **False** — duplicate items cause import errors.
3. Click **Import Configuration**. A confirmation message appears on success.
</Steps>

<Aside type="caution" title="Limitations">
- Migration between **different Trendz versions** is not supported.
- Migration between **different tenants** is not supported.
</Aside>

### Signing Key

For background operations (sending generated telemetry, continuous prediction model fitting, forecasting),
Trendz must authenticate with ThingsBoard on behalf of a user.
Storing user credentials in memory is insecure — instead, Trendz uses a **JWT signing key** stored in
configuration files on the server.

<Aside type="note">
You can store the signing key in the Trendz database automatically, but this method is **not recommended**
for security reasons.
</Aside>

**Getting the signing key from ThingsBoard**

<Steps>
1. Log in to ThingsBoard as a **System Administrator**.

   <ImageGallery images={[
     { src: '/src/assets/images/trendz/settings/signing-key-4.png', alt: 'ThingsBoard system administrator login screen' },
   ]} />

2. Go to **Security → General**.

   <ImageGallery images={[
     { src: '/src/assets/images/trendz/settings/signing-key-5.png', alt: 'Security → General page in ThingsBoard' },
   ]} />

3. Scroll to the **JWT Security Settings** section and copy the value from the **Signing key** field.

   <ImageGallery images={[
     { src: '/src/assets/images/trendz/settings/signing-key-6.png', alt: 'JWT Security Settings section with the signing key field' },
   ]} />
</Steps>

**Applying the signing key**

Once you have the key, add it to the Trendz configuration for your installation type:

<details>
<summary>Ubuntu installation</summary>

<Steps>
1. SSH into the server running Trendz.
2. Open the Trendz configuration file:
   ```bash
   sudo nano /etc/trendz/conf/trendz.conf
   ```
3. Add the signing key at the end of the file:
   ```bash
   export JWT_TOKEN_SIGNING_KEY=<signing-key>
   ```
   <ImageGallery images={[
     { src: '/src/assets/images/trendz/settings/signing-key-7.png', alt: 'Signing key added to the Trendz config file on Ubuntu' },
   ]} />
4. Save the file and restart the Trendz service.
</Steps>

</details>

<details>
<summary>Docker Compose installation</summary>

<Steps>
1. Open your `docker-compose.yml` file.
2. Add an environment variable under the Trendz service:
   ```yaml
   environment:
     - JWT_TOKEN_SIGNING_KEY=<signing-key>
   ```
   <ImageGallery images={[
     { src: '/src/assets/images/trendz/settings/signing-key-8.png', alt: 'JWT_TOKEN_SIGNING_KEY environment variable in docker-compose.yml' },
   ]} />
3. Save the file and restart the Trendz container.
</Steps>

</details>

### Trendz Widget Bundle

The Trendz Widget Bundle must be uploaded to enable sharing Trendz views in ThingsBoard dashboards.
See <DocLink product={props.product} path="embed-visuals">embedding Trendz visuals in ThingsBoard</DocLink> for usage instructions.

The **Trendz Widget Bundle** section shows one of the following statuses:

| Status | Meaning |
|--------|---------|
| Not installed | The bundle is missing. |
| Update required | A newer version is available. |
| Latest version installed | No action needed. |
| Bundle is invalid | The current bundle is corrupted or incomplete. |

If the status is anything other than **Latest version installed**, click the **Upload bundle** button to install or update it.
