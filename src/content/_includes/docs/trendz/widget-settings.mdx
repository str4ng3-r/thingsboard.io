Once a Trendz view is <DocLink product={props.product} path="embed-visuals">embedded on a ThingsBoard dashboard</DocLink>, the widget exposes several settings that control how it displays data, refreshes it, and shares filter state with other widgets on the same dashboard.

This page covers three settings available in the widget edit panel:
- **Use dashboard time window** — sync the widget's time range with the dashboard selector.
- **Auto refresh** — update the chart data automatically at a fixed interval.
- **Broadcast / Listen filter change event** — propagate a filter change from one Trendz widget to others on the same dashboard.

## Use the dashboard time window

By default, each Trendz view uses its own time range. You can override this so the widget follows the dashboard time window instead.

<Steps>
1. Open the dashboard in edit mode and click **Edit widget** on the Trendz widget.

2. Switch to the **Appearance** tab.

3. Enable **Use dashboard time window**.

4. Click **Apply**.
</Steps>

Now when you change the time range on the dashboard, the Trendz view automatically reloads with the appropriate time range.

## Auto refresh

You can configure a Trendz widget to update its data automatically at a fixed interval.

<Steps>
1. Open the dashboard in edit mode and click **Edit widget** on the Trendz widget.

2. Switch to the **Appearance** tab.

3. In the **Real-time update configuration** section, enable **Update data in real time**.

4. Set the desired **Interval for updating data in real time** in minutes.

5. Click **Apply**.
</Steps>

Now the Trendz view automatically reloads its data at the configured interval.

## Sharing filters between widgets

When you have multiple Trendz views on the same dashboard — for example a data table and several charts — you can make them all respond to a single filter change. Enabling **Broadcast filter change event** on the main view and **Listen filter change event** on dependent views means that whenever the main view's filter changes, every dependent view updates instantly to match.

<Aside type="note">
  All participating views must share the same filter fields and use the identical **Filter change event name**. A mismatch in either will prevent updates from propagating.
</Aside>

<Aside type="tip">
  To instead drive a Trendz view from a ThingsBoard entity alias — for example, updating the chart when a user selects a row in an Entities table — see <DocLink product={props.product} path="widget-filter-by-alias">Filter Trendz Widget by Dashboard Alias</DocLink>.
</Aside>

### Configure the main view

<Steps>
1. Open the dashboard in edit mode and click **Edit widget** on the main Trendz view.

2. Switch to the **Appearance** tab.

3. In the **Filters management** section, enter a name in **Filter change event name** (e.g. `energy-consumption-trendz-filter-event`). All connected views must use this exact name.

4. Enable **Broadcast filter change event**.
</Steps>

Click **Apply** — the main view will now broadcast its filter state whenever it changes.

### Configure each dependent view

<Steps>
1. Click **Edit widget** on a dependent Trendz view.

2. Switch to the **Appearance** tab.

3. Set **Filter change event name** to the same value used in the main view.

4. Enable **Listen filter change event**.
</Steps>

Click **Apply**. Repeat for every remaining dependent view.

<Aside type="tip">
  On dependent views you may want to hide the filter controls so users cannot change filters manually there. Use the <DocLink product={props.product} path="view-settings#view-mode-fields">View mode fields</DocLink> setting in the Trendz view to control which fields are visible.
</Aside>

### Save and verify

Save the dashboard. Now when you change a filter on the main view, all dependent views update instantly to reflect the same selection.

<ImageGallery images={[
  { src: '/src/assets/images/trendz/widget-settings/filter-sharing/filter-sharing-10.png', alt: 'Dashboard in view mode — state filter set to free and floor set to 1, all four Trendz views updated to show matching data', caption: 'When the filter on the main view changes, all other views update instantly.' },

<Aside type="tip">
  If all your views share the same filter fields, you can enable both **Broadcast filter change event** and **Listen filter change event** on every view at the same time. This makes filter changes propagate in any direction — changing a filter on any view updates all the others, with no single "main" view required.
</Aside>
