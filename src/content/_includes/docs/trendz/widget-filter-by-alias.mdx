This guide shows how to connect a Trendz view to a ThingsBoard dashboard alias so that selecting an entity in one widget automatically updates the Trendz chart to show data for that entity — without any page reload or manual filter change.

**What you will build:** a dashboard with an Entities table and a Trendz view side by side. When a user clicks a machine in the table, the chart re-renders instantly for the selected machine.

Throughout this guide we use a Vibration Chart with a **Machine** field in the **Filter** section as the example view.

## How it works

The **Advanced analytics View** widget accepts a ThingsBoard alias as its datasource. When the alias value changes — for example when a user clicks a row in an Entity List widget — ThingsBoard passes the selected entity name to the Trendz widget. Trendz injects that name into the view filter and the chart re-renders immediately.

```plaintext
  ┌─────────────────────────────────────┐
  │  Entity List widget                 │
  │  datasource: all_devices alias      │
  │  on row click → updates alias ──────┼──┐
  └─────────────────────────────────────┘  │ entity name
                                           ▼
  ┌─────────────────────────────────────┐
  │  Advanced analytics View widget     │
  │  datasource: selected_device alias  │
  │  Filter Name: "Machine"             │
  │                                     │
  │  → chart re-renders for the         │
  │    selected entity                  │
  └─────────────────────────────────────┘
```

The **Filter Name** on the widget must match the field name used in the Trendz view filter exactly. If the view has multiple filter fields, Trendz matches by name. When the alias is empty, the view shows data for all entities.

## Prerequisites

Before you start, make sure:

- The Trendz view has at least one field added to the **Filter** section and is saved.
- The <DocLink product={props.product} path="trendz-bundle">Advanced analytics Bundle</DocLink> is imported into ThingsBoard.

## Option 1 — Using the Share Button

The quickest way. Trendz automatically creates the required aliases and wires them to the widget — no manual alias configuration needed.

Click **Share** in the Trendz toolbar, then select **Add on Dashboard** or **Add on New Dashboard**. In the Share dialog, enable **Create alias from filter** and select the **Machine** filter as the alias source, then click **Add**. For a full walkthrough of the Share button flow, see <DocLink product={props.product} path="embed-visuals#create-alias-from-filter">Add view on Dashboard — Create alias from filter</DocLink>.

## Option 2 — Manual Configuration

Use this approach when you need to connect the Trendz view to an existing alias, use a different entity-selection widget, or want full control over the dashboard setup.

<Aside type="note">
  In this guide we configure everything for Device Type **Machine**, but this also works for any asset type.
</Aside>

### Configure ThingsBoard aliases

<Steps>
1. Open the dashboard in edit mode and click **Entity aliases** in the toolbar.

2. Click **Add alias** to create the first alias.

3. Name it **All Entities**, set **Filter type** to **Device type**, and select **Machine** as the device type. Click **Add**.

4. Click **Add alias** again. Name it **Selected Entity**, set **Filter type** to **Entity from dashboard state**, and enter **Selected Entity** as the state entity parameter name. Click **Add**.

5. Click **Save** to apply both aliases.
</Steps>

### Add the Entities table widget

<Steps>
1. Click **Add widget (+)** in the dashboard toolbar.

2. Select the **Tables** bundle.

3. Select **Entities table**.

4. In the widget configuration, switch to the **Entity alias** datasource tab and select **All Entities**.

5. Scroll down to **Actions** and click **Add action**.

6. Click **+** to add a new action.

7. Configure the action: set **Action source** to **On row click**, set **Action** to **Update current dashboard state**, set **Target dashboard state** to **default**, enable **Set entity from widget**, and set **State entity parameter name** to **Selected Entity**. Click **Add**.

8. The action appears in the list. Click **Save**, then click **Add** to place the widget on the dashboard.
</Steps>

### Configure Trendz widget

<Steps>
1. Click **Edit widget** on the Advanced analytics View widget.

2. On the **Data** tab, set **Entity alias** to **Selected Entity** and add **Name: name** as the data key.

3. Switch to the **Appearance** tab.

4. Enter **Machine** in **Filter name**.

5. Click **Apply**, then **Save**.
</Steps>

<Aside type="note">
  The value in **Filter name** must match the field name in the Trendz view filter exactly — including capitalisation.
</Aside>
