Defines how platform triggers alarms are delivers notifications.

- <DocLink product={props.product} path='user-guide/TBD/'>Alarm:</DocLink> State indicating a specific condition (e.g., high temperature, low battery). Supports severity levels, propagation to parent assets, and acknowledgment workflows.
- <DocLink product={props.product} path='user-guide/TBD/'>Alarm Rule:</DocLink> Logic that defines how alarm is created and cleared based on incoming telemetry and thresholds.
- <DocLink product={props.product} path='user-guide/TBD/'>Notification:</DocLink> Multi-channel message (Email, SMS, Slack, mobile app) triggered by user-defined rules in case of platform events or alarms.
- <DocLink product={props.product} path='user-guide/TBD/'>Notification Center:</DocLink> Centralized hub for users to configure and browse notifications.

<Aside type="tip">
Use **Alarm Propagation** to ensure critical device alarms are visible on parent asset dashboards without manual configuration.
</Aside>
