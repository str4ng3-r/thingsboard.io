Defines how platform triggers alarms are delivers notifications.

- <DocLink product={props.product} path='user-guide/alarms'>Alarm:</DocLink> State indicating a specific condition like **high temperature** or **low battery**. Supports severity levels, propagation to parent assets, and acknowledgment workflows.
- <DocLink product={props.product} path='user-guide/alarm-rules'>Alarm Rule:</DocLink> Logic that defines how alarm is created and cleared based on incoming telemetry and thresholds.
- <DocLink product={props.product} path='user-guide/notifications'>Notification:</DocLink> Multi-channel message triggered by user-defined rules in case of platform events or alarms: **email**, **SMS**, **Slack**, **Teams**, etc.
- <DocLink product={props.product} path='user-guide/notifications'>Notification Center:</DocLink> Centralized hub for users to configure and browse notifications.

<Aside type="tip">
Use **Alarm Propagation** to ensure critical device alarms are visible on parent asset dashboards without manual configuration.
</Aside>
