Defines incoming telemetry processing and workflows.

- <DocLink product={props.product} path='user-guide/calculated-fields'>Calculated Fields:</DocLink> Virtual telemetry or attributes derived from incoming data. Used for data transformations and enrichment.
- <DocLink product={props.product} path='user-guide/rule-engine'>Rule Engine:</DocLink> Framework for data processing, workflow automation, and external system integration.
  - <DocLink product={props.product} path='user-guide/rule-engine'>Rule Chain:</DocLink> Directed graph of connected Rule Nodes defining sequential data processing logic and execution paths.
  - <DocLink product={props.product} path='user-guide/rule-nodes'>Rule Node:</DocLink> Atomic processing component used to filter, transform, or route messages. Examples include data filtering, script execution, and external API calls.
  - <DocLink product={props.product} path='user-guide/rule-engine/queues'>Queues:</DocLink> Message persistence and delivery mechanism ensuring load balancing, fault tolerance, and guaranteed processing of telemetry.
- <DocLink product={props.product} path='user-guide/ai-models'>AI Models:</DocLink> Integrated AI models invoked via Rule Engine nodes for anomaly detection, pattern recognition, and Natural Language Generation (NLG) for telemetry.
