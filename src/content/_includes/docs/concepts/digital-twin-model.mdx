Defines how physical hardware is represented in the cloud.

- <DocLink product={props.product} path='user-guide/devices/'>Devices:</DocLink> Physical or virtual entity publishing telemetry (sensors) or receiving commands (actuators).
- <DocLink product={props.product} path='user-guide/TBD/'>Assets:</DocLink> Abstract entity used to group devices into logical structures (e.g., "Factory Floor" or "Building A").
- <DocLink product={props.product} path='user-guide/TBD/'>Relations:</DocLink> Connections between entities, allowing you to build parent-child hierarchies like Building (Asset) -> Floor (Asset) -> Thermostat (Device).
- <DocLink product={props.product} path='user-guide/TBD/'>Attributes:</DocLink> Entity metadata (e.g. serial number, firmware version, location).
- <DocLink product={props.product} path='user-guide/TBD/'>Telemetry:</DocLink> Time-series data points (temperature, humidity, voltage).
- <DocLink product={props.product} path='user-guide/TBD/'>Commands:</DocLink> Send commands from the platform back to the device and vice-versa.
