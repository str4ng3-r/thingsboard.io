## Introduction

The <a href="https://advdownload.advantech.com/productfile/Downloadfile5/1-24AIQLL/ADAM-6700_Series_User_Manual_Ed.2-FINAL.pdf" target="_blank" rel="noopener noreferrer">ADAM-6717</a> is an intelligent data acquisition gateway from Advantech's ADAM-6700 series. It features a built-in Node-RED graphic programming environment, making it straightforward to build data flows that send sensor and I/O data to cloud platforms over MQTT. This guide walks through creating a device in ThingsBoard, configuring MQTT credentials, and setting up a Node-RED flow on the ADAM-6717 to publish data to ThingsBoard.

## Prerequisites

- ADAM-6717 connected to the local network
- ThingsBoard instance — use <a href="/installations/choose-region/" target="_blank" rel="noopener noreferrer">ThingsBoard Cloud</a> or a local server
- A browser-accessible IP address for the ADAM-6717

## Step 1: Create a device in ThingsBoard

<Steps>
1. Log in to your ThingsBoard instance and go to **Entities &#8702; Devices**. Click <strong>+ Add device &#8702; Add new device</strong>.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-create-device.png',
     alt: 'ThingsBoard Devices page with Add device button',
     caption: 'Go to <strong>Entities &#8702; Devices</strong> and click the <strong>+ Add device &#8702; Add new device</strong> icon to create a new device.',
   }]} />

2. Enter the device name and any other relevant details, then click **Next: Credentials**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-device-details.png',
     alt: 'New device details form',
     caption: 'Enter the device name and other details, then click <strong>Next: Credentials</strong>.',
   }]} />

3. On the credentials page, set **Credentials type** to **MQTT Basic**. Enter a unique **Client ID**, and set a **Username** and **Password** for the device. Click **Add**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-mqtt-credentials.png',
     alt: 'MQTT Basic credentials configuration',
     caption: 'Set credentials type to <strong>MQTT Basic</strong> and fill in the Client ID, Username, and Password. Click <strong>Add</strong>.',
   }]} />
</Steps>

## Step 2: Access the ADAM-6717 Node-RED interface

The ADAM-6700 series has two Ethernet ports with default IP addresses printed on the device label: **10.0.0.1** (Eth0) and **11.0.0.1** (Eth1).

<ImageGallery images={[{
  src: '/src/assets/devices-library/adam-6717/adam-6717-network-ports.png',
  alt: 'ADAM-6717 network port label showing default IP addresses',
  caption: 'The device label shows two MAC IDs and their default IP addresses: <strong>10.0.0.1</strong> (Eth0) and <strong>11.0.0.1</strong> (Eth1).',
}]} />

<Steps>
1. Open a browser and enter the IP address of the ADAM-6717 port you are connected to. Click **Graphic programming tool by Node-RED** in the web interface.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-nodered-link.png',
     alt: 'ADAM-6717 web interface with Node-RED link',
     caption: 'Open the ADAM-6717 web interface and click <strong>Graphic programming tool by Node-RED</strong>.',
   }]} />

2. In the login popup, enter the default credentials and click **Sign in**:
   - **Username**: `root`
   - **Password**: `00000000`

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-nodered-login.png',
     alt: 'Node-RED login dialog',
     caption: 'Enter the default username <strong>root</strong> and password <strong>00000000</strong> to log in to Node-RED.',
   }]} />

3. The Node-RED interface opens. It has three sections: the **Nodes menu** (left), the **Programming area** (center), and the **Debug area** (right).

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-nodered-interface.png',
     alt: 'Node-RED interface with Nodes menu, Programming area, and Debug area',
     caption: 'The Node-RED interface: Nodes menu on the left, Programming area in the center, Debug area on the right.',
   }]} />
</Steps>

## Step 3: Configure the Node-RED flow

ThingsBoard receives telemetry in JSON format as key-value pairs. This step creates a minimal flow using an **inject** node and an **MQTT out** node to publish data.

<Steps>
1. Drag an **inject** node and an **MQTT out** node from the Nodes menu onto the workspace. Connect the output of the inject node to the input of the MQTT out node. Double-click the **MQTT out** node to open its settings.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-nodered-nodes.png',
     alt: 'Node-RED workspace with inject and MQTT out nodes connected',
     caption: 'Drag an <strong>inject</strong> node and an <strong>MQTT out</strong> node onto the workspace and connect them. Double-click the MQTT out node to configure it.',
   }]} />

2. In the **MQTT out** node settings, click the pencil icon next to the broker field to add a new MQTT broker. Fill in the connection details:

   | Field | Value |
   |-------|-------|
   | MQTT Broker | ThingsBoard Cloud host (e.g. `mqtt.thingsboard.cloud`) |
   | Port | `1883` |
   | Client ID | The Client ID set in Step 1 |

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-mqtt-broker.png',
     alt: 'MQTT broker connection settings',
     caption: 'Enter the ThingsBoard MQTT broker address and port <strong>1883</strong>, and set the Client ID matching the one created in ThingsBoard.',
   }]} />

3. Set the **Topic** to `v1/devices/me/telemetry` and the **QoS** to `1`.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-mqtt-topic.png',
     alt: 'MQTT out node topic and QoS settings',
     caption: 'Set the MQTT topic to <strong>v1/devices/me/telemetry</strong> and QoS to <strong>1</strong> for reliable message delivery.',
   }]} />

4. Switch to the **Security** tab and enter the **Username** and **Password** configured in Step 1. Click **Done** to save.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-mqtt-security.png',
     alt: 'MQTT broker security tab with username and password',
     caption: 'In the Security tab, enter the Username and Password set when creating the device in ThingsBoard.',
   }]} />

5. Click the **inject** button on the workspace to send a test payload to ThingsBoard. The inject node publishes JSON data to the MQTT topic, and ThingsBoard creates or updates the device telemetry.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/adam-6717/adam-6717-send-data.png',
     alt: 'Node-RED workspace showing inject button clicked and data sent',
     caption: 'Click the <strong>inject</strong> button to publish a test payload. Telemetry appears on the device in ThingsBoard.',
   }]} />
</Steps>
