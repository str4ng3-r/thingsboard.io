## Introduction

The <a href="https://nettra.tech/en/how-we-do-it/rtu-x/" target="_blank" rel="noopener noreferrer">Nettra RTU-X</a> is an IoT controller with digital and analog inputs and outputs, and integrated communication interfaces: modem, Ethernet, Bluetooth, 802.15.4, RS-485, RS-232, and GPS. It is configurable via the RTU-X Configuration Interface and runs a fully customizable script written in the Nettra scripting language.

This guide describes how to connect an RTU-X to ThingsBoard over Wi-Fi using MQTT, and visualize device data on a dashboard.

## Prerequisites

**Hardware:**

- Nettra RTU-X — 1×
- 12 VDC power supply — 1×

**Software:**

- <a href="http://wiki.nettra.tech/en/downloads" target="_blank" rel="noopener noreferrer">RTU-X Configuration Interface</a>
- ThingsBoard instance: use <a href="/installations/choose-region/" target="_blank" rel="noopener noreferrer">ThingsBoard Cloud</a> or install a <a href="/docs/pe/installation/" target="_blank" rel="noopener noreferrer">local server</a>

## Configure ThingsBoard

### Create a device

<Steps>
1. Go to **Entities > Devices** and click **+** to add a new device. Enter `RTU-X` as the name, leave the device type as **default**, and click **Add**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-tb-add-device.png',
     alt: 'ThingsBoard Devices page adding RTU-X device',
     caption: 'ThingsBoard Devices: creating the RTU-X device',
   }]} />

2. Open the device details and click **Copy access token**. Save this token — it will be used as `$RTU_DEMO_TOKEN` in the MQTT configuration.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-tb-credentials.png',
     alt: 'Device details with Copy access token button',
     caption: 'Device credentials: copying the access token ($RTU_DEMO_TOKEN)',
   }]} />
</Steps>

### Import the dashboard

Download the <a href="/partners/hardware/nettra/samples/resources/rtu-x-dashboard.json" download="rtu-x-dashboard.json">RTU-X dashboard JSON</a> and import it into ThingsBoard. See the <a href="/docs/user-guide/dashboards/#import-dashboard" target="_blank" rel="noopener noreferrer">dashboard import instructions</a> for details.

## Connect RTU-X to PC

The RTU-X creates a Wi-Fi access point on first use. Connect to it to configure the device, then switch it to join your local network.

<Steps>
1. Power on the RTU-X and connect your PC to the `RTU-X-******` Wi-Fi access point it broadcasts.

2. Open the RTU-X Configuration Interface. Go to **Home**, click **TCP/IP**, enter IP `192.168.4.1` and port `502`, then click **Connect**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-tcpip-connect.png',
     alt: 'RTU-X Configuration Interface TCP/IP connection dialog',
     caption: 'RTU-X Configuration Interface: connecting via TCP/IP to 192.168.4.1:502',
   }]} />

3. Once connected, the interface shows the device status.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-connected.png',
     alt: 'RTU-X Configuration Interface showing connected status',
     caption: 'Connection established: RTU-X status is visible in the Configuration Interface',
   }]} />

4. Go to **Communications** > **Wifi, Serial, Modbus**. Click **Station** and enter your Wi-Fi network credentials. Click **Apply Changes**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-wifi-setup.png',
     alt: 'RTU-X Wi-Fi Station configuration',
     caption: 'Wi-Fi Station mode: entering local network SSID and password',
   }]} />

5. Go back to **Home** and note the IP address shown under **WiFi STA information**. Disconnect from the RTU-X access point, reconnect your PC to your local network, then reconnect the Configuration Interface using the new IP address.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-ip-address.png',
     alt: 'RTU-X Home page showing WiFi STA IP address',
     caption: 'Home page: copying the Wi-Fi STA IP address assigned by the local router',
   }]} />
</Steps>

## Configure RTU-X

### MQTT

<Steps>
1. Go to **Communications** > **MQTT**.

2. Configure the MQTT connection:

   | Field | Value |
   |-------|-------|
   | Interface | Modem |
   | Format | ThingsBoard |
   | URI | `mqtt://mqtt.thingsboard.cloud:1883` |
   | Password | `$RTU_DEMO_TOKEN` (access token from the Create device step) |

3. Click **Apply Changes**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-mqtt-conf.png',
     alt: 'RTU-X MQTT configuration with ThingsBoard URI and access token',
     caption: 'MQTT settings: ThingsBoard Cloud URI and device access token as password',
   }]} />
</Steps>

### Load the script

Download the <a href="/partners/hardware/nettra/samples/resources/rtu-x-script.json" download="rtu-x-script.json">RTU-X script JSON</a> or use the code below.

<Aside type="note">
See the <a href="http://wiki.nettra.tech/en/script" target="_blank" rel="noopener noreferrer">Nettra script user manual</a> to write your own scripts.
</Aside>

```c
/*
 * DESCRIPTION:
 * - Send a variable to a ThingsBoard dashboard every 10 seconds.
 */

// Attributes
shared uint tLog = 10;

// Telemetry
telemetry float variable;

// Main loop
while (1)
{
    variable = 15;
    delay_loop(tLog * 1000);  // 10 seconds
    log(variable);
}
```

<Steps>
1. Go to **User Interface** and click **Load** to import the script file.

2. Click **Compile & Apply** to compile and deploy the script to the RTU-X.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/rtu-x/rtu-x-script.png',
     alt: 'RTU-X script editor with script compiled and applied',
     caption: 'Script editor: temperature script compiled and deployed to the RTU-X',
   }]} />
</Steps>
