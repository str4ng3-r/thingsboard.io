## Introduction

The **Things On Edge IOT Cricket** is an ultra-low-power Wi-Fi module designed for battery-powered IoT sensor nodes. It includes a built-in temperature sensor and supports external sensors like DS18B20. Cricket connects directly to Wi-Fi and sends data to ThingsBoard over HTTP — no gateway, no code required.

This guide covers assembling a battery-powered temperature sensor, connecting it to Wi-Fi, and configuring it to send data to ThingsBoard.

For full module specifications, see the <a href="https://docs.rs-online.com/a434/A700000007334637.pdf" target="_blank" rel="noopener noreferrer">IOT Cricket Wi-Fi module datasheet</a>.

## Prerequisites

- IOT Cricket Wi-Fi module — 1×
- 2× AAA battery holder — 1×
- 2× AAA batteries (use fresh, quality batteries like Duracell or Energizer)
- ThingsBoard instance: use <a href="/installations/choose-region/" target="_blank" rel="noopener noreferrer">ThingsBoard Cloud</a> or install a <a href="/docs/installation/" target="_blank" rel="noopener noreferrer">local server</a>

## Assemble the sensor

Cricket includes a built-in temperature sensor, so no additional hardware is required for a basic temperature reading. For more accurate results, an external sensor like DS18B20 is recommended.

Attach the batteries to the Cricket module. The device is ready to power on.

## Add the device in ThingsBoard

<Steps>
1. Go to **Entities &#8702; Devices** and click **+ Add device &#8702; Add new device**.

## Configure Cricket to send data to ThingsBoard

Cricket supports both MQTT and HTTP. This guide uses HTTP.

<Steps>
1. Return to the Cricket configuration portal (repeat the Wi-Fi connection steps if the portal has closed).
2. In the **URL** field, enter the ThingsBoard HTTP telemetry endpoint with your access token:

   ```
   https://YOUR_TB_HOST/api/v1/YOUR_ACCESS_TOKEN/telemetry
   ```

   For ThingsBoard Cloud, replace `YOUR_TB_HOST` with `thingsboard.cloud`.

3. In the **Payload** field, enter the JSON template:

   ```
   {"temperature":#temp}
   ```

   The `#temp` tag is replaced with the current temperature value before each transmission.

4. Set the transmission interval (the default is every 10 minutes).

   <ImageGallery images={[
     {
       src: '/src/assets/devices-library/iot-cricket-wifi-module/iot-cricket-wifi-module-cricket-config.png',
       alt: 'Cricket configuration portal showing URL, payload template, and interval fields filled in',
       caption: 'Cricket configuration: URL with access token, JSON payload template, and transmission interval',
     },
   ]} />

5. Press the reset button in the top-right corner of the portal to exit configuration mode and start sending data.

   <ImageGallery images={[
     {
       src: '/src/assets/devices-library/iot-cricket-wifi-module/iot-cricket-wifi-module-wifi-disconnect.png',
       alt: 'Cricket configuration portal with reset button highlighted in the top-right corner',
       caption: 'Press the reset button to exit configuration mode and start transmitting telemetry',
     },
   ]} />
</Steps>

<Aside type="tip">
  For more accurate temperature readings, connect an external DS18B20 sensor. See the <a href="https://thingsonedge.com/documentation" target="_blank" rel="noopener noreferrer">Things On Edge documentation</a> for wiring instructions.
</Aside>
