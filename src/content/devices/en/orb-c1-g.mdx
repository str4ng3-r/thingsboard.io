## Introduction

The <a href="https://www.senquip.com" target="_blank" rel="noopener noreferrer">Senquip ORB-C1-G</a> is a rugged, programmable telemetry device that connects to industrial sensors and systems including MODBUS, CAN Bus, current, voltage, and frequency inputs. It can maintain a simultaneous connection to the <a href="https://portal.senquip.com" target="_blank" rel="noopener noreferrer">Senquip Portal</a> and a third-party endpoint, allowing configuration changes and firmware updates from the Portal while sending measured data to ThingsBoard.

This guide describes how to send location, temperature, and CAN Bus data from an ORB-C1-G to ThingsBoard over MQTT, and visualize the data on a dashboard.

For the full hardware reference, see the <a href="https://docs.senquip.com/orb-c1-user-guide/" target="_blank" rel="noopener noreferrer">ORB-C1 User Guide</a> and <a href="https://docs.senquip.com/quickstart/ORB-X1_quickstart_1-0.pdf" target="_blank" rel="noopener noreferrer">Quick Start Guide</a>.

## Prerequisites

**Hardware:**

- Senquip ORB-C1-G — 1×

**Accounts:**

- <a href="https://portal.senquip.com" target="_blank" rel="noopener noreferrer">Senquip Portal</a> account with a registered device, configured for Wi-Fi or cellular connectivity
- ThingsBoard instance: use <a href="/installations/choose-region/" target="_blank" rel="noopener noreferrer">ThingsBoard Cloud</a> or install a <a href="/docs/pe/installation/" target="_blank" rel="noopener noreferrer">local server</a>

## Configure ThingsBoard

### Create a device

<Steps>
1. Go to **Entities &#8702; Devices** and click **+ Add device &#8702; Add new device**. Enter a name (like `Demo Senquip Device`) and a short label (like `SQ1`). Select **MQTT** as the transport type and click **Next: Credentials**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/orb-c1-g/orb-c1-g-tb-add-device.jpg',
     alt: 'ThingsBoard add device dialog with name, label, and MQTT transport type',
     caption: 'Add device: entering the device name, label, and selecting MQTT transport type',
   }]} />

2. Select **MQTT Basic** as the credentials type. Enter the **Client ID**, **User Name**, and **Password** that you will also configure on the ORB-C1-G. Click **Add**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/orb-c1-g/orb-c1-g-tb-credentials.jpg',
     alt: 'Device credentials dialog with MQTT Basic credentials fields',
     caption: 'Device credentials: setting MQTT Basic Client ID, User Name, and Password',
   }]} />
</Steps>

## Configure the ORB-C1-G

All configuration is performed on the <a href="https://portal.senquip.com" target="_blank" rel="noopener noreferrer">Senquip Portal</a>.

### General setup

<Steps>
1. Give the device a meaningful name: **Setup > General > Name** = `Demo Senquip Device`.

2. Set the measurement and transmission interval to 5 seconds: **Setup > General > Base Interval** = `5`.

3. Enable the GPS peripheral and set it to measure on every base interval: **Setup > Internal > GPS > Interval** = `1`.

4. Enable the CAN peripheral and set it to measure on every base interval: **Setup > External > CAN > Interval** = `1`. If this is the only CAN device on the network, select **Tx Enable** so the ORB-C1-G acknowledges CAN messages.

5. Click **Save**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/orb-c1-g/orb-c1-g-senquip-general.jpg',
     alt: 'Senquip Portal General settings page with name and base interval configured',
     caption: 'Senquip Portal General settings: device name, base interval, GPS, and CAN peripheral configuration',
   }]} />
</Steps>

### Configure the MQTT endpoint

<Steps>
1. Enable the MQTT endpoint and configure the connection:

   | Setting | Value |
   |---------|-------|
   | **Setup > Endpoint > MQTT** | Enabled |
   | Broker Address | `thingsboard.cloud:1883` (or your ThingsBoard server address) |
   | Client ID | The Client ID entered in ThingsBoard credentials |
   | Data Topic | `v1/devices/me/telemetry` |
   | Username | The User Name entered in ThingsBoard credentials |
   | Password | The Password entered in ThingsBoard credentials |

   Click **Save**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/orb-c1-g/orb-c1-g-senquip-mqtt.jpg',
     alt: 'Senquip Portal MQTT endpoint settings with broker address and credentials',
     caption: 'Senquip Portal MQTT endpoint: broker address, client ID, topic, and credentials',
   }]} />
</Steps>

## Verify data

Senquip devices send data in JSON format. Each measurement has a key and a value — for example, GPS latitude uses the key `gps_lat` with a value like `-32.70245`. Nested data such as CAN messages appears as separate rows.

<Steps>
1. In ThingsBoard, go to **Entities > Devices**, open the device, and navigate to the **Latest telemetry** tab. ThingsBoard automatically parses the JSON payload into a table of keys and values, including the nested CAN identifiers.

</Steps>

## Build a dashboard

<Steps>
1. On the **Latest telemetry** tab, click a data key row and press **Show on widget**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/orb-c1-g/orb-c1-g-tb-widget-select.jpg',
     alt: 'ThingsBoard Latest telemetry with a key selected and Show on widget button',
     caption: 'Latest telemetry: selecting a telemetry key and clicking Show on widget',
   }]} />

2. Choose a widget bundle matching your data type (for temperature, use **Analog gauges**), then click **Add to dashboard**. Create a new dashboard named `Senquip Demo` or add to an existing one.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/orb-c1-g/orb-c1-g-tb-add-widget.jpg',
     alt: 'Widget bundle selection with Add to dashboard option',
     caption: 'Widget selection: choosing an analog gauge widget and adding it to a new dashboard',
   }]} />

3. Repeat for additional telemetry keys. For map widgets, configure the **Latitude** and **Longitude** under advanced settings to use the `gps_lat` and `gps_lon` keys. Resize and arrange widgets as needed and click **Save**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/orb-c1-g/orb-c1-g-tb-dashboard.jpg',
     alt: 'ThingsBoard dashboard with ORB-C1-G telemetry widgets including map and gauges',
     caption: 'Completed dashboard: map, gauges, and charts displaying live ORB-C1-G telemetry',
   }]} />
</Steps>
