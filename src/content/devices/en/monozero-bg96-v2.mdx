## Introduction

The monoZero BG96 V2 is a cellular IoT development kit built around an STM32L4 Arm Cortex-M4 MCU and a Quectel BG96 modem that supports LTE-M and NB-IoT. It exposes 23 I/O peripherals (USB, LPUART, 2× UART, 3× I2C, 3× SPI, CAN) and is programmable via SWD. Combined with the monoZero Grove Board, peripheral sensors connect through standard QWIIC/Grove ports.

This guide walks through building an LPWA GPS tracker: assemble the hardware, flash firmware using the monoZ SDK, and display live GPS location data on a ThingsBoard dashboard.

For the full hardware reference, see the <a href="https://docs.monoz.io/v1/md__u_i_d_o_c__b_g96__v2doc.html" target="_blank" rel="noopener noreferrer">official monoZero BG96 V2 documentation</a>.

## Prerequisites

**Hardware:**

- monoZero BG96 V2 — 1×
- monoZero Grove Board — 1×
- NEO-6M GPS module — 1×
- LTE-M or NB-IoT nano SIM — 1×
- RP-SMA antenna (2.4–2.6 GHz) — 1×
- STLink V2 debugger (for SWD flashing) — 1×
- USB-B cable — 1×

**Software:**

- <a href="https://monoz.io/" target="_blank" rel="noopener noreferrer">monoZ SDK</a>
- <a href="https://www.st.com/en/development-tools/stm32cubeide.html" target="_blank" rel="noopener noreferrer">STM32CubeIDE</a>

## Hardware setup

### monoZero BG96 board

The monoZero BG96 V2 hosts a Quectel BG96 modem with global LTE-M and NB-IoT coverage alongside an STM32L4 MCU. Firmware is loaded via the SWD interface using an STLink V2 debugger.

<ImageGallery images={[{
  src: '/src/assets/devices-library/monoz/monoz-bg96-board.png',
  alt: 'monoZero BG96 V2 board',
  caption: 'monoZero BG96 V2: STM32L4 MCU and Quectel BG96 modem with 23 I/O peripherals',
}]} />

### monoZero Grove Board

The Grove Board exposes pin-based peripherals through QWIIC/Grove ports, simplifying sensor wiring. When stacked on the BG96 V2, Grove port 3 maps to the LPUART peripheral used by the GPS module.

<ImageGallery images={[{
  src: '/src/assets/devices-library/monoz/monoz-grove-board.png',
  alt: 'monoZero Grove Board',
  caption: 'monoZero Grove Board: Grove port 3 maps to LPUART for the GPS module connection',
}]} />

### Assembly

Connect the NEO-6M GPS module to Grove port 3 on the Grove Board, then stack the Grove Board on the monoZero BG96 V2. Attach the SMA antenna and power the board via USB-B.

## Configure firmware

Download the GPS tracker project from the <a href="https://github.com/Meritech-monoZ/GPS_NEO6M" target="_blank" rel="noopener noreferrer">Meritech-monoZ/GPS_NEO6M repository</a> and open it in STM32CubeIDE.

<Aside type="note">
See <a href="https://docs.monoz.io" target="_blank" rel="noopener noreferrer">docs.monoz.io</a> for STM32CubeIDE setup, CLI configuration, and SWD flashing instructions.
</Aside>

<Steps>
1. Open **Lib > tool_gen > MZ_GPSSensor** and update the MQTT client configuration to match the ThingsBoard device credentials you will create in the next section.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-sdk-mqtt-config.png',
     alt: 'monoZ SDK MQTT client configuration in STM32CubeIDE',
     caption: 'MZ_GPSSensor config: set Client ID, Username, and Password to match the ThingsBoard device credentials',
   }]} />

2. Open **Lib > tool_gen > MZ_modemconfig** and update the APN for your SIM provider.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-sdk-apn-config.png',
     alt: 'monoZ SDK modem APN configuration',
     caption: 'MZ_modemconfig: set the APN string for your LTE-M or NB-IoT SIM provider',
   }]} />

3. Build the project and flash it to the board via SWD. A successful flash produces the following CLI output.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-cli-output.png',
     alt: 'STM32 CLI output after successful firmware flash',
     caption: 'Expected CLI output after successful flash: modem initialized and MQTT client ready',
   }]} />
</Steps>

## Configure ThingsBoard

### Create a device

<Steps>
1. Go to **Entities > Devices** and click **+** to add a new device.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-device-group.png',
     alt: 'ThingsBoard Devices page with Add button',
     caption: 'ThingsBoard Devices: clicking + to create a new device',
   }]} />

2. Enter a device name, for example `R40`, and click **Add**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-add-device.png',
     alt: 'Add device dialog',
     caption: 'Add device dialog: entering a name and clicking Add',
   }]} />

3. Click the **Manage credentials** (shield) icon on the newly created device.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-credentials.png',
     alt: 'Device list showing Manage credentials icon',
     caption: 'Device list: clicking the shield icon to open credential management',
   }]} />

4. Set **Credentials type** to **MQTT Basic**. Enter the **Client ID**, **Username**, and **Password** that match the values configured in the firmware, then click **Save**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-mqtt-credentials.png',
     alt: 'MQTT Basic credentials form',
     caption: 'MQTT Basic credentials: Client ID, Username, and Password must exactly match the firmware configuration',
   }]} />
</Steps>

## Create a dashboard

<Steps>
1. Power on the monoZero. Once the device connects and sends data, open its details in ThingsBoard and go to the **Latest telemetry** tab. Select the `latitude` and `longitude` keys, then click **Show on widget**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-telemetry.png',
     alt: 'Latest telemetry tab with latitude and longitude keys selected',
     caption: 'Latest telemetry: selecting latitude and longitude, then clicking Show on widget',
   }]} />

2. Select the **OpenStreetMap** widget from the **Maps** bundle, then click **Add to dashboard**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-widget-select.png',
     alt: 'OpenStreetMap widget selected in Maps bundle',
     caption: 'Widget picker: selecting OpenStreetMap from the Maps bundle',
   }]} />

3. Select an existing dashboard or create a new one, then click **Add**.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-add-dashboard.png',
     alt: 'Add to dashboard dialog',
     caption: 'Choosing a target dashboard and clicking Add to place the map widget',
   }]} />

4. Open the dashboard. The device location appears live on the map.

   <ImageGallery images={[{
     src: '/src/assets/devices-library/monoz/monoz-tb-dashboard.png',
     alt: 'ThingsBoard dashboard with OpenStreetMap widget showing GPS location',
     caption: 'Completed dashboard: live GPS location from the monoZero BG96 V2 tracker',

</Steps>
