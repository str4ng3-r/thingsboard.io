## Introduction

<a href="https://machineastro.com/vibit/" target="_blank" rel="noopener noreferrer">VIBit</a> is a compact, rugged, tri-axial vibration, temperature, and acoustic sensor designed for industrial anomaly detection. It supports Wi-Fi connectivity and integrates seamlessly with the MachineAstro VIBit Application for configuration and monitoring.<br /><br /><br /><br /><br />

## Prerequisites

To continue with this guide, we will need the following:
- <a href="https://localhost" target="_blank" rel="noopener noreferrer">ThingsBoard account</a>

- <a href="/docs/devices-library/resources/manuals/vibit.pdf" target="_blank" rel="noopener noreferrer">Sensor user guide</a>
- <a href="https://machineastro.com/vibit/" target="_blank">VIBit</a>

- MachineAstro VIBit Application installed on a smartphone.
- Wi-Fi network credentials for sensor connectivity.
- MQTT Basic credentials:
  - **Username**: sensor
  - **Password**: cimcon
  - **Client ID**: VIBit Device BLE MAC (Bluetooth MAC address of the sensor)

## Create Device profile on ThingsBoard

First, create a new device profile in ThingsBoard that uses the **MQTT transport** and the standard MQTT topics for telemetry and attributes.

<Steps>

1. Log in to your ThingsBoard instance.
2. Navigate to **Device profiles** page under the **Profiles** section.
3. Click the "**plus**" button, and then select "**Add new device profile**" from drop-down menu.
4. Enter **MQTT** as the device profile name.
5. Click "**Next: Transport configuration**" button.
6. Select **MQTT** as transport type.
7. Leave default **MQTT device topic filters**.
8. Click **Add** to create device profile.

</Steps>

## Create Device on ThingsBoard

<Steps>

1. Navigate to **Devices** page under the **Entities** section.
2. Click on "**+**" to add a new device, and then select "**Add new device**" from drop-down menu.
3. Enter a name using the VIBit Sensor BLE MAC address (e.g., "F4B3B1A1EFAC").
4. Select the previously created **MQTT** device profile in the "Device profile" field.
5. Click **Next: Credentials** button.
6. On the **Credentials** tab:
    - Select **MQTT Basic** as the credential type.
    - Use the **BLE MAC address of the VIBit device** as the **Client ID**.
    - Enter **sensor** as the **Username**.
    - Enter **cimcon** as the **Password**.
7. Click **Add** button to create the device.

</Steps>

## Connect device to ThingsBoard via VIBit Mobile Application

### Prerequisites

- A user must be present on Thingsboard platform.
- VIBit Device must be present on Thingsboard platform.
   <br />

- Power on the VIBit sensor.
- Open the VIBit Mobile Application on your smartphone.
- The VIBit mobile application can connect to any ThingsBoard platform by selecting the "**Custom**" option from the drop-down menu and entering either the static public IP address or the HTTPS domain name. Below is one sample image.

<DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-1.png" alt="image" />

<DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-2.png" alt="image" />

<Steps>

1. Go to Connect Section on the application. It will display list of VIBit sensors available in the Bluetooth Range.

   <DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-3.png" alt="image" />

2. Search the respective device and click on the **Connect** button. (Assuming that Device is already exists on the Thingsboard.)

   <DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-4.png" alt="image" />

3. Configure the sensor&#39;s Wi-Fi settings using the app:
   - Enter the Wi-Fi SSID and password.
   - Input the ThingsBoard server URL.
   - Enter the MQTT **Username** as sensor, **Password** as cimcon, and **Client ID** as the BLE MAC address.
   - Save the settings to apply the configurations.

</Steps>

<DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-5.png" alt="image" />

<DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-6.png" alt="image" />

- Check Cloud and Wi-Fi Connectivity Status in the Mobile Application.

<DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-7.png" alt="image" />

### Check Data on ThingsBoard

- In ThingsBoard, navigate to the "**Devices**" section and select your VIBit device.
- Access the "**Latest telemetry**" tab to view real-time data such as:
  - 3-axis acceleration and velocity RMS
  - FFT Timeseries Data
  - Temperature and acoustic readings

<DocImage src="/src/assets/devices-library/ready-to-go-devices/machine-astro/vibit/vibit-mobile-app-8.png" alt="image" />

<Aside type="note">

Use the <a href="/docs/user-guide/dashboards/" target="_blank" rel="noopener noreferrer">Dashboard</a> to visualize data trends and analyze equipment health over time.

</Aside>

## Conclusion

Integrating VIBit with ThingsBoard using MQTT Basic authentication ensures secure and efficient monitoring of industrial equipment, facilitating predictive maintenance and reducing downtime. The combination of real-time data transmission and comprehensive dashboards empowers maintenance teams to make informed decisions.
