## Introduction

<a href="https://www.iotracker.eu/products/iobutton" target="_blank" rel="noopener noreferrer">ioButton Panic button</a> ioButton is a versatile solution, pre-
configured for various emergency
scenarios. When the panic button is
activated, users receive immediate
notification through the prominent LED
ring, accompanied by haptic feedback
from the vibration motor. With a standard battery life of over 10 years, virtually no maintenance is required.
The ioButton can send 1,000,000 messages on SF-7 and 100,000 on SF-12!
By default, the ioButton sends a heartbeat message every 4 hours.

<Banner variant="peFeature">This feature is available only for **Platform Integrations**.</Banner>

<br />

## Prerequisites

To continue with this guide we will need the following:

- <a href="https://www.iotracker.eu/products/iobutton" target="_blank">ioButton Panic button </a>
- LoRaWAN® gateway (in our case <a href="/device-library/ug56-lorawan-gateway/?platform=pe" target="_blank" rel="noopener noreferrer">UG56 LoRaWAN® Gateway</a>)
- Configured integration on networks server and ThingsBoard
- [Network Server account](#device-connection)

- <a href="https://localhost" target="_blank" rel="noopener noreferrer">ThingsBoard account</a>

## Device connection

### Device configuration

To connect and send data we should configure the device and network server.
At first we are going to configure the device, and save required information for network server configuration.
To add a device to network server and get information from it, we will need the following device parameters:

- **Device EUI** — device identifier
- **Application EUI** — application identifier
- **Application Key** — application key to identify device. We recommend to use a generated key, not from the example!

The parameters above are required for connection.

Depending on the network server, you may also need to provide join type (OTAA) and LoRaWAN version.

### Prerequisites

- **Android app & account**
  Install the ioTracker Configurator app from Google Play and log in with your ioTracker account.
- **Device compatibility**
  Only with ioTracker accounts after February 1, 2022 are supported.
- **Bluetooth & permissions**
  Enable Bluetooth and grant Location/"Nearby devices" permissions to the app.

### 1. Scanning & Connecting

<Steps>

1. Open **ioTracker Configurator** and tap **Refresh** (or **Scan**) to discover devices.
2. Select your ioTracker or ioButton from the list and tap **Login**; enter your credentials to establish Bluetooth.

</Steps>

### 2. LoRaWAN Menu

From the **Main menu**, tap **Configuration** to access LoRaWAN functions.

<DocImage src="/src/assets/devices-library/io-things-lorawan-menu.png" alt="LoRaWAN menu screenshot" />

### Change LoRaWAN Keys

<Steps>

1. Click the **edit/pencil** icon next to **DevEUI**, **AppEUI**, and **AppKey**.
2. Enter your new LoRaWAN **DevEUI**, **AppEUI**, and **AppKey**.
3. Click **Update**.
4. **Reboot** the device to activate the new keys.

</Steps>


<DocImage src="/src/assets/devices-library/io-things-lorawan-lorakeys.png" alt="Change LoRaWAN keys screenshot" />

### Check LoRaWAN Status

- **Active Region**: the frequency band in use.
- **LoRaWAN Version**: the specification version.
- **Join Status**: **JOINED** or **NOT JOINED**.
- **Last Uplink**: timestamp, payload, message type, DataRate, channel, TxPower, TxToA, UpCntr.
- **Last Downlink**: timestamp, payload, DataRate, RxSlot, port, RSSI, SNR, ACK flag, DownCntr.

### Add a device on the Chirpstack

We need to add a device on the <a href="https://chirpstack.io" target="_blank" rel="noopener noreferrer">Chirpstack</a>.

To add a device, you can follow next steps:

<Steps>

1. Login to Chirpstack server.
2. Go to the **Device profiles** page and click on **Add device profile** button.
3. Fill the fields and click on **Submit** button.
4. Go to the **Applications** page, click on your application and press **Add device** button.
5. Fill parameters with values from the device configuration. Then choose previously created device profile and click on **Submit** button.
6. Put your **Application key** to the field and click on **Submit** button to save the device.

</Steps>

### Uplink converter in ThingsBoard integration

<b>Step 1: Download the JSON file</b>

Download the JSON file with the uplink converter settings by clicking the button below.

<br /><b>Step 2: Import the Uplink converter into ThingsBoard</b>

After downloading the file with the Uplink converter configuration, import it into ThingsBoard:

<Steps>

1. Navigate to the "**Data converters**" page in the "**Integration center**" section.
2. Click the "**+**" icon in the top-right corner of the page and select "**Import converter**" from the drop-down menu.
3. Drag and drop the downloaded JSON file (containing the uplink converter configuration) into the import window, then click "**Import**".
4. The uplink converter is now successfully imported.

</Steps>

## Device connection

### Device configuration

To connect and send data we should configure the device and network server.
At first we are going to configure the device, and save required information for network server configuration.
To add a device to network server and get information from it, we will need the following device parameters:

- **Device EUI** — device identifier
- **Application EUI** — application identifier
- **Application Key** — application key to identify device. We recommend to use a generated key, not from the example!

The parameters above are required for connection.

Depending on the network server, you may also need to provide join type (OTAA) and LoRaWAN version.

### Prerequisites

- **Android app & account**
  Install the ioTracker Configurator app from Google Play and log in with your ioTracker account.
- **Device compatibility**
  Only with ioTracker accounts after February 1, 2022 are supported.
- **Bluetooth & permissions**
  Enable Bluetooth and grant Location/"Nearby devices" permissions to the app.

### 1. Scanning & Connecting

<Steps>

1. Open **ioTracker Configurator** and tap **Refresh** (or **Scan**) to discover devices.
2. Select your ioTracker or ioButton from the list and tap **Login**; enter your credentials to establish Bluetooth.

</Steps>

### 2. LoRaWAN Menu

From the **Main menu**, tap **Configuration** to access LoRaWAN functions.

### Change LoRaWAN Keys

<Steps>

1. Click the **edit/pencil** icon next to **DevEUI**, **AppEUI**, and **AppKey**.
2. Enter your new LoRaWAN **DevEUI**, **AppEUI**, and **AppKey**.
3. Click **Update**.
4. **Reboot** the device to activate the new keys.

</Steps>

### Check LoRaWAN Status

- **Active Region**: the frequency band in use.
- **LoRaWAN Version**: the specification version.
- **Join Status**: **JOINED** or **NOT JOINED**.
- **Last Uplink**: timestamp, payload, message type, DataRate, channel, TxPower, TxToA, UpCntr.
- **Last Downlink**: timestamp, payload, DataRate, RxSlot, port, RSSI, SNR, ACK flag, DownCntr.
