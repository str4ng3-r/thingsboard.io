The EXXN IoT Gateway is a versatile device with an ARM processor that can be adapted to a variety of use cases, including monitoring environmental sensors, energy consumption, radon gas levels, marine environments, and edge computing.

### Prerequisites

To continue with this guide we will need the following:

- <a href="https://exxn.es/en/iot-gateway/" target="_blank">IoT Gateway Cell 1024 </a>
- Device have to be connected to the Internet via Ethernet, Modem or WIFI.
- <a href="https://YOUR_TB_HOST" target="_blank" rel="noopener noreferrer">ThingsBoard account</a>

## Create device on ThingsBoard

Open your platform instance or ThingsBoard <a href="https://YOUR_TB_HOST" target="_blank" rel="noopener noreferrer">cloud</a> in browser and login as tenant administrator.

<Steps>

1. Go to "<b>Device groups</b>" tab -> "<b>All</b>" and click "plus" button to add a new device. Enter device name, select existing or create a new device profile and click on "<b>Add</b>" button.
2. Your device has been created. Open its details and copy auto-generated <b>access token</b> by clicking on the "Copy access token" button.

</Steps>

## Connect device to ThingsBoard



<Steps>

1. Open a Browser to the administration web of Cell 1024 using the URL: https://[IP_DEVICE] and go to the "<b>Cloud</b>" tab.
2. Activate the Cloud control and configure all the parameters to connect the device to the specific ThingsBoard platform via MQTT.
3. Click "<b>Save configuration</b>" button.

</Steps>

<ImageGallery images={[{"src":"/src/assets/devices-library/ready-to-go-devices/iot-gateway-cell-1024/conn1.png","alt":"Open a Browser to the administration web of Cell 1024 using the URL: https://[IP_DEVICE] and go to the \"<b>Cloud</b>\" tab.","caption":"Open a Browser to the administration web of Cell 1024 using the URL: https://[IP_DEVICE] and go to the \"<b>Cloud</b>\" tab."},{"src":"/src/assets/devices-library/ready-to-go-devices/iot-gateway-cell-1024/conn2.png","alt":"Activate the Cloud control and configure all the parameters to connect the device to the specific ThingsBoard platform via MQTT.","caption":"Activate the Cloud control and configure all the parameters to connect the device to the specific ThingsBoard platform via MQTT."},{"src":"/src/assets/devices-library/ready-to-go-devices/iot-gateway-cell-1024/conn3.png","alt":"Click \"<b>Save configuration</b>\" button.","caption":"Click \"<b>Save configuration</b>\" button."}]} />

| Configuration parameters | Description |
|-|-|
|**Cloud Platform**| Select ThingsBoard. |
|**MQTT broker URL**| URL to de Broker of the server we want to integrate to. |
|**MQTT broker port**| Number of the port used by the server. |
|**TLS**| Select true if the server use the Transport Layer Security protocol. |
|**Connection Type**| Select 'Access Token' option. We will use an Access token previously created in ThingsBoard. |
|**Access Token**| Indicate the Access token previously copied in ThingsBoard. |

<Aside type="note">

**Note**

Currently, the IoT EXXN Gateways use the 'Access Token' integration method.
EXXN team is working on a Pre-Provisioning integration method that will eliminate the need to copy this Access token on the device.

</Aside>

To verify that the device is connected correctly to ThingsBoard, go to the **Device groups** menu -> **All** devices, select your device.
In the **device details** select **client attributes** tab and check if the client attributes have been communicated to the device.

<Steps>

1. If you did everything is correct, we will see client attributes like the <i>serial_number</i>, <i>last_reboot</i>, <i>device_model</i>, etc.

The EXXN IoT Gateway will connect to ThingsBoard using the MQTT API.
We have previously covered how to configure the device to connect to ThingsBoard.
Now, we will show the steps to configure the device in ThingsBoard in order to monitor data and manage the device.

## Check data on ThingsBoard

Create a <a href="/docs/pe/user-guide/dashboards/" target="_blank" rel="noopener noreferrer">dashboard</a> to visualize telemetry values in widgets.

<Steps>

1. Go to the <b>Dashboard groups</b> tab -> <b>All</b>. Creat new dashboard by clicking the "plus" button in the upper right corner of the dashboards page. Enter dashboard name and click on "<b>Add</b>" button.
2. Create <b>Radial gauge</b> widget from the <b>Analog gauges</b> bundle. We used <b><i>sc0_temperature</i></b>.
3. The data will be shown on the selected widget.

</Steps>

### Troubleshooting
 - The most common problem in the integration process is not having a connection with the MQTT Broker. Make sure the device is connected to the internet and is able to communicate with the ThingsBoard Broker.
 - Another common problem is not setting the correct Access Token. Check that the provided Access token is configured in ThingsBoard for the device.
