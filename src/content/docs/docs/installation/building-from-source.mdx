This guide walks you through downloading and building ThingsBoard from source code. Instructions are tested on Ubuntu 24.04 LTS and CentOS 8/9.

## Required tools

### Java

ThingsBoard is built using **Java 17**. See <DocLink product={Products.CE} path="installation/ubuntu#step-1-install-java-openjdk">Install Java 17 on Ubuntu</DocLink> for installation instructions.

Verify the installation:

```bash
java -version
```

### Maven

ThingsBoard requires **Maven 3.1.0** or newer.

<Tabs>
<TabItem label="Ubuntu">

```bash
sudo apt-get install maven
```

</TabItem>
<TabItem label="CentOS">

```bash
sudo yum install maven
```

</TabItem>
</Tabs>

<Aside type="note">
On certain Linux machines, Maven installation may set Java 7 as the default JVM. If this happens, reconfigure your system to use Java 17.
</Aside>

## Source code

Clone the <a href="https://github.com/thingsboard/thingsboard" target="_blank">ThingsBoard repository</a> from GitHub:

<Code code={cloneRepo} lang="bash" />

<Aside type="note">
**Building Docker images on Windows requires LF line endings.** Before cloning the repo, set the Git <a href="https://git-scm.com/docs/git-config#Documentation/git-config.txt-coreautocrlf" target="_blank">core.autocrlf</a> configuration option to `input`:

```bash
git config --global core.autocrlf input
```

Failing to do so may cause runtime errors when running Docker images.
</Aside>

## Build the project

From the project root directory, run:

```bash
mvn clean install -DskipTests
```

This command builds the project, skips unit and integration tests, and produces installation artifacts.

## Build local Docker images (optional)

To build local Docker images for ThingsBoard services, ensure <a href="https://docs.docker.com/engine/install/" target="_blank">Docker</a> is installed. Then run:

```bash
mvn clean install -DskipTests -Ddockerfile.skip=false
```

List the built images:

```bash
docker images
```

## Build artifacts

After a successful build, installation packages are available in:

```bash
application/target
```

This directory contains:
- `.deb` packages for Debian/Ubuntu
- `.rpm` packages for CentOS/RHEL
- Windows installation packages

## Running tests (optional)

This requires <a href="https://docs.docker.com/engine/install/" target="_blank">Docker</a> and <a href="https://docs.docker.com/compose/install/" target="_blank">Docker Compose</a>. Make sure <a href="https://docs.docker.com/engine/install/linux-postinstall/#manage-docker-as-a-non-root-user" target="_blank">Docker is configured to run as a non-root user</a>.

<Aside type="note">
Master and release branches are already tested, so you can skip tests and avoid installing Docker.
</Aside>

### Unit and integration tests

```bash
mvn clean install
```

### Black-box tests

Black-box tests are located in the <a href="https://github.com/thingsboard/thingsboard/tree/master/msa/black-box-tests" target="_blank">msa/black-box-tests</a> directory. Follow the instructions in the <a href="https://github.com/thingsboard/thingsboard/blob/master/msa/black-box-tests/README.md" target="_blank">README</a> to execute them.

<Aside type="note">
Estimated runtime is approximately 1 hour, depending on hardware performance.
</Aside>

## Tips and tricks

Clean Maven cache:

```bash
rm -rf ~/.m2/repository
```

Clean Gradle cache:

```bash
rm -rf ~/.gradle/caches/
```

Clean node modules:

```bash
rm -rf ui-ngx/node_modules
```

Build in parallel, format headers, build Docker images:

```bash
mvn -T 0.8C license:format clean install -DskipTests -Ddockerfile.skip=false
```

## Common build and runtime errors

### Docker runtime error

If you see the following error when running a locally built Docker image:

```bash
Standard_init_linux.go:175 exec user process caused no such file
```

This usually indicates incorrect <a href="https://git-scm.com/docs/git-config#Documentation/git-config.txt-coreautocrlf" target="_blank">line endings</a>.

**Solution:** reconfigure Git to use LF line endings, re-clone the repository, and rebuild the Docker images. See the [Source code](#source-code) section above for details.
