---
title: Managing instances
description: How to manage running ThingsBoard PE instances through the License Server, including cluster and floating mode setups.
---

The **ThingsBoard License Server** allows managing running instances. Once the license key is applied, the ThingsBoard client receives an InstanceID from the License Server. One license may be associated with a single instance.

With a modern microservices deployment approach, a single license key may run per all nodes in the ThingsBoard cluster. This minimizes efforts for cluster management and removes the manual work required to add/remove nodes in the cluster. While launching several cluster nodes in a floating mode, you may use the license key within Node 1, then deactivate that node and apply the existing key within Node 2.

:::caution
The same key may not be used within two or more nodes simultaneously. According to the License Server [architecture](/docs/license-server/what-is-license-server/#architecture), each ThingsBoard client sends a license check request. If the license key is bound with more than one InstanceID, ThingsBoard will shut down the instances.
:::
