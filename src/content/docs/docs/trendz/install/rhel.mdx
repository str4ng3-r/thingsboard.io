This guide explains how to install **Trendz Analytics** on **RHEL/CentOS 7, 8, or 9**.

<Aside type="caution" title="Docker is recommended">
Certain Trendz features — **predictions, metric explorations, and Python calculation fields** — require the Trendz Python Executor, which runs only via Docker. Even on RHEL/CentOS, Docker is needed to fully utilize all Trendz capabilities.

Consider using the <DocLink product={Products.TRENDZ} path="install/docker">Docker (Linux, macOS)</DocLink> installation guide instead. Proceed here only if you have a specific reason to use an RPM package setup.
</Aside>

## Prerequisites

<Prerequisites product={Products.TRENDZ} />
- <a href="https://docs.docker.com/engine/installation/" target="_blank">Docker CE</a> and <a href="https://docs.docker.com/compose/install/" target="_blank">Docker Compose</a> (required for the Python Executor)

**CentOS 7 only** — install `wget`, `nano`, and the EPEL repository before proceeding:

```bash
sudo yum install -y nano wget
sudo yum install -y https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
```

### Step 3. Configure database

Trendz uses PostgreSQL. You can install it on the same server or use a managed PostgreSQL service.

**Install PostgreSQL 16:**

<Tabs>
<TabItem label="RHEL/CentOS 8">

```bash
sudo dnf update
sudo dnf -y install https://download.postgresql.org/pub/repos/yum/reporpms/EL-8-x86_64/pgdg-redhat-repo-latest.noarch.rpm
sudo dnf -qy module disable postgresql
sudo dnf -y install postgresql16 postgresql16-server postgresql16-contrib
```

</TabItem>
<TabItem label="RHEL/CentOS 9">

```bash
sudo dnf update
sudo dnf -y install https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm
sudo dnf -qy module disable postgresql
sudo dnf -y install postgresql16 postgresql16-server postgresql16-contrib
```

</TabItem>
<TabItem label="CentOS 7">

```bash
sudo yum update
sudo yum -y install https://download.postgresql.org/pub/repos/yum/reporpms/EL-7-x86_64/pgdg-redhat-repo-latest.noarch.rpm
sudo yum -y install postgresql16 postgresql16-server postgresql16-contrib
```

</TabItem>
</Tabs>

```bash
sudo /usr/pgsql-16/bin/postgresql-16-setup initdb
sudo systemctl start postgresql-16
sudo systemctl enable --now postgresql-16
```

Set a password for the `postgres` user:

```bash
sudo -u postgres psql -c "\password"
```

Enter and confirm the password when prompted.

**Configure MD5 authentication:**

Edit `pg_hba.conf`:

```bash
sudo nano /var/lib/pgsql/16/data/pg_hba.conf
```

Locate the IPv4 local connections line and replace `ident` with `md5`:

```text
# IPv4 local connections:
host    all             all             127.0.0.1/32            md5
```

Restart PostgreSQL to apply the change:

```bash
sudo systemctl restart postgresql-16.service
```

**Create the Trendz database:**

```bash
psql -U postgres -d postgres -h 127.0.0.1 -W
```

```sql
CREATE DATABASE trendz;
```

Press `Ctrl+D` to exit.

**Configure the database connection:**

```bash
sudo nano /etc/trendz/conf/trendz.conf
```

Add the following lines. Replace `PUT_YOUR_POSTGRESQL_PASSWORD_HERE` with your actual password:

```bash
# DB Configuration
export SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/trendz
export SPRING_DATASOURCE_USERNAME=postgres
export SPRING_DATASOURCE_PASSWORD=PUT_YOUR_POSTGRESQL_PASSWORD_HERE
```

### Step 4. Run the installation script

```bash
sudo /usr/share/trendz/bin/install/install.sh
```

### Step 5. Start Trendz service

```bash
sudo service trendz start
```

## Troubleshooting

Trendz logs are stored in `/var/log/trendz`. Check for errors with:

```bash
cat /var/log/trendz/trendz.log | grep ERROR
```
