This guide explains how to install and run **Trendz Analytics** using Docker on **Linux or macOS**.

## Prerequisites

<Prerequisites product={Products.TRENDZ} />
- <a href="https://docs.docker.com/engine/installation/" target="_blank">Docker CE</a> and <a href="https://docs.docker.com/compose/install/" target="_blank">Docker Compose</a> installed.

<Aside type="caution">
Add your user to the `docker` group so commands run without `sudo`. See <a href="https://docs.docker.com/engine/install/linux-postinstall/#manage-docker-as-a-non-root-user" target="_blank">Manage Docker as a non-root user</a>.
</Aside>

## Installation steps

Trendz Analytics can be installed in two ways:

- **Run alongside ThingsBoard PE** — recommended. Trendz shares the same `docker-compose.yml` and PostgreSQL instance as ThingsBoard PE.
- **Standalone** — Trendz runs independently with its own PostgreSQL. Use this if you are not running ThingsBoard PE on the same host or prefer a fully isolated setup.

<TabItem label="Run alongside ThingsBoard PE">

### Step 1. Docker Compose setup

Create a new `docker-compose.yml`:

```bash
nano docker-compose.yml
```

Add the following content:

<Code code={composeYaml} lang="yml" meta="maxLines=25 collapsible" />

**Docker Compose parameters:**

| Parameter | Description |
|---|---|
| `8888:8888` | Maps host port 8888 to Trendz HTTP port 8888 |
| `trendz-conf:/trendz-config-files` | Persists Trendz configuration files |
| `trendz-data:/data` | Persists Trendz application data |
| `trendz-postgres-data:/var/lib/postgresql/data` | Persists PostgreSQL data |
| `SCRIPT_ENGINE_DOCKER_PROVIDER_URL` | Address of the Python Executor service |
| `SCRIPT_ENGINE_TIMEOUT` | Python script execution timeout in milliseconds |
| `restart: always` | Auto-restarts the container on failure or system reboot |

### Step 2. Start Trendz service

Run the following from the directory containing your `docker-compose.yml`:

```bash
docker compose up -d && docker compose logs -f trendz
```

Once started, open `http://localhost:8888` in your browser — you should see the Trendz login page.

## Manage containers

Detach from the log stream without stopping the container: press `Ctrl-p` then `Ctrl-q`.

| Action | Command |
|---|---|
| View Trendz logs | `docker compose logs -f trendz` |
| Stop Trendz | `docker compose stop trendz` |
| Start Trendz | `docker compose start trendz` |

## Troubleshooting

### DNS issues

If you see errors like:

```text
127.0.1.1:53: cannot unmarshal DNS message
```

Configure your system to use Google public DNS. See the <a href="https://developers.google.com/speed/public-dns/docs/using#linux" target="_blank">Linux</a> or <a href="https://developers.google.com/speed/public-dns/docs/using#mac_os" target="_blank">macOS</a> instructions.
