<Aside type="caution" title="Required only for legacy versions">
This page applies **only** if you are running **Trendz 1.14.0 or older** or **ThingsBoard 4.2.1 or older**. On newer versions these steps are handled automatically.
</Aside>

After installing Trendz Analytics, complete the following setup steps to make it ready for use.

## Signing Key

Trendz uses ThingsBoard's JWT signing key to authenticate API calls. You must configure this key after installation.

### Acquire the signing key

1. Log in to ThingsBoard as a **System Administrator**.

   <DocImage src="/src/assets/images/trendz/signing-key-4.png" alt="Log in as System Administrator" />

2. Go to **Security → General**.

   <DocImage src="/src/assets/images/trendz/signing-key-5.png" alt="Security General page" />

3. Scroll to the **JWT Security Settings** section.

   <DocImage src="/src/assets/images/trendz/signing-key-6.png" alt="JWT Security Settings section" />

4. Copy the value from the **Signing key** field.

### Configure the signing key

<Tabs>
<TabItem label="Ubuntu / CentOS/RHEL">

SSH into the server running Trendz and open the configuration file:

```bash
sudo nano /etc/trendz/conf/trendz.conf
```

Add the following line at the end of the file, replacing `<signing-key>` with the value you copied:

```bash
export JWT_TOKEN_SIGNING_KEY=<signing-key>
```

<DocImage src="/src/assets/images/trendz/signing-key-7.png" alt="Signing key in trendz.conf" />

Save the file and restart the Trendz service.

</TabItem>
<TabItem label="Docker Compose">

Open your `docker-compose.yml` and add the environment variable under the `trendz` service:

```yaml
environment:
  JWT_TOKEN_SIGNING_KEY: <signing-key>
```

<DocImage src="/src/assets/images/trendz/signing-key-8.png" alt="Signing key in docker-compose.yml" />

Save the file and restart the Trendz container.

</TabItem>
</Tabs>

### Verify the signing key

1. Log in to Trendz as a **Tenant Administrator**.
2. Go to **Settings**.

   <DocImage src="/src/assets/images/trendz/signing-key-1.png" alt="Trendz Settings navigation" />

3. Confirm that the **Signing Key** field shows **Active** status.

   <DocImage src="/src/assets/images/trendz/signing-key-3.png" alt="Signing Key Active status" />

Read more in the <DocLink product={Products.TRENDZ} path="settings#signing-key">Signing Key settings</DocLink> reference.

## Trendz Widget Bundle and JS Module

These steps add the Trendz widget bundle and JS module to your ThingsBoard installation.

<Aside type="note">
These actions modify ThingsBoard.
</Aside>

In the Trendz **Settings** page:

- **Trendz Widget Bundle** — click **Upload bundle**. You will see *Latest version installed* when done.
- **Trendz JS Summary Module** — click **Upload module**. You will see *Latest version installed* when done.

For more details see the <DocLink product={Products.TRENDZ} path="settings#trendz-widget-bundle">Trendz Widget Bundle</DocLink> and <DocLink product={Products.TRENDZ} path="settings#trendz-js-summary-module">JS Summary Module</DocLink> references.

## Link to ThingsBoard

Configure the URL of your ThingsBoard instance in Trendz settings to enable opening dashboards in a new tab directly from Trendz views.

1. Log in to Trendz and go to **Settings**.

   <DocImage src="/src/assets/images/trendz/signing-key-1.png" alt="Trendz Settings navigation" />

2. Enter the URL of your ThingsBoard instance (e.g., `https://your-thingsboard-host`) in the designated field.

   <DocImage src="/src/assets/images/trendz/link-to-tb.png" alt="ThingsBoard URL field in Trendz Settings" />

Once configured, you can open the dashboard where a Trendz view was added by enabling **Open dashboard in a separate window**.

<DocImage src="/src/assets/images/trendz/open-dashboard-in-separate-tab.png" alt="Open dashboard in separate tab option" />
